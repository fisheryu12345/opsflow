职责清晰 · 内部 API 和开放 API 物理隔离

📐 设计原则

  • 内外隔离 — 对外 API 使用独立路由前缀 (/api/v2/open/)，与内部 API 物理隔离
  • 应用级权限 — 每个第三方应用一个 ApiApp，独立 Token、独立配额、独立 IP 白名单
  • 双向 — 既支持第三方调用 OpsFlow (Open API)，也支持 OpsFlow 回调第三方 (Webhook)
  • 规范化 — 统一响应格式 (复用 DetailResponse/ErrorResponse 标准)，提供 OpenAPI 文档
