Django 数据模型设计

# itsm/models/incident.py

class Incident(CoreModel):              # 继承 dvadmin CoreModel → 自动 id/creator/.../deleted
    incident_id  = CharField(max_length=32, unique=True)
    title        = CharField(max_length=255)
    description  = TextField()
    status       = CharField(choices=INCIDENT_STATUS)  # 状态机
    priority     = CharField(choices=PRIORITY)        # P1-P4
    source       = CharField(choices=SOURCE)          # 监控告警/用户提报/API
    category     = ForeignKey(ServiceCategory)       # 服务分类
    assignee     = ForeignKey(User, null=True)       # 当前处理人
    sla_policy   = ForeignKey(SlaPolicy)
    sla_deadline = DateTimeField()                   # SLA 截止时间
    sla_status   = CharField(choices=SLA_STATUS)     # 正常/警告/违例

    # CMDB 关联 (Neo4j 节点 ID)
    cmdb_biz_id  = CharField(max_length=64, null=True)
    cmdb_host_id = CharField(max_length=64, null=True)  # 关联受影响的主机
    cmdb_data    = JSONField(default=dict)             # 快照: 资产详情

    # 来源告警 (监控联动)
    alert_id     = CharField(max_length=64, null=True)  # Grafana Alert ID
    alert_data   = JSONField(default=dict)             # 告警原始数据

SLA 体系设计

优先级 响应时间 解决时间 升级策略
P1 危急 ≤ 15 分钟 ≤ 4 小时 10 分钟未响应 → 升级到主管
P2 高 ≤ 1 小时 ≤ 8 小时 30 分钟未响应 → 升级到主管
P3 中 ≤ 4 小时 ≤ 24 小时 2 小时未响应 → 升级
P4 低 ≤ 24 小时 ≤ 72 小时 24 小时未响应 → 提醒

你关心的要点：

  • 工单通过 cmdb_biz_id / cmdb_host_id 关联 CMDB 资产 (Neo4j 节点 ID)
  • 工单创建时快照资产数据到 cmdb_data JSON 字段（防止资产变更后工单信息丢失）
  • 监控告警自动创建的事件工单自动携带 alert_id / alert_data
  • 服务请求类工单可关联 OpsFlow 执行计划实现自动交付
  • 审批节点复用 OpsFlow 现有 approval 机制
