设计细节确认

在写入正式设计文档前，确认几个实现细节

问题 1：图标来源

目前节点上显示插件类型图标，哪种方案更好？

A

前端统一映射表

在 shapes.ts 中维护一个 pluginGroup → { color, icon } 硬编码映射，不改后端

B

后端 PluginMeta 扩展

给 PluginMeta 模型加 icon 和 color 字段，API 返回给前端，灵活可配置

问题 2：节点尺寸

当前 180x48，设计稿中 220x60（多了一行副标题）。你觉得？

A

220×60，显示完整信息

标题 + 副标题（分组名/状态描述），信息密度高

B

保持 180×48，紧凑

只显示标题和图标，副标题通过 tooltip 展示
