方案三：全面 IT 运维分派体系 — 架构总览

分层架构，每层职责清晰

┌─────────────────────────────────────────────────┐
│ 接入层 — 工单入口（手动创建/API/告警联动） │
│ Ticket.create → submit → pipeline.run │
└────────────────┬────────────────────────────────┘
↓
┌─────────────────────────────────────────────────┐
│ 路由决策层 — AssignEngine │
│ ① RuleEngine 匹配路由规则（category→group） │
│ ② 值班检查 → 当前主班/备班 │
│ ③ 负载评估 → 选待办最少的成员 │
│ ④ 执行分派 → 写入 Ticket + 创建 SignTask │
└────────────────┬────────────────────────────────┘
↓
┌─────────────────────────────────────────────────┐
│ 执行层 — 处理人流转 │
│ 手动转派 / 自动升级 / SLA联动 │
│ TransferLog 全审计 │
└────────────────┬────────────────────────────────┘
↓
┌─────────────────────────────────────────────────┐
│ 监控层 — 团队面板 + 负载仪表盘 │
│ Celery 定时任务 → 超时检测 → 升级触发 │
│ 值班日历 → 排班管理界面 │
└─────────────────────────────────────────────────┘

涉及的新增/修改模块

模块类型说明
SkillGroup新增 model技能组 + 组长 + 成员
OnDutySchedule新增 model排班（日期+主备班）
EscalationLevel新增 modelL1→L2→L3 升级配置
AssignRule (路由规则)新增 modelIF category+priority THEN group
TicketTransferLog新增 model转派审计

EscalationService新增 service升级检测 + 执行

看板面板新增 vue技能组负载 + 待办统计
