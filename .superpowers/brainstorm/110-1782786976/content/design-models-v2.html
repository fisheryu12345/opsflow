方案三：数据模型设计

新增 5 个模型 + 扩展 2 个已有模型

1. SkillGroup (技能组) — 新增

一个 IT 运维团队，如"网络组""数据库组""应用组"

字段类型说明
name / codevarchar如"网络组" / "net"
leaderFK→Users组长（升级默认目标）
membersM2M→Users组员
is_activeboolean是否启用

2. ServiceCategory 扩展 — 改已有字段

原模型在 incident.py，新增两个可选字段

新增字段类型说明
default_groupFK→SkillGroup (nullable)默认分派技能组
auto_assignboolean default=false是否启用自动分派

3. OnDutySchedule (排班) — 新增

字段类型说明
groupFK→SkillGroup技能组
userFK→Users值班人
duty_datedate值班日期
duty_typechoice(primary/backup)主班/备班

4. AssignRule (路由规则) — 新增

IF 条件 THEN 动作 — 控制自动分派逻辑

字段类型说明
namevarchar规则名
priorityint匹配优先级（数字小优先）
match_categoryFK→ServiceCategory (nullable)匹配服务分类
match_priorityvarchar (nullable)匹配优先级 P1/P2/P3/P4
match_itsm_typevarchar (nullable)匹配工单类型 incident/change
target_groupFK→SkillGroup目标技能组
assign_modechoiceto_group / to_onduty / least_busy
is_activeboolean是否启用

5. EscalationLevel (升级级别) — 新增

字段类型说明
namevarchar级别名 L1/L2/L3
levelint升级顺序(1→2→3)
groupFK→SkillGroup所属技能组
timeout_minutesint超时阈值
actionchoicenotify_only / transfer_to_leader / transfer_to_next_level
notify_usersM2M→Users触发时通知

6. TicketTransferLog (转派记录) — 新增

字段类型说明
ticketFK→Incident/Ticket关联工单
from_userFK→Users(nullable)转出人（空=系统自动）
to_userFK→Users转入人
from_group / to_groupFK→SkillGroup(nullable)转入/出技能组
reasontext原因
transfer_typechoicemanual / auto_escalation / auto_assign
