分派引擎流程 — AssignEngine

工单创建/提交时自动触发，也可手动重新分派

① 触发：
  Ticket.submit() 或 Ticket.assign() 或 API 创建

② 路由匹配（优先级从低到高）：
  ┌─ this.ticket.itsm_type === rule.match_itsm_type
  ├─ this.ticket.category === rule.match_category
  ├─ this.ticket.priority === rule.match_priority
  └─ 匹配 → 使用 rule.target_group

③ 若无规则匹配，回退：
  this.ticket.category?.default_group → fallback

④ 确定人选 (assign_mode)：
  to_group → 全组均可认领 (RECEIVING 状态)
  to_onduty → 查 OnDutySchedule，今天主班优先
  least_busy → 统计各成员当前待办工单数，选最少者

⑤ 执行分派：
  TicketStatus.status = 'RECEIVING' 或 'RUNNING'
  Ticket.assignee = selected_user
  Log → TicketTransferLog(transfer_type='auto_assign')
  Notify → 通知被分派人

SLA 联动升级流程

① APScheduler 每分钟扫描：
  Ticket WHERE status='running'/assigned'
  AND sla_status IN ('warning','normal')
  AND now() > sla_deadline

② 超时 → 执行当前 EscalationLevel：
  notify_only → 发通知给 notify_users
  transfer_to_leader → 转给 group.leader
  transfer_to_next_level → 升级到下一级

③ 记录转派：
  TicketTransferLog(transfer_type='auto_escalation')
  更新 Ticket.assignee
  更新 Ticket.sla_status = 'escalated'

④ 递归触发：
  升级到下一级后，重新设置该级的超时计时
  再次超时 → 再往上升
