How it works

  • Click a tree node → right panel populates with that menu's full details (read-only view)
  • Edit button → opens the existing drawer (MenuFormCom) pre-filled for editing
  • Quick status toggle → inline switch for status/visible without opening the drawer
  • Delete → confirmation dialog, then remove
  • The existing drawer form (add/edit) stays unchanged
  • No new backend APIs needed — all data is already available from the existing GET /api/iam/menu/{id}/ endpoint
