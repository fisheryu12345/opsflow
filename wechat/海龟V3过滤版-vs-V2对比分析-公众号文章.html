海龟策略V3：震荡过滤让收益从-6.65%到+0.48%

量化交易实战系列 · 第2篇 · 2026-05-20 · 全文约2500字

摘要：V2账户（510976，无过滤）运行21天，净值从100万跌至933,512.55，亏损-6.65%。V3账户（510977，震荡过滤）运行9天，净值1,004,830.10，盈利+0.48%。两者唯一的参数差异——skip_choppy=True/False——造成了超过7个百分点的收益差距。震荡过滤是昙花一现的巧合，还是趋势跟踪系统的必备"降噪器"？本文用实盘数据深度拆解。

一、V2的困境：震荡行情中的"钝刀割肉"

V2账户（510976，skip_choppy=False）自4月30日启动，初始资金100万，截至5月20日净值933,512.55，累计亏损-6.65%。21个交易日中，资金曲线经历了两轮冲高和一轮"雪崩"：

第一轮（04-30至05-07）：净值从997,978攀升至1,008,288（+1.03%），初现盈利。但05-08迅速回落至990,614，利润回吐殆尽。

第二轮（05-08至05-11）：净值从990,614快速拉升至1,023,114（+3.28%），创下历史峰值。彼时多个持仓同时浮盈，仿佛趋势行情已至。然而……

第三轮（05-11至05-19）：6个交易日内净值从1,023,114暴跌至930,928，回撤幅度-9.01%。这一阶段是典型的"震荡绞杀"——市场并未出现单边极端行情，但多个品种同步反向波动，止损被密集触发。单个亏损3,000~5,000尚可接受，但连续6笔止损叠加，直接击穿了净值防线。

胜率14.3%（1/7），盈亏比3.7:1。亏损交易集中在8~13天的短线持仓——这是震荡假突破的典型特征：入场后短暂浮盈、随即反向、触发止损。

更令人担忧的是V2的持仓结构。截至5月20日，V2持有10个品种、共计21个单位的多空仓位，几乎覆盖了所有触发突破信号的品种——hc2610 2U多、ru2609 3U多、al2606 3U空、AP610 3U空、lc2609 2U多、SA609 1U空、ao2609 1U空、UR609 2U空、jd2607 3U多、fu2609 1U多。如此分散且无筛选的持仓，说明V2对突破信号"来者不拒"，缺乏信号质量甄别能力。

二、V3的解决方案：开仓之前先"过三关"

V3账户（510977，skip_choppy=True）与V2同步运行，初始资金同为100万。唯一的代码差异是一个布尔参数开关。这个开关控制的震荡检测逻辑，本质上是一套"开仓三关"的过滤机制：

  • 第一关：振幅是否足够？ 如果过去N日最高-最低的区间幅度小于某个ATR倍数，说明价格波动尚未有效扩张。此时发生的突破，很可能是窄幅震荡中的噪音，而非趋势的起点。
  • 第二关：通道穿越是否过于频繁？ 统计单位时间内收盘价穿越唐奇安通道上下轨的次数。高频穿越=震荡市中价格反复"假摔"；低频穿越=趋势正在形成，价格稳定在通道外侧运行。
  • 第三关：综合评分是否超限？ 前两关的指标加权生成一个震荡评分，超过阈值则判定为"震荡市"，拒绝开仓。

这三关本质上在回答同一个问题："这次突破，是真的吗？" V2从不问这个问题——只要价格突破20日高低点就进场；V3每次都问——只有通过震荡检测的突破才能建仓。这多花的一毫秒计算时间，锁住了大量潜在的假突破亏损。

核心逻辑示意：

def should_enter(symbol):
    if not price_breakout(symbol, channel_period=20):
        return False
    if skip_choppy and is_choppy_market(symbol):
        return False  # V3: 震荡市拒绝入场
    return True       # V2: 直接入场

四、深度拆解：震荡过滤在三个层面改变了什么？

1. 交易频率控制：从7笔到1笔，拒绝假突破

V2的7笔亏损交易中，有5笔（si2609、CF609、SR609、FG609、RM609）是典型的震荡假突破——持仓8~20天，平均亏损-5,804。合计亏损-29,020，占V2总亏损的43.7%。V3的过滤逻辑在同期拒绝了至少5个类似信号，相应的亏损也被有效规避。

注意V3不是"不做交易"——它做了1笔（jd2606，-1,050），且是快速认赔（仅5天）。这说明V3的过滤并非一刀切地拒绝所有信号，而是筛选出那些可信度较高的突破。当突破确实发生时，V3的入场、止损逻辑与V2完全一致——所以该亏的一样亏，只不过亏得更少、更精。

2. 持仓质量提升：21U vs 6U，少即是多

V3仅有的4个持仓（AP610 1U空、jd2607 3U多、SA609 1U空、ru2609 1U多）有一个共同特征：突破发生在波动率扩张的环境中——振幅足够、通道穿越频率低，趋势可持续性更强。而V2的10个持仓中不少品种的突破本身发生在震荡阶段，方向性差，随时可能触发止损。

持仓集中度差异的本质是信号信噪比的差异。V2的信噪比低——大量信号是噪音，资金被分散在大概率亏损的头寸中。V3的信噪比高——虽然可能错过个别趋势（这是V3的代价），但避免了资金在震荡中无效损耗。

3. 回撤悬崖消失：-9.01% vs -0.72%

最直观的差异是回撤形态。V2从5月11日峰值1,023,114跌至5月19日低谷930,928，历时6天回撤-9.01%。V3同期最大回撤仅-0.72%（从1,000,000至992,760）。注意V3也经历了连续回撤（5月14日至19日，连跌4天），但深度和速度都被有效控制。

更重要的是V3的恢复能力。5月20日V3单日反弹+1.22%，净值从992,760跃升至1,004,830，直接越过本金线。这得益于V3持仓的"低损耗"特性——在震荡期没有消耗过多本金，一旦趋势行情到来，净值的弹性远大于V2。

五、V3的代价：没有免费的午餐

震荡过滤并非万能药。V3最大的代价是可能错过趋势初期的入场点。如果市场在震荡整理后直接启动一轮大趋势，V3的过滤逻辑可能在趋势初期仍然判定为"震荡"，导致入场滞后甚至完全错过。这对于一个依赖"抓住少数大趋势"的低胜率系统来说，是致命的。

另一个潜在风险是参数过拟合。当前V3的震荡阈值是针对近期行情特征校准的。如果市场转入强趋势阶段——例如连续出现多个单边行情——V3可能过度过滤，仓位持续不足，收益大幅落后于V2。我们需要更长周期（至少3~6个月）的实盘验证来确认参数的稳健性。

此外，V3的持仓集中度更高，虽然降低了假突破亏损，但单笔亏损的冲击更大。如果6个单位中有1个出现极端反向走势，对净值的伤害比例会高于V2的21个单位分散持仓。好在V3使用与V2完全相同的三层止损轮询（14:57 + 15:32 + 09:02），单笔风险仍被严格控制在预设范围内。

使用边界提醒：

强趋势市场 V3可能弱于V2 过滤掉部分有效信号
横盘震荡市场 V3显著优于V2 设计场景，优势最大
V型反转行情 V3可能滞后 过滤需要确认时间
品种波动差异大 需单独校准参数 统一参数效果不佳

六、总结：震荡过滤是趋势跟踪的"降噪器"

从V2到V3，我们只做了一件事——在开仓前加了一道震荡检测。但就是这道过滤，把收益率从-6.65%拉升至+0.48%，把最大回撤从-9.01%压缩至-0.72%。在一套完整的止损体系（自适应止损、保本止损、缺口保护、三层轮询）之上叠加一道信号筛选，产生了远超预期的效果。

核心结论如下：

  1. 趋势跟踪的最大敌人不是趋势反转，而是震荡。海龟策略本质上是低胜率系统（30%~40%），震荡期连续假突破会把胜率压到15%以下，资金曲线急速下滑。
  2. 震荡过滤是"降噪器"而非"放大器"。它不会让你在趋势中赚更多，但能让你在震荡中亏更少。这种不对称的收益风险特征长期来看对复利增长极为有利。
  3. 简单规则往往最有效。V3仅基于ATR和通道穿越频率的两条规则，就产生了显著效果。"简单且鲁棒"优于"复杂且脆弱"。
  4. 需要更长的验证周期。9天的样本量（V3）虽然对比鲜明，但不足以评估策略的长期表现。关键观察窗口：当市场出现一波明显趋势时，V3能否充分参与。
  5. 多账户并行是策略迭代的最佳基础设施。V2和V3同时运行，在真实市场条件下进行A/B测试，这是回测无法替代的验证手段。

下一步优化方向：

  • 动态过滤阈值：根据品种波动特性自动调整震荡判定参数，避免统一参数导致的品种适配问题。
  • 多时间框架确认：在日线突破基础上加入小时线趋势方向的一致性确认，进一步提高信号质量。
  • 板块分组过滤：黑色系、化工、农产品等板块内部相关性较高，板块整体震荡时，板块内所有品种的信号都应受限制。
