"我想写一个期货自动交易程序，从哪里开始？"

这是后台收到最多的读者提问。做量化交易，数据接口和交易接口是最基础也最让人头疼的一步——券商的 CTP 接口门槛高、文档晦涩，自己爬数据又面临反爬、实时性差、历史数据不完整等一系列问题。

TqSDK（天勤量化）是目前国内最流行的期货量化 SDK 之一，它把繁杂的底层通信封装成了 Python 调用，让开发者可以用十几行代码就拿到实时行情、K 线数据和账户信息。本文从零开始，带你走完安装→数据→下单→策略的全流程。

本文目标读者：有 Python 基础、想入门期货量化但不知道从哪里开始的开发者。不需要期货实盘经验，不需要 CTP 或易盛等接口知识。
◆ ◆ ◆

一、TqSDK 是什么？

TqSDK 是上海天勤推出的一款 Python 量化 SDK，它做了三件最重要的事：

功能说明替代方案对比
行情数据Tick / K线 / 五档行情vs 爬虫：实时、稳定、全品种
交易接口下单、撤单、持仓查询vs CTP：Python 原生，不用 protobuf
回测/仿真本地模拟 + 快交所模拟vs 文华：代码可控，回测可嵌入流程

TqSDK 的核心理念是 "一个 API 搞定所有"：无论是期货公司直连、快交所模拟还是本地回测，API 调用方式完全一致。你只需要在创建账号时切换参数，业务代码一行都不用改。

架构示意：
你的策略代码 → TqSDK API → TqSdk 内核 → 【期货公司 / 快交所 / 本地仿真】
切换后端时，只需改动 TqAccount(期货公司) 或 TqKq(快交所) 或 TqSim(本地) 这一行代码。
◆ ◆ ◆

二、安装与连接

安装

一行命令搞定：

pip install tqsdk

依赖 Python 3.8+，支持 Windows / macOS / Linux。安装完成后，在天勤官网注册账号获取 license（免费版即可使用行情和本地仿真交易）。

创建连接

TqSDK 中一切操作都围绕 TqApi 实例展开：

from tqsdk import TqApi, TqAuth

# 本地仿真账户（无需期货账号，适合学习）
api = TqApi(auth=TqAuth("your_phone", "your_pwd"))

# 期货公司实盘/仿真（需要期货账号）
# api = TqApi(TqAccount("期货公司", "账号", "密码"), auth=TqAuth("手机号", "密码"))

# 快交所模拟
# api = TqApi(TqKq(), auth=TqAuth("手机号", "密码"))

初学者强烈建议从 TqSim（本地仿真）开始。它不依赖任何外部交易所，完全在本地模拟撮合，可以无限次练习而不用担心亏钱。

重要提醒：TqSDK 使用 WebSocket 协议保持长连接，创建 api 后不会立即断开。程序退出前需要调用 api.close() 释放资源，否则下次启动可能因 license 冲突报错。

三、获取行情数据

TqSDK 的数据获取 API 非常直观。两种最常用的数据：实时行情（Quote）和 K 线（KlineSerial）。

📊 实时行情 Quote

quote = api.get_quote("SHFE.rb2510")
print(quote.last_price) # 最新价
print(quote.ask_price1) # 卖一价
print(quote.bid_price1) # 买一价
print(quote.volume) # 成交量

合约代码格式：交易所.品种代码（如 SHFE.rb2510、DCE.m2509、CZCE.MA501）。TqSDK 支持全市场品种：上期所(SHFE)、大商所(DCE)、郑商所(CZCE)、中金所(CFFEX)、能源中心(INE)。

📈 K 线数据 KlineSerial

# 获取60秒K线，最多100根
klines = api.get_kline_serial("SHFE.rb2510", 60, data_length=100)

# klines 是 DataFrame，包含字段：
# datetime / open / high / low / close / volume / open_oi
print(klines.iloc[-1]) # 最新一根K线

K 线周期可选：60(1分钟)、300(5分钟)、3600(1小时)、86400(日线)。注意 data_length 是最大缓存条数，不是"获取多少条"——TqSDK 会在内核中持续维护这个序列，新数据到达时自动追加。

核心概念：get_quote 和 get_kline_serial 返回的不是快照，而是实时更新的引用。每次 wait_update() 调用时，这些对象内部的数据会自动刷新。
◆ ◆ ◆

四、wait_update —— TqSDK 的心脏

如果你只记住 TqSDK 的一个知识点，请记住这个：所有数据更新和交易动作，都需要通过 wait_update() 来驱动。

没有调用 wait_update()，quote 里的价格永远是创建瞬间的静态值，订单会卡在"已发送"状态不会被交易所确认，TargetPosTask 也不会执行任何调仓动作。

while True:
  api.wait_update()
  if api.is_changing(quote.last_price):
    print(f"最新价更新: {quote.last_price}")
  if api.is_changing(klines.iloc[-1], "close"):
    print(f"K线收盘价更新: {klines.close.iloc[-1]}")

⏰ 三个关键函数

函数作用新手常见错误
wait_update()阻塞等待新数据/新事件忘了调用 → 数据永远不更新
is_changing(obj, field)判断某个字段是否改变了直接用 if 判断值 → 重复触发
get_trade_log()获取成交记录在 wait_update 前查询 → 空列表
💡 死循环 vs 带超时的循环：
wait_update() 默认永久阻塞。生产环境建议设置超时：api.wait_update(timeout=60)。如果超时内没有数据更新，函数返回 False，你可以做健康检查或重连。

五、下单交易

TqSDK 提供了两套下单方式，分别对应不同的需求层次：

方式一：直接下单 insert_order

order = api.insert_order(
  symbol="SHFE.rb2510",
  direction="BUY",  # BUY 或 SELL
  offset="OPEN",    # OPEN 或 CLOSE
  volume=1,
  limit_price=quote.limit_up  # 不指定则市价
)

# 等待订单成交
while order.status != "FINISHED":
  api.wait_update()

方式二：目标持仓 TargetPosTask（推荐）

对于大多数策略场景，你关心的是"我想持有多少手"，而不是"下一笔怎么下"。TargetPosTask 封装了自动调仓逻辑：

from tqsdk import TargetPosTask

# 创建目标持仓任务
target_pos = TargetPosTask(api, "SHFE.rb2510")

# 设置目标：持有 5 手多单
target_pos.set_target_volume(5)

# wait_update 驱动调仓（自动拆单、追单）
while True:
  api.wait_update()

TargetPosTask 会自动计算当前持仓和目标持仓的差值，生成逐笔订单并执行。你不需要关心当前持有哪些合约、需要下几笔、用什么价格——TqSDK 帮你处理了。

实践经验：TargetPosTask 是我们的主力下单方式。在实盘运行中，它处理了数千笔调仓，包括开仓、加仓、止损平仓、移仓换月等各种场景。配合 wait_update 驱动循环，行为稳定可预期。

六、第一个迷你策略：双均线交叉

把前面的知识串起来，写一个完整的交易程序：5分钟K线双均线交叉策略。

from tqsdk import TqApi, TqAuth, TargetPosTask
from tqsdk.tafunc import ma

api = TqApi(auth=TqAuth("your_phone", "your_pwd"))
symbol = "SHFE.rb2510"

# 获取5分钟K线
klines = api.get_kline_serial(symbol, 300, data_length=50)
target_pos = TargetPosTask(api, symbol)

while True:
  api.wait_update()

  # 只在K线收盘时检查信号
  if api.is_changing(klines.iloc[-1], "close"):
    close = klines.close
    ma10 = ma(close, 10)
    ma30 = ma(close, 30)

    if ma10.iloc[-1] > ma30.iloc[-1] and ma10.iloc[-2] <= ma30.iloc[-2]:
      target_pos.set_target_volume(2)  # 金叉开多
    elif ma10.iloc[-1] < ma30.iloc[-1] and ma10.iloc[-2] >= ma30.iloc[-2]:
      target_pos.set_target_volume(0)  # 死叉平多

这段代码中值得注意的点：

行作用如果写错了会怎样
is_changing检测K线收盘不检查：每个tick都重复开仓
ma10.iloc[-1] vs ma10.iloc[-2]检测交叉（而非单纯比较）只用大小判断：金叉后永远满足条件
TargetPosTask自动调仓不用：需要自己管理订单状态机
扩展阅读：上述代码可在 TqKq（快交所）或 TqSim（本地仿真）中直接运行，零成本测试。我们系统里所有策略（海龟V2/V3、HVOB-MBI等）都运行在类似架构上——一个永久的 wait_update 循环 + 信号判断 + TargetPosTask 执行。你完全可以从这个迷你策略开始，逐步替换成自己的信号逻辑。

七、新手最常踩的 4 个坑

#问题表现解决
1忘记 wait_update数据不更新、订单不执行必须放在主循环中驱动
2合约代码过期get_quote 返回 None每月更新合约后缀（rb2510→rb2511）
3is_changing 误用反复触发信号只在新K线收盘时检查
4非交易时间下单订单排队无效检查 quote.is_trading
💡 合约换月提醒：主力合约通常每 3 个月换月一次。TqSDK 提供了 api.find_contract(symbol_pattern) 可以按品种前缀自动发现当前主力合约。建议在程序中定期检测，不要硬编码合约代码。
