# 注意事项

## 为什么修改了 dvadmin 下的代码

实现 IAM 时需要修改 `dvadmin/system/views/menu.py`，原因如下：

`web_router` 接口控制了**每个用户能看到哪些菜单**。现有的 RBAC 逻辑是：用户只能看到自己被授权角色绑定的菜单。但 IAM 的核心是 **"没有权限的用户也能申请权限"**——这就产生了矛盾：

- 不改 `web_router`：用户看不到"权限申请"菜单 → 无法申请
- 改了 `web_router`：IAM 菜单对所有人可见 → 用户才能点进去申请

修改的内容只是在 `menu.py` 的非超级用户分支里加了 IAM 菜单 ID，不影响其他逻辑：

```python
# 在原有角色菜单基础上，额外加上IAM菜单（所有人可见）
iam_catalog = Menu.objects.filter(web_path='/iam', is_catalog=True).first()
if iam_catalog:
    iam_menu_ids = list(Menu.objects.filter(
        Q(id=iam_catalog.id) | Q(parent=iam_catalog)
    ).values_list('id', flat=True))
    all_menu_ids |= set(iam_menu_ids)  # 合并到最终菜单列表
```

## 子页面 web_path 不能与父级相同

权限申请（子页面）的 `web_path` 不能与权限管理（父级目录）相同。两者都是 `/iam` 会导致 Vue Router 路由冲突，页面渲染空白。

目前子页面使用 `web_path="/iam_apply"`。如果将来修改路径，确保与父级目录路径不同。
