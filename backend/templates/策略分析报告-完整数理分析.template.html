二、策略一：原版海龟交易法则

T

Account 6 — 510988（商品期货原版）

2.1 策略定义

最接近原版海龟交易法则的实盘实现：纯唐奇安 20 日突破开仓，无 MA 趋势过滤，无震荡行情跳过。金字塔加仓最多 3 Unit，动态 ATR 止损。

DonchianHigh[t] = max(Phigh[t-20], ..., Phigh[t-1])
DonchianLow[t] = min(Plow[t-20], ..., Plow[t-1])

做多: Close[t] > DonchianHigh && units == 0
做空: Close[t] < DonchianLow && units == 0

2.2 核心参数

参数值含义
突破周期20 日唐奇安通道
风险基数¥Equity×1%每单位风险 = 权益×1% / (2×N×乘数)
最大单位3 U金字塔上限（原版为4）
止损倍数2.0 × N固定2N硬止损（入场 N 锁定）
保本机制无原版海龟无保本兜底
跳空保护无盘中实时入场，无跳空拦截

2.3 盘中实时开仓

原版海龟在符合最小开仓量要求时使用盘中实时突破入场，而非等待 T+1 开盘执行。这是与原版海龟信号执行框架的主要差异点。

重要： Acc 6 直接使用 Turtle 突破入场逻辑，盘中实时触发。与 Acc 1/5 的"收盘信号->次日开盘执行"框架不同。

三、策略二：海龟V2增强版

V2

Account 1 — 510976（商品期货加强V2）

3.1 V2 增强特性

海龟V2 在保留 20 日唐奇安突破核心框架的基础上引入以下改进：

增强项机制效果
趋势因子自适应止损MA10/20/40 乖离 -> [0, 0.5] 连续映射止损宽度 2.0~3.0 ATR 连续调整
保本机制盈利 > 2.5xATR 且满仓 3U -> 保本价兜底防止大幅回吐
跳空保护同向跳空 > 1.5xATR 时跳过开仓反向跳空不拦截
三层止损收盘后(15:32) + 临收盘(14:57) + 次日开盘(09:02/21:02)多道防线
T+1 信号执行收盘信号 -> 次日开盘执行避免盘中假突破
V2 定位： 标准增强版。不过滤震荡行情（skip_choppy=False），追求捕捉所有趋势信号。2026-05-26 实盘启动，初始权益 ¥502,000，配置 12 个品种，暂无历史绩效数据。

五、策略四：HVOB-MBI 日内高波动突破系统

H

Account 10 — 510978

5.1 策略概述

High Volatility Opening Range Breakout x Market Breadth Index — 纯日内系统，当日开仓当日平仓。核心思想：在高波动品种的开盘区间突破方向入场，通过 MBI 控制整体仓位暴露。

5.2 运行流程

21:00 夜盘开盘-> 21:30 区间锁定-> 夜盘突破-> 09:00 日盘开盘-> 跳空检查-> 09:30 MBI计算-> 日盘突破-> 14:55 强制平仓

5.3 MBI 多空判定

每个品种投票 [-3, +3] = 隔夜跳空 +/-1 + 区间重心 +/-1 + 突破方向 +/-1
总得分 -> 归一化到 [0,1] -> 5 个 MBI 标签

极强多头 >0.75: 做多重1.0, 做空0.0
多头 >0.65: 做多1.0, 做空0.5
中性 0.45~0.65: 多空均1.0
空头 0.35~0.45: 做多0.5, 做空1.0
极强空头 <=0.35: 做多0.0, 做空1.0

5.4 风险计算

风险敞口 = 余额 x 0.5%
止损距离 = R x 0.2 x 2
手数 = max(1, 风险敞口 / (止损距离 x 合约乘数))

八、仓位计算数学建模

原版海龟：
Unitlots = max(1, int( Equity × 1% / (2 × N × Vmultiple) ))
Equity = TradingAccount.current_equity, N = ATR(20)

V2/V3：
Unitlots = max(1, round( R / (ATR × 2 × Vmultiple) ))
R = ¥4,000

8.2 HVOB-MBI

risk = balance x 0.5%
stop_dist = R x 0.2 x 2 = 0.4R
volume = max(1, risk / (stop_dist x Vmultiple))
