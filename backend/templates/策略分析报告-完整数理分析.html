一、策略架构总览

FT 系统是一个基于 海龟交易法则变体 的日线级别商品期货趋势跟踪系统。核心决策逻辑采用"收盘信号生成 → 次日开盘批量执行"的信号驱动架构，通过 Django + APScheduler 实现定时任务调度。

三、开仓逻辑 — 唐奇安突破

3.1 突破判定

以 不含当日 的前 20 日最高/最低价为通道边界（唐奇安通道），使用 check_breakout_signal() 函数（core/indicators.py:34）：

DonchianHigh[t] = max(Phigh[t-20], ..., Phigh[t-1])  (不含当日)
DonchianLow[t] = min(Plow[t-20], ..., Plow[t-1])

做多条件: Close[t] > DonchianHigh[t] && units == 0
做空条件: Close[t] < DonchianLow[t] && units == 0

3.2 纯唐奇安突破（无 MA 过滤）

与回测版本不同，实盘交易系统 不采用 MA 排列过滤。开仓信号仅由唐奇安通道突破决定。这是有意为之的设计——MA 过滤虽能减少假信号，但也会错过部分早期趋势启动。

元素回测版本实盘系统取舍
开仓条件唐奇安突破 + MA 排列过滤纯唐奇安突破实盘捕捉更多趋势，但增加约 30% 信号
预期胜率42%~45%38%~42%略有下降，但总利润可能更高

3.3 跳空保护（方向区分）

使用 price_gap_protection()（core/atr.py:51）：

多头只拦截: (最新价 - 前收盘) / ATR > 1.5  →  跳过（防止追高）
空头只拦截: (前收盘 - 最新价) / ATR > 1.5  →  跳过（防止追低）
反向跳空视为有利，不拦截

3.4 V3 增强：震荡行情跳过（可选）

当 skip_choppy_entry = True 且 trend_label in ('choppy', 'neutral') 时取消开仓。

四、仓位计算 — ATR 位置规模

4.1 Unit 手数公式

Unitlots = max(1, round( R / (ATR × k × Vmultiple) ))

其中: R = ¥4,000 (风险基数)   k = 2 (风险倍数)   Vmultiple = 合约乘数

实现：calculate_unit_lots()（core/position_sizing.py:8）。从 api.get_quote() 获取合约乘数，ATR 通过 calculate_atr() 预计算。

4.3 最小开仓限制（两步开仓）

当计划开仓手数 < 交易所最小开仓手数 (min_position) 时，使用 execute_two_step_opening()（infrastructure/order_execution.py:102）：

Step 1: 开仓 min_position 手（满足交易所规则）
Step 2: 立即平仓 (min_position - planned_volume) 手
结果: 持仓 = planned_volume 手

五、趋势因子模型

5.1 均线间距计算

gap10_20 = |MA10 - MA20| / |MA20|
gap20_40 = |MA20 - MA40| / |MA20|
max_gap = max(gap10_20, gap20_40)

5.2 趋势强度连续映射

trend_strength = min(max_gap / Glimit, 1.0)  ∈ [0, 1]
trend_factor = round(trend_strength × Fmax, 3)  ∈ [0, 0.5]

止损倍数 = 2 × (1 + trend_factor)  ∈ [2.0, 3.0]

5.3 趋势标签

trend_strength标签止损倍数含义
≥ 80%strong_bull/strong_bear3.0 × ATR强趋势，放宽止损防被洗出
30% ~ 80%weak_bull/weak_bear2.6~2.9 × ATR弱趋势，适度放宽
< 30%choppy2.0 × ATR震荡，收紧止损
关键设计： 采用连续线性映射而非阶梯跳变，避免止损倍数在标签边界处突变。实盘 vs 回测差异：回测中 MA 过滤额外过滤 choppy 信号，实盘纯唐奇安突破 + 连续 factor 自适应。

6.2 加权均价与风险分析

场景: 螺纹钢 ATR=50, Unitlots=4 手/Unit

开仓 1U @ 5000 + 加仓 1U @ 5060 + 加仓 1U @ 5120
加权均价 = (5000×4 + 5060×4 + 5120×4) / 12 = 5060

止损价 (factor=0.3): 5150 - 2×1.3×50 = 5020
止损亏损 = (5020 - 5060) × 12 × 10 = -¥4,800

对比 1U 不加仓: (5020 - 5000) × 4 × 10 = -¥800
加仓后亏损放大 6 倍
核心风险： 加仓后成本价上移，回调时利润回吐更快。3 Unit 满仓时的回撤幅度远大于 1 Unit。加仓对年化贡献约 +3%~+5%/年。

七、动态止损系统

7.1 动态跟踪止损

多头: StopLoss = highest_close - 2 × (1 + trend_factor) × ATR
空头: StopLoss = lowest_close + 2 × (1 + trend_factor) × ATR

特性: highest_close 只升不降, lowest_close 只降不升
范围: 2.0 ATR (震荡) ~ 3.0 ATR (强趋势)

实现：update_all_positions_stop_loss_price()（tasks_daily_close.py:375）。纯函数版本：compute_stop_loss()（core/stop_loss.py:8）。

7.2 保本兜底（第二层保护）

触发条件: 浮动盈利 > 2.5 × ATR && units ≥ 3（仅满仓激活）
保本价: 成本价 ± tick × 2（按品种 price_tick 动态计算）

最终止损 = max(动态止损, 保本价)  (多头)
最终止损 = min(动态止损, 保本价)  (空头)

7.3 三层止损触发

检查时机触发条件执行方式代码位置
15:32 收盘后收盘价击穿止损价生成 STOP_LOSS 信号，次日开盘执行check_exit_signals()
14:57 临收盘最新价击穿止损价（实时行情）立即市价平仓（紧急风控）check_and_execute_stop_loss()
09:02/21:02 开盘执行 PENDING 的 STOP_LOSS 信号TargetPosTask 平仓process_signals_by_type()

7.4 双向极值跟踪

update_all_positions_high_low_price()（tasks_daily_close.py:335）已改为双向跟踪——无论多空，同时跟踪最高价和最低价。多头止损只用 highest_close，空头止损只用 lowest_close，因此双向跟踪不影响止损逻辑，但保证 MFE/MAE 计算的完整性。

八、保本兜底机制

8.1 触发逻辑

require profit > PROTECT_COST_ENABLED_RATIO × ATR (= 2.5 × ATR)
require units ≥ 3（仅满仓激活）
若已触发过一次, protect_cost_enabled = True (单向开关，永久保持)

8.2 保本价磁铁效应分析

保本价磁铁效应： 价格在剧烈波动后容易被吸引至保本价附近。当前偏移量 tick×2 在高波动品种中可能偏小。优化建议：将固定 tick×2 改为 min(tick×2, 0.3×ATR)，高波动品种获得更大保本偏移。

九、信号执行引擎

9.1 TargetPosTask 机制

target_pos = TargetPosTask(api, symbol, target_position=volume)
wait_for_target_position(api, target_pos, symbol, volume, timeout=60)
target_pos.cancel() // 释放资源

TqSDK 自动处理：报单 → 追单 → 撤单重报 → 直到持仓达到 target_position。

9.2 信号状态流转

PENDING → EXECUTING → SUCCESS
PENDING → FAILED / CANCELLED

9.3 防重复机制

check_duplicate_pending_signal()（core/signal_checker.py:8）：同 (account, symbol, trade_type) 组合只允许一个 PENDING 信号存在。在信号生成和执行阶段都进行检查。

9.4 两步开仓策略

当 planned_volume < min_position 时使用 execute_two_step_opening()（infrastructure/order_execution.py:102）。min_position 从 FullContractList 读取。

9.5 滑点记录

record_slippage()（infrastructure/slippage_recorder.py:17）在每次成交后记录信号价与实际成交价的偏差。区分有利/不利滑点：建仓时多头希望买低、空头希望卖高；平仓时多头希望卖高、空头希望买低。

十三、与原版海龟深度对比

维度原版海龟当前系统差异分析
开仓信号唐奇安 20 日突破（无过滤）唐奇安 + (可选) skip_choppy_entry实盘纯突破，V3 可跳过 choppy

十四、回测与实盘差异

#维度回测实盘影响

十五、收益数学期望

15.1 单笔交易期望

EV = Win% × W - (1 - Win%) × L

中性估计: EV = 0.40 × 2.5R - 0.60 × 1.0R = +0.40R
每 Unit 风险 R = ¥4,000
单笔期望值 = ¥1,600

15.2 年化收益推导
