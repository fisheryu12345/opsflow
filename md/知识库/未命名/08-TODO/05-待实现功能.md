# 待实现功能清单

> 基于系统全面审查（2026-05-10），运行一年后出具完整绩效报告所需的功能和数据采集改进。
> 按优先级排列，P0=必须、P1=强烈建议、P2=锦上添花。

---

## P0 — 运行一年后出具全面报告必须

### P0-1: 月度/季度/年度自动化绩效报告

**现状**：只有每日信号邮件报告，无任何定期绩效报告。

**需求**：
- 每月1日自动生成上月绩效报告（月度收益率、最大回撤、胜率、盈亏比）
- 每季度生成季度报告（含品种盈亏排行、资金曲线、回撤分析）
- 每年生成年度报告（含全部指标、与回测预期对比）

**实现方式**：新增 APScheduler 定时任务 `tasks_monthly_report.py` / `tasks_quarterly_report.py`，输出 HTML 或 PDF

**涉及文件**：新建 `backend/stock/scheduler/tasks_report.py`

**预计工作量**：3~5天

---

### P0-2: 资金入金/出金记录

**现状**：`TradingAccount.initial_balance` 是固定值，中途资金变动会导致收益率计算偏差。

**需求**：新增 `CapitalFlow` 模型记录每一笔入金/出金：
- `account`, `amount`(正=入金/负=出金), `flow_date`, `remark`
- 收益率计算时根据资金流调整：`真实收益率 = (期末权益 - 期初权益 - 净入金) / 期初权益`

**涉及文件**：`backend/stock/models.py` + serializer + viewset + frontend

**预计工作量**：2天

---

### P0-3: 按品种盈亏分解存储

**现状**：`DailyEquitySnapshot` 只记录总权益和总浮动盈亏，无法按品种分解。

**需求**：
- 方案A（推荐）：在 `DailyEquitySnapshot` 或新表 `SymbolDailyPnl` 中记录每日每个持仓品种的盈亏贡献
- 字段：`account`, `trade_date`, `symbol`, `pnl`(日盈亏), `float_pnl`(浮动盈亏)

**涉及文件**：`backend/stock/models.py` + `tasks_daily_close.py` 中绩效更新逻辑

**预计工作量**：1~2天

---

## P1 — 强烈建议（大幅提升报告质量）

### P1-1: 回测结果存入数据库

**现状**：回测运行在独立的内存环境中，结果不写入数据库，无法与实盘数据对比。

**需求**：
- 新增 `BacktestRun` 模型记录每次回测的配置和结果摘要
- 新增 `BacktestEquityCurve` 模型存储回测资金曲线
- 前端 Dashboard 可叠加显示"回测预期曲线"和"实盘实际曲线"
- 回测/实盘指标对比表

**涉及文件**：`backend/stock/models.py` + `backend/stock/backtest/reporter.py`(改造) + frontend

**预计工作量**：3~5天

---

### P1-2: 滑点统计分析

**现状**：完全没有滑点记录，无法评估"信号价 vs 实际成交价"的偏差。

**需求**：
- 在 `DailyStrategySignal` 或新表 `SlippageRecord` 中记录：
  - 信号生成时的理论价（收盘价）
  - 实际成交均价
  - 滑点 = 成交价 - 信号价
  - 滑点方向（有利/不利）
- 生成滑点统计报告：平均滑点、滑点分布、最大滑点

**涉及文件**：`backend/stock/infrastructure/order_execution.py` + `backend/stock/models.py`

**预计工作量**：1~2天

---

### P1-3: 信号执行统计

**现状**：`executed_status` 记录了状态，但没有成体系的执行统计。

**需求**：
- 信号执行率：PENDING → SUCCESS 的转化率
- 按交易类型统计：ENTRY/STOP_LOSS/ADD_ON/ROLLOVER 各自的成功率
- 平均执行延迟：信号生成时间 → 开始执行时间 → 完成执行时间
- 执行失败原因分布

**涉及文件**：`backend/stock/views/performance.py`(新增统计接口) + frontend

**预计工作量**：1~2天

---

## P2 — 锦上添花（有更好，没有也能出报告）

### P2-1: 委托单级别明细记录

**现状**：只有平仓汇总记录，没有委托单的完整生命周期。

**需求**：新增 `OrderRecord` 模型记录每笔委托：
- 关联信号 + 委托状态变更日志（已报/部分成交/全部成交/已撤单）
- 每次报单价格、数量、成交明细
- 用于深度分析报单策略效果和交易所反馈

**涉及文件**：`backend/stock/models.py` + `backend/stock/infrastructure/order_execution.py`

**预计工作量**：2~3天

---

### P2-2: 市场环境标记

**现状**：系统没有量化标记市场环境状态。

**需求**：
- 每日收盘后计算并存储市场环境标签：
  - 波动率状态（高/中/低，基于 20 日 ATR 百分位）
  - 趋势状态（趋势/震荡，基于 ADX 或趋势因子）
  - 板块相关性（各板块间的平均相关系数）
- 报告生成时按市场环境分类统计绩效

**涉及文件**：`backend/stock/core/indicators.py` + `backend/stock/models.py`

**预计工作量**：2天

---

### P2-3: 盘中实时权益采样

**现状**：只有每日收盘后的权益快照，缺少日内数据。

**需求**：
- 每分钟（或每5分钟）记录一次账户权益，存入 `RealtimeEquitySnapshot`
- 用于分析日内最大回撤、滑点冲击、日内收益分布
- 可选：只在开盘时段采样

**涉及文件**：`backend/stock/models.py` + 新增定时任务或 TqSDK 事件回调

**预计工作量**：2~3天

---

### P2-4: 回测系统集成到 Django 管理后台

**现状**：回测通过 CLI 运行，无 Web 界面。

**需求**：
- 在 Django admin 中添加回测触发按钮
- 回测进度显示（WebSocket 或轮询）
- 回测结果在线预览（不依赖本地 HTML 文件）

**涉及文件**：`backend/stock/admin.py` + `backend/stock/views/backtest.py` + frontend

**预计工作量**：3~5天

---

## 实施路径建议

```
第1阶段（运行前准备好）：
  ├── P0-2 资金入金/出金记录 ← 如果中途会追加资金，这个必须在运行前做好
  ├── P1-2 滑点统计分析     ← 运行第一天就需要记录
  └── P1-3 信号执行统计     ← 已有数据，只需开发统计接口

第2阶段（运行中开发）：
  ├── P0-3 按品种盈亏分解
  ├── P2-3 盘中实时权益采样
  └── P2-2 市场环境标记

第3阶段（满1年出具报告前完成）：
  ├── P0-1 月度/季度/年度报告
  ├── P1-1 回测结果存入数据库
  ├── P2-1 委托单级别明细
  └── P2-4 回测集成到管理后台
```

> **优先级建议**：P0-2（资金记录）是数据准确性的根基，建议在实盘运行前就做好。其余功能可以在实盘运行过程中逐步开发，只要在出具年度报告前完成即可。
