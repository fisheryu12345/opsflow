## P0 — 运行一年后出具全面报告必须

### P0-1: 月度/季度/年度自动化绩效报告

**状态**：✅ **已实现**（2026-05-10）

**实现内容**：
- `backend/stock/infrastructure/pdf_report_generator.py` — 报告数据计算与 PDF 生成
- `backend/templates/performance_report.html` — 报告 HTML 模板（A4 优化）
- `backend/stock/scheduler/tasks_report.py` — 三个定时任务函数
- `backend/stock/scheduler/scheduler.py` — 注册 cron：
  - 月度：每月 1 日 09:30
  - 季度：1/4/7/10 月 1 日 09:30
  - 年度：1 月 1 日 10:00
- `backend/stock/tasks/send_mail.py` — `send_report_email_task` 支持 PDF 附件

**报告包含指标**：
- KPI：累计盈亏、收益率、交易次数、胜率、最大回撤
- 权益变动流程（期初 → 变动 → 期末）
- 月度收益明细（复合收益率、日胜率）
- 品种盈亏分析（按品种盈亏排行、品种胜率）
- 回撤分析（最大回撤率/金额、当前回撤）
- 滚动绩效指标（120日夏普、索提诺、波动率、盈亏比）
- 账户总览（累计/年化收益率、卡尔玛、盈利因子、连胜负、极端值）

**待改进**：
- 回测对比需等 P1-1 回测结果入库后才可加入报告
- PDF 中文渲染依赖系统字体，部署时需确认中文字体安装

### P0-2: 资金入金/出金记录

**现状**：`TradingAccount.initial_balance` 是固定值，中途资金变动会导致收益率计算偏差。

**需求**：新增 `CapitalFlow` 模型记录每一笔入金/出金：
- `account`, `amount`(正=入金/负=出金), `flow_date`, `remark`
- 收益率计算时根据资金流调整：`真实收益率 = (期末权益 - 期初权益 - 净入金) / 期初权益`

**涉及文件**：`backend/stock/models.py` + serializer + viewset + frontend

**预计工作量**：2天

---

### P0-3: 按品种盈亏分解存储

**状态**：✅ **已实现**（2026-05-10）

**实现内容**：
- 新增 `SymbolDailyPnl` 模型，字段：`account`, `trade_date`, `symbol`, `product_code`, `realized_pnl`(已实现盈亏), `float_pnl`(浮动盈亏), `volume`(持仓手数)
- 新增 `save_symbol_daily_pnl()` 函数在 `backend/stock/core/performance.py`
- 数据来源：`ClosedPositionRecord` 聚合 realized_pnl + `PositionState` 市值计价计算 float_pnl
- 自动在每日收盘绩效更新流程 (`update_all_performance_metrics`) 末尾调用
- 迁移文件: `stock/migrations/0071_symboldailypnl.py`

**涉及文件**：
- `backend/stock/models.py` — SymbolDailyPnl 模型
- `backend/stock/core/performance.py` — save_symbol_daily_pnl 函数 + 在 update_all_performance_metrics 中调用

## P1 — 强烈建议（大幅提升报告质量）

### P1-1: 回测结果存入数据库

**现状**：回测运行在独立的内存环境中，结果不写入数据库，无法与实盘数据对比。

**需求**：
- 新增 `BacktestRun` 模型记录每次回测的配置和结果摘要
- 新增 `BacktestEquityCurve` 模型存储回测资金曲线
- 前端 Dashboard 可叠加显示"回测预期曲线"和"实盘实际曲线"
- 回测/实盘指标对比表

**涉及文件**：`backend/stock/models.py` + `backend/stock/backtest/reporter.py`(改造) + frontend

**预计工作量**：3~5天

---

### P1-2: 滑点统计分析

**状态**：✅ **已实现**（2026-05-10）

**实现内容**：
- 新增 `SlippageRecord` 模型：`account`, `symbol`, `product_code`, `trade_type`, `trade_date`, `signal_price`, `fill_price`, `slippage`, `slippage_ticks`, `is_favorable`, `position_direction`, `volume`
- 新增 `backend/stock/infrastructure/slippage_recorder.py` — `record_slippage()` 工具函数
- ENTRY 滑点：在 `order_signals.py:execute_entry_order` 中记录，signal_price=Donchian通道价
- ADD_ON 滑点：在 `order_signals.py:execute_add_on_order` 中记录，signal_price=上次加仓价
- EXIT 滑点：在 `order_signals.py:execute_exit_order` 中记录，signal_price=止损价
- STOP_LOSS 滑点：在 `stop_loss_executor.py:execute_stop_loss_exit` 中记录，signal_price=止损价
- 有利/不利滑点自动判断（按交易方向和类型）
- 滑点以价格点和最小变动价位(ticks)两种单位存储

---

### P1-3: 信号执行统计

**状态**：✅ **已实现**（2026-05-10）

**实现内容**：
- 后端 API `GET /api/stock/signal-stats/` 提供：
  - 总体概况：总数、成功/失败/取消/待执行、整体成功率
  - 按交易类型统计：ENTRY/ADD_ON/STOP_LOSS/EXIT/ROLLOVER 各自的成功率
  - 失败原因分布（按 remark 字段聚合 Top 10）
  - 平均执行延迟（updated_at - created_at）
- 前端页面 `web/src/views/signal/signal-stats/`：
  - 日期范围过滤器
  - 7 个统计卡片（总数、成功、失败、取消、待执行、成功率、平均延迟）
  - 按交易类型统计表（含成功率进度条）
  - 失败原因分布表
- 通过 Django admin 添加菜单条目即可访问

## P2 — 锦上添花（有更好，没有也能出报告）

### P2-1: 委托单级别明细记录

**现状**：只有平仓汇总记录，没有委托单的完整生命周期。

**需求**：新增 `OrderRecord` 模型记录每笔委托：
- 关联信号 + 委托状态变更日志（已报/部分成交/全部成交/已撤单）
- 每次报单价格、数量、成交明细
- 用于深度分析报单策略效果和交易所反馈

**涉及文件**：`backend/stock/models.py` + `backend/stock/infrastructure/order_execution.py`

**预计工作量**：2~3天

---

### P2-2: 市场环境标记

**现状**：系统没有量化标记市场环境状态。

**需求**：
- 每日收盘后计算并存储市场环境标签：
  - 波动率状态（高/中/低，基于 20 日 ATR 百分位）
  - 趋势状态（趋势/震荡，基于 ADX 或趋势因子）
  - 板块相关性（各板块间的平均相关系数）
- 报告生成时按市场环境分类统计绩效

**涉及文件**：`backend/stock/core/indicators.py` + `backend/stock/models.py`

**预计工作量**：2天

---

### P2-3: 盘中实时权益采样

**现状**：只有每日收盘后的权益快照，缺少日内数据。

**需求**：
- 每分钟（或每5分钟）记录一次账户权益，存入 `RealtimeEquitySnapshot`
- 用于分析日内最大回撤、滑点冲击、日内收益分布
- 可选：只在开盘时段采样

**涉及文件**：`backend/stock/models.py` + 新增定时任务或 TqSDK 事件回调

**预计工作量**：2~3天

---

### P2-4: 回测系统集成到 Django 管理后台

**现状**：回测通过 CLI 运行，无 Web 界面。

**需求**：
- 在 Django admin 中添加回测触发按钮
- 回测进度显示（WebSocket 或轮询）
- 回测结果在线预览（不依赖本地 HTML 文件）

**涉及文件**：`backend/stock/admin.py` + `backend/stock/views/backtest.py` + frontend

**预计工作量**：3~5天

## 实施路径建议

```
第1阶段（运行前准备好）：
  ├── P0-2 资金入金/出金记录 ← 如果中途会追加资金，这个必须在运行前做好
  ├── ✅ P1-2 滑点统计分析     ← 已完成
  ├── ✅ P1-3 信号执行统计     ← 已完成

第2阶段（运行中开发）：
  ├── ✅ P0-3 按品种盈亏分解 ← 已完成
  ├── P2-3 盘中实时权益采样
  └── P2-2 市场环境标记

第3阶段（满1年出具报告前完成）：
  ├── ✅ P0-1 月度/季度/年度报告 ← 已完成
  ├── P1-1 回测结果存入数据库
  ├── P2-1 委托单级别明细
  └── P2-4 回测集成到管理后台
```

> **优先级建议**：P0-2（资金记录）是数据准确性的根基，建议在实盘运行前就做好。其余功能可以在实盘运行过程中逐步开发，只要在出具年度报告前完成即可。
