   - `tasks_exit_before_close.py` — 重写为逐账户模式，每个账户独立 API 连接
   - `stop_loss_executor.py` — `check_and_execute_stop_loss(api, account=)` 新增可选 account 参数

### P0-5: 累计手续费字段与盘中查询定时任务

**状态**：✅ **已实现**（2026-05-11）

**需求背景**：
系统原累计手续费仅存储在 `AccountPerformanceSummary.commission_total`（需聚合查询），TradingAccount 层缺少直接字段。新增盘中 14:58 定时任务提前捕获当日手续费，避免收盘任务异常时数据丢失。

**实现内容**：

1. **模型**（`backend/stock/models.py`）
   - `TradingAccount.total_commission` — 累计手续费字段，默认 0，每日自动更新

2. **定时任务**（`backend/stock/scheduler/tasks_daily_commission.py`）
   - `job_daily_commission_query()` — 每日 14:58 执行
   - 遍历所有活跃账户，创建对应模式的 TqApi 连接
   - 获取 `api.get_account()` 中的 commission、balance 等数据
   - 创建/更新当日 `DailyEquitySnapshot`（含手续费数据）
   - 聚合所有 `DailyEquitySnapshot.commission` 并更新 `TradingAccount.total_commission`
   - 共用 Redis 分布式锁 `lock:daily_commission`，防止重复执行

3. **绩效计算**（`backend/stock/core/performance.py`）
   - `update_all_performance_metrics()` 在 `save_daily_snapshot` 后调用 `update_trading_account_total_commission()` 同步累计手续费
   - `update_account_summary()` 直接读取 `account.total_commission`（不再聚合 DailyEquitySnapshot）

**涉及文件**：
- `backend/stock/models.py` — TradingAccount.total_commission 字段
- `backend/stock/migrations/0074_tradingaccount_total_commission.py` — 迁移文件
- `backend/stock/scheduler/tasks_daily_commission.py` — 定时任务
- `backend/stock/scheduler/scheduler.py` — 注册 cron（14:58）
- `backend/stock/core/performance.py` — 绩效计算中直接读取
- `backend/stock/core/performance.py` — 绩效计算同步
- `web/src/stores/account.ts` — AccountItem 接口新增 total_commission

### P0-2: 资金入金/出金记录

**现状**：`TradingAccount.initial_balance` 是固定值，中途资金变动会导致收益率计算偏差。

**需求**：新增 `CapitalFlow` 模型记录每一笔入金/出金：
- `account`, `amount`(正=入金/负=出金), `flow_date`, `remark`
- 收益率计算时根据资金流调整：`真实收益率 = (期末权益 - 期初权益 - 净入金) / 期初权益`

**涉及文件**：`backend/stock/models.py` + serializer + viewset + frontend

**预计工作量**：2天

### P1-2: 滑点统计分析

**状态**：✅ **已实现**（2026-05-10）

**实现内容**：
- 新增 `SlippageRecord` 模型：`account`, `symbol`, `product_code`, `trade_type`, `trade_date`, `signal_price`, `fill_price`, `slippage`, `slippage_ticks`, `is_favorable`, `position_direction`, `volume`
