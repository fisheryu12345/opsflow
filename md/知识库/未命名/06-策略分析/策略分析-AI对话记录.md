## 一、保本机制设计分析

### 1.1 原有 Bug：保本即锁定

**问题**：原代码中，一旦保本启用，止损价被永远锁在保本价：

```python
if position.protect_cost_enalbed:
    new_stop_loss = position.stop_loss_price   # ← 永远读数据库里旧的保本价写回
```

**后果**：价格继续上涨，动态止损本应跟着上移，但被保本价卡死。导致本该继续跟踪的止损停在原地，很容易被回调触发。

### 1.2 修复方案：保本是底线

修复后的两层逻辑：

```
第一层：动态跟踪止损（主角）
  多头：highest_close - 2×(1+factor)×ATR   只升不降
  空头：lowest_close + 2×(1+factor)×ATR    只降不升

第二层：保本兜底（安全垫）
  启用条件：盈利 > 2.5 × ATR
  保本价：成本价 ± tick × 2（按品种最小变动价位）
  
  多头最终止损 = max(动态止损, 保本价)
  空头最终止损 = min(动态止损, 保本价)
```

**单向开关**：`protect_cost_enalbed` 一旦开启永久保持，不会因价格回落而关闭。开关状态只会 0 → 1 变化一次。

### 1.3 是否过度设计？

按不同场景分析保本兜底的实际生效情况：

| 场景 | factor | 动态止损距成本 | 保本触发时 | 兜底是否生效 | 意义 |
|------|--------|--------------|-----------|------------|------|
| 弱趋势/震荡 | 0 | 2.0×ATR = 6 | 盈利6点时，止损=100，保本=101 | 仅提升1点 | 微乎其微，心理安慰 |
| 强趋势 | 0.5 | 3.0×ATR = 12 | 盈利6点时，止损=94，保本=101 | 提升7点到保本价 | 意义重大，防止盈利回吐 |
| ATR跳升 | 0→0.5 | ATR从3跳到12 | 止损从114跌到96 | 兜底到101 | 波动加剧时守住底线 |

**结论**：不是过度设计。强趋势和波动加剧场景下保本兜底有实质保护作用。弱趋势场景下效果微弱，但无负面影响。

### 1.4 保本价计算改进

**旧方案**：固定 `±1` 元

| 品种 | 成本价 | 保本偏移 | 偏移比例 | 效果 |
|------|--------|---------|----------|------|
| 螺纹 rb | 3200 | 1 元 | 0.03% | 有效 |
| 碳酸锂 lc | 198800 | 1 元 | 0.0005% | 几乎无效 |
| 黄金 au | 600 | 1 元 | 0.17% | 过宽 |

**新方案**：`成本价 ± tick × 2`

| 品种 | tick | 保本偏移 | 效果 |
|------|------|---------|------|
| 螺纹钢 | 1 | 2 元 | 合理 |
| 碳酸锂 | 50 | 100 元 | 合理 |
| 黄金 | 0.02 | 0.04 元 | 合理 |
| 股指 IF | 0.2 | 0.4 点 | 合理 |

### 1.5 关键设计决定

| 问题 | 讨论过程 | 最终决定 | 理由 |
|------|---------|---------|------|
| 保本价偏移量 | ±1 → 按tick → 按百分比 | tick×2 | 跨品种合理，实现简单 |
| 保本启用阈值 | 2.0 vs 2.5 vs 3.0×ATR | 2.5×ATR | 平衡过早保本和过度回撤 |
| 保本质 | 绝对底线 vs 软保护 | 底线+动态跟踪 | 动态止损为主，保本不拖后腿 |

## 二、趋势分类灵敏度分析

### 2.1 问题发现

在三级阶梯分类（strong/weak/choppy）下，factor 在阈值处存在不连续跳变。分析发现：

**跳变路径与影响**：

| 跳变路径 | 触发条件 | factor变化 | 止损宽度变化 |
|---------|---------|-----------|-------------|
| strong → weak | 任一个 gap 跌破 STRONG | 0.5 → 0 | 3.0→2.0 ATR（收窄33%） |
| weak → choppy | 两个 gap 都低于 WEAK | 0 → -0.2 | 2.0→1.6 ATR（收窄20%） |
| choppy → weak | 任一个 gap 突破 WEAK | -0.2 → 0 | 1.6→2.0 ATR（放宽25%） |
| weak → strong | 两个 gap 都高于 STRONG | 0 → 0.5 | 2.0→3.0 ATR（放宽50%） |

### 2.2 跳变的量化分析

**核心发现**：价格回调约 0.2%（10点左右），即可导致 factor 从 0.5 跳到 0（止损收窄33%）。

以中波动品种（MA20=5000, ATR=50）为例：

```
Day 1: price=5055, MA10=5051, MA20=5000
        gap_10_20 = 1.02% > 1.0% → strong_bull, factor=0.5, 止损=150点

Day 2: price=5045（回调10点）
        MA10更新后≈5049
        gap_10_20 = 0.98% < 1.0% → weak_bull, factor=0, 止损=100点

结论：10点回调导致止损收窄50点（33%突变）
```

### 2.3 灵敏度问题的根源

**根因一：硬边界阈值**

阶梯函数在阈值处存在不连续跳变，没有平滑过渡：

```python
# 旧逻辑（三级阶梯）
if gap > STRONG and gap > STRONG:
    factor = 0.5
elif gap > WEAK or gap > WEAK:
    factor = 0
else:
    factor = -0.2
```

**根因二：双条件非对称性**

- strong → weak：两个条件需同时满足，只要一个失败就降级（易出）
- weak → strong：两个条件需同时满足（难进）
- 导致：strong 状态"易出难进"

**根因三：动态阈值的自我强化**

```
r = ATR / MA20
STRONG = max(base_strong, r × multiplier)

趋势越强 → ATR 越大 → r 上升 → STRONG 阈值被拉高 → 更难达到 strong
```

这与直觉相反：强趋势时应该放宽止损，但阈值机制在"抑制"放宽。

### 2.4 改进：三级阶梯 → 连续线性映射

**方案**：将阶梯函数改为连续值映射

```python
# 当前逻辑（连续线性）
gap_10_20 = abs(ma_10 - ma_20) / abs(ma_20)
gap_20_40 = abs(ma_20 - ma_40) / abs(ma_20)
max_gap = max(gap_10_20, gap_20_40)

trend_strength = min(max_gap / TREND_GAP_LIMIT, 1.0)  # TREND_GAP_LIMIT = 0.03
trend_factor = round(trend_strength * TREND_FACTOR_MAX, 3)  # TREND_FACTOR_MAX = 0.5
```

**改进效果对比**：

| 场景 | 旧逻辑（阶梯） | 新逻辑（连续） |
|------|--------------|--------------|
| 价格回调0.2%（10点） | factor 从0.50跳到0（33%突变） | factor 从0.50平滑降到0.47（~6%微降） |
| 高位横盘3天 | strong↔weak↔choppy来回跳 | 在0.5→0.3→0.2之间平滑下降 |
| 新开仓日差异 | 同价开仓，止损差50%（3.0 vs 2.0 ATR） | 同价开仓，止损差<10% |

**旧逻辑 vs 新逻辑的止损宽度变化趋势**（以 max_gap 从 0.2% 增长到 3.0% 为例）：

| max_gap | 旧逻辑（阶梯） | 新逻辑（连续） |
|---------|--------------|--------------|
| 0.2% | 2.0 ATR | 2.07 ATR |
| 0.5% | 2.0 ATR | 2.17 ATR |
| 0.9% | 2.0 ATR | 2.30 ATR |
| **1.0%** | **3.0 ATR（跳涨）** | **2.33 ATR（微涨）** |
| 1.5% | 3.0 ATR | 2.50 ATR |
| 2.0% | 3.0 ATR | 2.67 ATR |
| 3.0% | 3.0 ATR | 3.00 ATR |

## 四、边界条件与注意事项

### 4.1 加仓信号的"严格大于"

```python
if price_diff > Decimal('1') * atr_value:      # 严格 >
elif price_diff > Decimal('0.5') * atr_value:  # 严格 >
```

如果 ΔP 恰好等于 1.0×ATR 或 0.5×ATR，不会触发加仓。实际运行中因 Decimal 精度问题，精确相等的概率极低。

### 4.2 1Unit时直接满仓的跳变

价格从 P0 直接跳到 P0 + 1.1×ATR，会一次性加 2 Unit（从1到3），跳过了2Unit阶段。这是设计意图——避免在快速趋势中踏空。

### 4.3 没有"部分减仓"

系统中不存在 3→2 或 2→1 的部分减仓。只有全部平仓（止损）或等量移仓。

### 4.4 加仓执行 vs 信号生成的时滞

加仓信号在收盘后（15:02）生成，次日开盘（09:02）执行。执行价与信号价可能存在隔夜跳空差异。`last_add_price` 记录的是实际成交价，而非信号生成价。

### 4.5 高波动品种的"strong 陷阱"

高波动品种（如股指期货）：
- ATR/MA20 比值高 → 动态阈值被拉高
- STRONG≈2.9%，需要 MA 差距 145点以上
- 即使处于明显单边趋势，也可能被长期判定为 weak
- 始终使用 2.0ATR 止损，但 ATR=120 时 2.0ATR=240点，不一定偏紧

### 4.6 低波动品种的"过度宽松"

低波动品种（如债券期货）：
- ATR/MA20 比值低 → 动态阈值低
- STRONG≈0.5%，MA 差距 25点即可判定 strong
- 容易进入 strong 状态，使用 3.0ATR 宽止损
- ATR=20 时 3.0ATR=60点，一次正常回调就可能回吐大量利润
