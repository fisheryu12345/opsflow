# 回测系统架构与设计

## 概述

回测系统是一个独立于实盘交易的离线模拟引擎，基于 Python + Pandas + NumPy 构建，不使用 Django ORM 或 TqSDK（数据加载阶段除外）。它以日K线为最小时间粒度，模拟海龟趋势跟踪策略的完整交易周期。

**代码位置**: `backend/stock/backtest/`  
**数据位置**: `backend/stock/backtest/data/`（主连CSV + 个体合约CSV）  
**结果位置**: `backend/stock/backtest/results/`（各品种 + 投资组合）

---

## 核心设计

### 1. Day-by-Day Close→Open 循环 (engine.py)

回测引擎以"日"为最小模拟粒度，每天执行两个步骤：

```
Day T-1 Close → 生成信号 (Signal) → Day T Open → 执行信号 → Day T Close → MTM → ...
```

**循环伪代码**:
```python
for i in range(lookback, len(timeline) - 1):
    today = tl.iloc[i]          # 今天的 K 线
    tomorrow = tl.iloc[i + 1]   # 明天的 K 线
    
    # Open 步骤：执行昨日收盘生成的信号
    execute_signals_at_open(account, signals, today_open, ...)
    
    # Close 步骤：基于今日收盘判断信号
    signals = run_close_step(position, today, klines_window, config, ...)
    
    # 移仓跳空调整
    adjust_rollover_gap(rollover_gap)
    
    # 逐日盯市
    update_mtm(today_close)
```

### 2. 信号生成 (strategy.py run_close_step)

每个交易日收盘后，按优先级依次判断：

| 优先级 | 信号类型 | 触发条件 |
|--------|----------|----------|
| 1 | STOP_LOSS | 持仓中，收盘价触及动态止损价 |
| 2 | ENTRY | 无持仓，收盘价突破唐奇安通道 + MA趋势确认 |
| 3 | ROLLOVER | 持仓中，主力合约发生切换 |
| 4 | ADD_ON | 持仓中，价格朝有利方向移动超过 N×ATR |

### 3. 信号执行 (strategy.py execute_signals_at_open)

开盘后按固定优先级顺序执行：

1. **STOP_LOSS** → 市价平仓（开盘价）
2. **ENTRY** → 检查跳空保护 → 滑点调整 → 开仓
3. **ROLLOVER** → 双边滑动 → 平旧开新
4. **ADD_ON** → 滑点调整 → 加仓

### 4. 账户模型 (broker.py)

全部为纯 Python 数据类，不使用数据库：

| 数据类 | 字段 | 说明 |
|--------|------|------|
| `BacktestAccount` | cash, position, equity_curve, trades | 账户主对象 |
| `PositionState` | direction, units, cost_price, highest/lowest_close, atr_value, stop_loss_price | 持仓状态 |
| `TradeRecord` | date, type, direction, price, volume, pnl | 交易记录 |
| `EquitySnapshot` | date, total_equity, cash, unrealized_pnl | 日终快照 |

## 关键技术实现

### 移仓跳空调整 (engine.py)

连续合约在主力切换时产生价格跳空，导致虚假盈亏。回测引擎检测到跳空后，对持仓成本进行平移：

```python
if rollover_gap != 0 and position.is_open:
    position.cost_price += rollover_gap      # 成本平移
    position.first_open_price += rollover_gap  # 首次开仓价平移
    position.last_add_price += rollover_gap    # 最后一次加仓价平移
```

**无需真实移仓操作**，成本平移消除了跳空的虚假 PnL，持仓可以"跨越"换月继续持有。

### 止损计算 (strategy.py _compute_stop)

与实盘完全一致：
- **多头**: `止损失 = 最高收盘价 - 2×(1+factor)×ATR`，向下取整到 tick
- **空头**: `止损失 = 最低收盘价 + 2×(1+factor)×ATR`，向上取整到 tick

区别：回测使用浮点数运算，实盘使用 Decimal。

### 突破开仓 + MA 趋势过滤 (strategy.py)

**开仓条件 = 唐奇安通道突破 + MA趋势确认**:
1. 收盘价突破 20 日唐奇安上轨 → 潜在多头信号
2. 收盘价跌破 20 日唐奇安下轨 → 潜在空头信号
3. **MA 过滤器**：趋势标签不为 'choppy'，且突破方向与趋势方向一致（多头突破需要 bull 趋势，空头突破需要 bear 趋势）

### 加仓逻辑 (indicators.py compute_add_on_units)

海龟金字塔加仓：

| 当前单位数 | 基准价格 | 加仓条件 | 加仓数量 |
|------------|----------|----------|----------|
| 1 | last_add_price | 价格朝有利方向移动 > 0.5×ATR | 1 单位 |
| 1 | last_add_price | 价格朝有利方向移动 > 1.0×ATR | 2 单位（跳级满仓） |
| 2 | first_open_price | 累计移动 > 1.0×ATR | 1 单位 |

## 配置参数 (BacktestConfig)

| 参数 | 默认值 | 对应实盘配置 | 说明 |
|------|--------|-------------|------|
| `risk_base_amount` | 4000 | POSITION_RISK_BASE_AMOUNT | 每单位风险金额 |
| `risk_multiplier` | 2 | POSITION_RISK_MULTIPLIER | 风险乘数 |
| `max_units` | 3 | POSITION_MAX_UNITS | 最大持仓单位数 |
| `protect_cost_ratio` | 2.5 | PROTECT_COST_ENABLED_RATIO | 保本触发倍数 |
| `gap_protection_ratio` | 1.5 | GAP_PROTECTION_RATIO | 跳空保护倍数 |
| `trend_gap_limit` | 0.03 | TREND_GAP_LIMIT | 趋势强度归一化阈值 |
| `trend_factor_max` | 0.5 | TREND_FACTOR_MAX | 最大趋势因子 |
| `initial_capital` | 1,000,000 | — | 初始资金 |
| `slippage` | 1 tick | — | 滑点（tick 数） |

## 使用方式

### 单品种回测

```bash
# 螺纹钢 2019-2026
python -m stock.backtest.run_backtest --product rb

# 自定义参数
python -m stock.backtest.run_backtest --product hc --start 2020 --end 2025 --capital 500000

# 输出目录: backend/stock/backtest/results/{product}_{start}_{end}/
```

### 投资组合回测

```bash
# 全部品种
python -m stock.backtest.run_portfolio_backtest

# 指定品种子集 + 共享资金池
python -m stock.backtest.run_portfolio_backtest --products rb,hc,MA,SA --total-capital 5000000

# 仅下载数据（不运行回测）
python -m stock.backtest.run_portfolio_backtest --download-only

# 跳过下载，直接使用已有数据
python -m stock.backtest.run_portfolio_backtest --skip-download
```

### 数据下载

```bash
# 单独更新某个品种的主连数据
python -m stock.backtest.download_data --product rb

# 下载个体合约数据（用于构建OI时间线）
python -m stock.backtest.download_data --product rb --include-contracts
```
