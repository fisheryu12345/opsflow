# 回测系统使用指南

## 数据准备

### 数据目录结构

```
data/
├── rb_main_daily_20190101_20260509.csv    # 主力连续合约 K 线
├── rb1905_daily_20190101_20190515.csv      # 个体合约 K 线
├── rb1910_daily_20190516_20190915.csv
├── hc_main_daily_20190101_20260509.csv
├── ...
└── products.json                           # 24品种元数据缓存
```

主连 CSV 命名格式：`{品种代码}_main_daily_{起始日期}_{截止日期}.csv`  
个体合约 CSV 命名格式：`{品种代码}{年份}{月份}_daily_{起始日期}_{截止日期}.csv`

### 下载数据

```bash
cd backend

# 下载全部品种数据
python -m stock.backtest.run_portfolio_backtest --download-only

# 下载单个品种
python -m stock.backtest.download_data --product rb

# 下载单个品种的个体合约（用于OI时间线）
python -m stock.backtest.download_data --product rb --include-contracts

# 强制覆盖已存在数据
python -m stock.backtest.run_portfolio_backtest --download-only --overwrite-data
```

> ⚠️ TqSDK 下载需要约 60 秒等待时间。全部 24 个品种约需 2-3 分钟。

## 运行回测

### 单品种回测

```bash
# 螺纹钢（默认）
python -m stock.backtest.run_backtest

# 热卷 - 指定区间和资金
python -m stock.backtest.run_backtest --product hc --start 2020 --end 2025 --capital 500000

# 静默模式（只输出摘要）
python -m stock.backtest.run_backtest --product i --quiet
```

**输出示例**：

### 投资组合回测

```bash
# 全品种默认（约 5-10 分钟）
python -m stock.backtest.run_portfolio_backtest

# 快速测试（4个品种）
python -m stock.backtest.run_portfolio_backtest --products rb,hc,MA,SA

# 共享资金池模式（总资金小于各品种独立资金之和）
python -m stock.backtest.run_portfolio_backtest --capital 500000 --total-capital 3000000
```

### HTML 报告

`portfolio_report.html` 是一个自包含文件（约 1.2MB），内嵌 ECharts。包含：

- **资金曲线 + 回撤**：组合权益走势及下方回撤区域
- **月度收益热力图**：绿赚红亏，纵轴年份、横轴月份
- **品种盈亏排行**：横向柱状图，红绿区分盈亏
- **绩效指标对比表**：品种间 CAGR、夏普、回撤对比

## 参数调整

### 修改策略参数

编辑 `backend/stock/backtest/config.py` 中的 `BacktestConfig` 默认值，或运行时通过 CLI 参数传入：

```bash
# 下不修改 config.py，直接构造自定义配置
python -c "
from stock.backtest.config import BacktestConfig
from stock.backtest.engine import BacktestEngine
from stock.backtest.data_loader import build_daily_timeline

config = BacktestConfig(
    product='rb',
    initial_capital=2_000_000,
    risk_base_amount=6000,     # 提高风险敞口
    gap_protection_ratio=2.0,  # 放宽跳空保护
)
timeline = build_daily_timeline('rb')
engine = BacktestEngine(config, timeline)
results = engine.run()
print(results['account'].total_equity)
"
```

## 常见问题

### Q: 回测结果显示"数据不足"
A: 该品种的 CSV 文件可能不完整或天数不够。检查 `data/` 目录下的对应文件，确保至少有 70 个以上交易日数据。

### Q: 投资组合中有品种显示 FAILED
A: 部分品种（如 `UR`、`AP`）可能上市时间较晚，2019 年初的数据不足。不影响其他品种的回测。

### Q: HTML 报告太大无法打开
A: 默认 HTML 约 1-2MB，包含所有品种的详细图表。如果品种过多，可以减少 `--products` 参数缩小范围。

### Q: 回测结果与实盘有差异
A: 参见《回测与实盘差异分析》文档。主要原因包括：
1. 回测有 MA 趋势过滤，实盘没有
2. 回测使用收盘价判断信号，实盘使用实时行情
3. 回测假设以开盘价成交，实盘存在滑点和流动性影响
