当前系统经过模拟盘测试，数据库中积累了模拟交易数据。准备切换到实盘前，需要清理模拟数据，确保实盘运行的起始状态干净、准确。

## 清理项清单（按优先级排列）

### P0 - 必须处理（影响实盘交易正确性）

#### 1. StrategyConfig.is_simulation 改为 False

- **现状：** 510976、510977、510988 全部为 `is_simulation=True`
- **影响：** 定时任务和下单逻辑中可能根据此标志走模拟分支
- **操作：** 
  ```sql
  UPDATE stock_strategyconfig SET is_simulation=0 
  WHERE account_id IN (1, 5);
  ```
- **注意：** 先确认 510988 是否要参与实盘，再决定是否修改

#### 2. 清除 PositionState 模拟持仓

- **现状：** 510976 有 8 个活跃持仓（units>0），包括：
  - `GFEX.lc2609` units=2 多
  - `DCE.jd2606` units=3 多
  - `SHFE.rb2610` units=3 多
  - `CZCE.AP610` units=1 空
  - `CZCE.CF609` units=1 多
  - `SHFE.al2606` units=2 空
  - `SHFE.ru2609` units=3 多
  - `GFEX.si2609` units=3 多
  - 另有 14 个 units=0 的记录
- **影响：** 不清除的话，系统会认为这些品种已有持仓，不会生成入场信号，且开仓均价等数据完全错误
- **操作：** 删除 510976 的全部 PositionState 记录（包括 units=0 的记录，实盘后会重新生成）
  ```sql
  DELETE FROM stock_positionstate WHERE account_id = 1;
  ```
- **注意：** 510977 和 510988 无持仓，无需处理

### P1 - 必须处理（影响绩效数据展示）

#### 3. 清除 510976 的模拟绩效数据

- **现状：**
  - `DailyEquitySnapshot`：7 条记录（2026-04-30 ~ 2026-05-12）
  - `ClosedPositionRecord`：2 条记录，total_pnl = -6930
  - `SymbolDailyPnl`：9 条记录
  - `RollingPerformanceMetrics`：N 条记录
  - `AccountPerformanceSummary`：1 条记录
- **影响：** Dashboard 首页展示的累计收益、夏普比率、最大回撤等全部基于模拟数据，实盘后混在一起无法区分
- **操作：**
  ```sql
  DELETE FROM stock_dailyequitysnapshot WHERE account_id = 1;
  DELETE FROM stock_closedpositionrecord WHERE account_id = 1;
  DELETE FROM stock_symbol_daily_pnl WHERE account_id = 1;
  DELETE FROM stock_rollingperformancemetrics WHERE account_id = 1;
  DELETE FROM stock_accountperformancesummary WHERE account_id = 1;
  ```
- **注意：** 510977/510988 的 1 条快照（2026-05-12）也是测试产生的，建议一并清除
  ```sql
  DELETE FROM stock_dailyequitysnapshot WHERE account_id IN (5, 6);
  DELETE FROM stock_accountperformancesummary WHERE account_id IN (5, 6);
  ```

#### 4. 清除 SlippageRecord

- **现状：** 3 条滑点记录（模拟盘产生）
- **操作：**
  ```sql
  DELETE FROM stock_slippagerecord;
  ```

#### 5. 重置 TradingAccount.current_equity 为初始值

- **现状：** 510976 的 `current_equity=1,023,114.39`（模拟盈亏后的值），510977/510988 的 `current_equity=1,000,000`
- **操作：** 510976 重置为 `initial_balance` 的值（当前为 1,000,000）
  ```sql
  UPDATE stock_tradingaccount SET current_equity = initial_balance WHERE id = 1;
  ```

### P2 - 建议处理

#### 6. 清除模拟 TradeLog

- **现状：** 573 条交易日志，包含模拟期间的信号、开仓、平仓等记录
- **理由：** 实盘日志混在其中难以区分，且日志量不大
- **操作：**
  ```sql
  DELETE FROM stock_tradelog;
  ```
- **备选：** 如果希望保留历史记录，可不清理

#### 7. 清理无效的 AccountContractConfig

- **现状：** 3 个 Inactive 账户（id=2,3,4）有完整的 107 个品种配置记录（`sync_contracts --seed` 自动生成）
- **操作：**
  ```sql
  DELETE FROM stock_accountcontractconfig WHERE account_id IN (2, 3, 4);
  ```

#### 8. 清理 TradingAccount 中的无效账户

- **现状：** id=2,3,4 三个账户（`is_active=False`）由 `post_save` signal 自动创建
- **操作：** 保留或删除均可，不影响业务逻辑。建议删除以保持整洁

### P3 - 保留不变

#### 9. KlineData（保留）

- **现状：** 17,464 条日K线（2016-01-03 ~ 2026-05-10）
- **理由：** 这是市场行情数据，与账户无关，实盘继续使用。且有 `--skip-kline` 参数，重刷成本高

#### 10. FullContractList（保留）

- **现状：** 77 个品种的合约信息
- **理由：** 合约基础信息，实盘继续使用

#### 11. DailyStrategySignal（保留或清除均可）

- **现状：** 无 pending 信号
- **理由：** 信号数据仅用于复盘参考，不清除不影响实盘。如果已通过绩效数据清除间接关联，可以保留

## 清理执行顺序

```
Step 1: StrategyConfig.is_simulation = False
Step 2: DELETE PositionState       (510976 模拟持仓)
Step 3: DELETE 绩效数据             (510976 + 510977 + 510988)
Step 4: DELETE SlippageRecord      (全部)
Step 5: UPDATE current_equity      (510976 重置)
Step 6: DELETE TradeLog            (可选)
Step 7: DELETE AccountContractConfig (无效账户)
Step 8: DELETE TradingAccount      (无效账户, 可选)
```
