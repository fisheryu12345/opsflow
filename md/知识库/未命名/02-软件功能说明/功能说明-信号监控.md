# 信号监控功能说明

## 功能概述

信号监控模块展示策略每日生成的交易信号，包括开仓、加仓、止损、移仓四种类型的信号记录。用于追踪策略决策过程、排查漏单/误单问题。

---

## 信号类型

| 类型 | 代码 | 说明 | 执行时机 |
|------|------|------|---------|
| 开仓 | `ENTRY` | 唐奇安突破开仓信号 | 开盘任务（09:02/21:02） |
| 加仓 | `ADD_ON` | 金字塔加仓信号 | 开盘任务 |
| 止损 | `STOP_LOSS` | 动态跟踪止损触发 | 开盘任务 / 14:57 临收盘 |
| 移仓 | `ROLLOVER` | 主力合约换月 | 开盘任务 |

---

## 执行状态

| 状态 | 说明 |
|------|------|
| `PENDING` | 待执行（信号已生成，等待开盘处理） |
| `EXECUTING` | 执行中（临收盘止损时标记） |
| `SUCCESS` | 执行成功 |
| `FAILED` | 执行失败（超时/异常） |
| `CANCELLED` | 已取消（跳空保护/超限/非交易时间） |

---

## 核心字段

| 字段 | 说明 |
|------|------|
| `trade_date` | 信号生成的交易日期 |
| `symbol` | 合约代码 |
| `trend_factor` | 信号生成时的趋势因子值 |
| `trend_label` | 信号生成时的趋势标签（强多/弱多/震荡/弱空/强空） |
| `donchian_upper` | 唐奇安上轨（开仓信号时有效） |
| `donchian_lower` | 唐奇安下轨（开仓信号时有效） |
| `is_breakout` | 是否突破 |
| `signal_direction` | 信号方向（1=多，-1=空，0=无） |
| `trade_type` | 交易类型（ENTRY/ADD_ON/STOP_LOSS/ROLLOVER） |
| `executed_status` | 执行状态 |
| `contract_target_number` | 目标新增单位数（加仓信号时有效） |
| `remark` | 备注（突破说明、过滤原因、失败原因等） |

---

## 防重复机制

- 同一品种、同一 `trade_type`、同一天只生成一个 `PENDING` 信号
- 若前一日信号未执行（仍为 PENDING），次日不会重复生成
- 执行失败的信号不会自动重试，需人工排查后手动处理

---

## 常见备注说明

| 备注内容 | 含义 |
|---------|------|
| "收盘价 xxx 突破唐奇安上轨 xxx" | 正常开仓信号 |
| "跳空保护" | 跳空幅度 > 1.5×ATR，信号被取消 |
| "加仓后将超限" | 加仓后 units > 3，信号被取消 |
| "多头止损触发: 最新价 xxx < 止损价 xxx" | 正常止损信号 |
| "需要移仓到新主力合约 xxx" | 移仓信号 |
