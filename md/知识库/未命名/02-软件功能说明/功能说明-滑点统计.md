# 滑点统计功能说明

## 功能概述

记录每笔交易的理论触发价与实际成交价之间的偏差（滑点），提供多种维度的统计分析，帮助评估交易执行质量和优化下单参数。

滑点数据在交易执行时由调度任务自动记录，通过 Dashboard 首页的指标卡片和品种滑点统计图表展示。

---

## 页面位置

- **滑点记录查询**: API 接口提供（暂无独立管理页面）
- **滑点统计概览**: Dashboard 首页 → 指标卡片 → "平均滑点/有利滑点率"
- **品种滑点图表**: Dashboard 首页 → 品种滑点统计（柱状图，位于品种胜率统计与品种盈亏排行之间）

## 数据模型

### SlippageRecord（`backend/stock/models.py:598`）

| 字段 | 类型 | 说明 |
|------|------|------|
| `account` | FK→TradingAccount | 所属账户 |
| `signal` | FK→DailyStrategySignal | 关联信号（可选） |
| `symbol` | CharField | 合约代码（交易所前缀格式） |
| `product_code` | CharField | 品种代码 |
| `trade_type` | CharField | 交易类型: ENTRY / EXIT / STOP_LOSS / ADD_ON |
| `trade_date` | DateField | 交易日期 |
| `position_direction` | IntegerField | 持仓方向: 1=多头, -1=空头 |
| `volume` | IntegerField | 成交手数 |
| `signal_price` | Decimal(12,2) | 信号触发时的理论价格 |
| `fill_price` | Decimal(12,2) | 实际成交均价 |
| `slippage` | Decimal(12,2) | 滑点(价格点) = fill_price - signal_price |
| `slippage_ticks` | Decimal(8,1) | 滑点(最小变动价位倍数) |
| `is_favorable` | BooleanField | 是否有利滑点（成交价优于信号价） |

### 有利滑点判断逻辑

| 交易类型 | 持仓方向 | 有利条件 | 说明 |
|---------|---------|---------|------|
| ENTRY / ADD_ON | 多头 (1) | fill < signal | 买得比信号价低 |
| ENTRY / ADD_ON | 空头 (-1) | fill > signal | 卖得比信号价高 |
| EXIT / STOP_LOSS | 多头 (1) | fill > signal | 卖得比信号价高 |
| EXIT / STOP_LOSS | 空头 (-1) | fill < signal | 买得比信号价低 |

```typescript
    items.push({
        label: '平均滑点/有利滑点率',
```

**颜色规则**:
| 有利滑点率 | 颜色 |
|-----------|------|

| 40% ~ 60% | 黄色（中等） |
| < 40% | 红色（较差） |

### 2. 品种滑点统计图表（柱状图）

Dashboard 首页的品种滑点统计图表，位于品种胜率统计与品种盈亏排行之间。

**初始化函数**: `initSlippageBySymbolChart()`

**数据来源**: `getSlippageStats()` → `response.data.by_symbol`

**展示特点**:
- X 轴：品种代码（按平均滑点降序排列）
- Y 轴：平均滑点（跳）
- 柱色按有利滑点率染色：
  - 绿色（有利率 ≥ 60%）
  - 橙色（有利率 40% ~ 60%）
  - 红色（有利率 < 40%）
- MarkLine 标注基准线 1 跳
- Tooltip 显示品种详情（平均滑点、有利滑点率、交易次数）

**API 封装**: `web/src/views/system/home/api.ts:361`

## 数据流

```
调度任务执行交易
    │
    ▼
record_slippage()
    │  signal_price / fill_price / price_tick
    ▼
SlippageRecord.create()
    │
    ├──► SlippageRecordViewSet (记录明细查询)
    │
    └──► SlippageStatsView (统计汇总)
            │
            ▼
        Dashboard 首页
            ├── 指标卡片 (平均滑点/有利滑点率)
            └── 品种滑点统计图表
```

## 业务价值

1. **执行质量评估**：量化每笔交易的滑点成本，评估券商/交易所的成交质量
2. **品种对比**：识别哪些品种滑点较大（流动性差），辅助仓位分配决策
3. **交易类型分析**：区分开仓/平仓/止损的滑点特征，优化下单参数
4. **止损价格校准**：止损滑点过大时，调整止损价的缓冲余量
5. **有利滑点率监控**：有利滑点率持续低于 40% 时，需调查是否信号价计算有偏差
