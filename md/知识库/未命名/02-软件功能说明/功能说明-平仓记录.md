# 平仓记录功能说明

## 功能概述

平仓记录模块记录每一笔平仓交易的完整信息，包括合约、方向、成交量、成交价、成本价、盈亏金额、持仓天数等。是绩效统计（胜率、盈亏比）和品种分析的数据源。

**新增（2026-05-12）**：平仓记录现在同时记录开仓/平仓时的市场状态（趋势因子、趋势标签、ATR）以及持仓期间的最大浮盈/浮亏（MFE/MAE），用于后期策略绩效归因分析。

## 趋势分析字段

### 入场趋势快照（开仓时冻结，永不修改）

| 字段 | 类型 | 说明 |
|------|------|------|
| `entry_trend_factor` | Decimal | 开仓时的趋势因子（0.000 ~ 0.500），反映当时趋势强度 |
| `entry_trend_label` | String | 开仓时的趋势标签（strong_bull/weak_bull/choppy 等） |
| `entry_atr` | Decimal | 开仓时的 ATR(20) 波动率值 |

### 出场趋势快照（平仓时写入）

| 字段 | 类型 | 说明 |
|------|------|------|
| `exit_trend_factor` | Decimal | 平仓时的趋势因子值 |
| `exit_trend_label` | String | 平仓时的趋势标签 |
| `exit_atr` | Decimal | 平仓时的 ATR(20) 波动率值 |

### 极值分析

| 字段 | 类型 | 公式 | 说明 |
|------|------|------|------|
| `max_favorable_excursion` | Decimal | 多头=最高价-成本价；空头=成本价-最低价 | 最大有利偏移（MFE），即持仓期间最有利的价格距离 |
| `max_adverse_excursion` | Decimal | 多头=成本价-最低价；空头=最高价-成本价 | 最大不利偏移（MAE），即持仓期间最不利的价格距离 |

> 极值基于 **收盘价** 计算（非盘中极值），通过每日收盘后的 `update_all_positions_high_low_price()` 双向跟踪更新。

### 分析场景示例

| 分析方向 | SQL 思路 | 目的 |
|---------|---------|------|
| 入市趋势对胜率的影响 | `GROUP BY entry_trend_label` 对比各标签的胜率 | 验证震荡行情开仓是否值得 |
| 趋势延续性分析 | 对比 `entry_trend_label` vs `exit_trend_label` | 看趋势延续/反转对盈亏的影响 |
| 波动率归因 | 按 `entry_atr` 分桶统计 | 高波动率开仓是否更容易亏损 |
| MFE/MAE 分布 | 计算 `mfe / exit_atr` 归一化比率 | 优化止盈目标和止损距离 |

## 数据来源

平仓记录由以下场景自动创建：
1. **止损平仓**：`tasks_exit_before_close.py` 或 `tasks_daily_open.py`
2. **移仓换月**：`execute_rollover_order()` 平仓旧合约阶段

趋势分析字段的写入流程：
```
开仓 → execute_entry_order 将 signal.trend_factor/label + ATR 写入 PositionState
                                                       ↓
每日收盘 → update_all_positions_high_low_price() 双向更新 highest/lowest
                                                       ↓
平仓 → record_and_reset_position() / execute_rollover_order()
  ├─ 入场趋势：从 PositionState.entry_* 直接复制
  ├─ 出场趋势：从平仓信号 signal.trend_* 读取
  ├─ 出场 ATR：实时调用 calculate_atr() 计算
  └─ MFE/MAE：从 PositionState.highest_close/lowest_close 和 cost_price 计算
```

> 不支持手动添加或编辑，确保数据真实可信。

## 统计用途

平仓记录是以下指标的数据源：
- **品种胜率**：`GET /api/stock/symbol-win-rate/`
- **累计盈亏**：`GET /api/stock/cumulative-stats/`
- **三层绩效**：`core/performance.py` 中的胜率、盈亏比、连续盈亏等
- **趋势归因分析**：通过趋势分析字段，可按入市/出市市场状态分段统计
