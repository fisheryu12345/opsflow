# 多账户系统功能说明

## 功能概述

多账户系统实现了用户级别的数据隔离。每个系统用户注册后自动拥有独立的交易账户（TradingAccount），
用户可以自主选择交易的品种，系统按账户执行策略、计算绩效。不同用户之间完全隔离，互不可见。

---

## 数据隔离架构

```
User (系统用户)
  └── TradingAccount (交易账户，user FK)          ← 用户注册时自动创建
        ├── AccountContractConfig (品种选择)       ← 用户选择哪些品种
        ├── PositionState (持仓状态)               ← 逐账户持仓
        ├── DailyStrategySignal (交易信号)          ← 逐账户信号
        ├── ClosedPositionRecord (平仓记录)        ← 逐账户记录
        ├── DailyEquitySnapshot (日权益快照)        ← 逐账户绩效
        ├── RollingPerformanceMetrics (滚动指标)    ← 逐账户绩效
        ├── AccountPerformanceSummary (账户总览)    ← 逐账户绩效
        ├── TradeLog (交易日志)                    ← 逐账户日志
        └── ErrorLog (错误日志)                    ← 逐账户日志
```

### 核心隔离规则

| 角色 | 可见范围 |
|------|----------|
| 普通用户 | 仅可见 `user=自己` 的 TradingAccount 及其所有关联数据 |
| 超级管理员 | 不受限制，可查看所有账户数据 |

### 过滤机制

后端自动通过 `UserAccountFilterBackend` 过滤器实现，无需在每个 ViewSet 中手动编写过滤逻辑：

```
请求 → UserAccountFilterBackend
  → 当前用户是 superuser？ → 不过滤
  → 否 → 获取 user=当前用户的 TradingAccount ID 列表
       → queryset.filter(account_id__in=该列表)
```

## 用户注册流程

```
管理员创建用户
  → stock.signals.auto_create_trading_account 触发
  → TradingAccount 自动创建
    → name = 系统用户名（如 510976、510977）
    → initial_balance = 1,000,000
    → is_active = True
```

信号代码位于 `backend/stock/signals.py`，通过 `post_save` 监听 Users 模型。

## 交易品种选择流程

```
1. 用户登录 → 进入"交易品种管理"页面
2. 页面展示全市场合约列表（来自 `FullContractList`，全局合约主表，不与特定账户绑定）
3. 用户通过开关激活/停用品种 → POST /api/account-contracts/toggle/
4. 后台创建/更新 AccountContractConfig 记录
5. 收盘任务读取 AccountContractConfig → 确定各账户活跃品种
```

## 任务执行流程

### 收盘任务 (15:02)

```
job_daily_close_calculation()
  │
  ├─ 1. 创建 TqApi 连接
  ├─ 2. 检查交易日
  ├─ 3. 同步合约列表（全局 FullContractList）
  │
  ├─ 4. 确定活跃品种
  │     ↓
  │     AccountContractConfig.objects.filter(is_active=True)
  │     → 获取所有账户激活的 product_code
  │     → 计算对应合约的技术指标（一次，避免重复 API 调用）
  │
  └─ 5. 遍历每个活跃 TradingAccount
        ├─ 过滤该账户激活的品种
        ├─ 生成开仓信号
        ├─ 更新持仓跟踪价格
        ├─ 计算止损价
        ├─ 检查平仓信号
        ├─ 检查加仓信号
        ├─ 检查移仓信号
        ├─ 发送邮件报告
        └─ 更新三层绩效指标
```

### 开盘任务 (09:02 / 21:02)

```
job_daily_open_process()
  └─ 遍历每个活跃 TradingAccount（按账户处理信号执行）
       ├─ 处理 STOP_LOSS 信号
       ├─ 处理 ENTRY 信号
       ├─ 处理 ROLLOVER 信号
       ├─ 处理 ADD_ON 信号
       └─ 发送开盘报告
```

### 止损执行 (14:57)

`check_and_execute_stop_loss()` 已支持逐账户遍历，`_execute_stop_loss_for_account()` 按 `account` 过滤 PositionState。

### 合约同步

`contract_sync.py` 中的 `_ensure_position_states()` 函数为每个激活了该品种的账户创建 PositionState：

```python
def _ensure_position_states(symbol, product_code):
    configs = AccountContractConfig.objects.filter(
        product_code=product_code, is_active=True
    )
    for cfg in configs:
        PositionState.objects.get_or_create(
            account=cfg.account, symbol=symbol,
            defaults={'product_code': product_code}
        )
```

## 前端账户选择

### 登录初始化

用户登录成功后，`useAccountStore().fetchAccounts()` 自动调用 `/api/tradingaccount/` 获取当前用户的账户列表，默认选中第一个（或从 localStorage 恢复上次选择）。

### 账户 Store

`web/src/stores/account.ts` 提供：

| 属性/方法 | 作用 |
|-----------|------|
| `accounts` | 当前用户可访问的账户列表 |
| `currentAccountId` | 当前选中的账户 ID（持久化到 localStorage） |
| `fetchAccounts()` | 获取账户列表，初始化选中状态 |
### 数据联动

所有交易页面（持仓、信号、平仓、日志、绩效）通过 `watch(accountStore.currentAccountId, reloadData)` 实现账户切换时的数据自动刷新。

## API 变更汇总

### 新增接口

| Method | Path | 说明 |
|--------|------|------|
| GET | `/api/account-contracts/` | 获取当前账户的品种配置列表 |
| POST | `/api/account-contracts/toggle/` | 切换品种激活状态 |
| GET | `/api/account-contracts/available/` | 获取可选合约列表+已激活标记 |

### 受影响接口

所有 `/api/stock/*` 接口现在自动受 `UserAccountFilterBackend` 影响：
- 普通用户：仅返回 `account__user=当前用户` 的数据
- 超级管理员：不受影响，返回全部数据

---

## 数据模型变更

### 新增模型：AccountContractConfig

| 字段 | 类型 | 说明 |
|------|------|------|
| `account` | FK → TradingAccount | 所属账户 |
| `product_code` | CharField(10) | 品种代码（rb, MA, IF） |
| `is_active` | BooleanField | 是否启用交易 |
| `allow_open` | BooleanField | 是否允许开仓 |

唯一约束：`(account, product_code)`

### TradingAccount 新增字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `user` | FK → system.Users | 所属系统用户，用于数据隔离 |

FullContractList 保持为全局主合约列表（无 account 字段，与用户无关），不做多用户复制。

---

## 限制说明

| 限制项 | 说明 |
|--------|------|
| TqSDK 账户共享 | 所有 TradingAccount 共用同一个 TqSDK 账号（通过 StrategyConfig.tqapi_account 配置）。目前不支持每个用户独立 TqSDK 账号。 |
| 合约同步共享 | 合约数据来自一次全局 TqSDK 同步，不支持按用户独立定制合约列表来源。 |
| 信号执行顺序 | 多账户信号按 TradingAccount 顺序依次执行，非并发。 |
