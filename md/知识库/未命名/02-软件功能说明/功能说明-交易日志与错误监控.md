# 交易日志与错误监控功能说明

## 功能概述

交易日志和错误日志模块用于记录策略运行过程中的关键操作和异常信息，支持按时间、函数名、日志级别、合约代码等维度检索，是系统运维和策略排查的核心工具。

---

## 模块划分

| 模块 | 用途 | 数据来源 |
|------|------|---------|
| **交易日志 (TradeLog)** | 记录正常业务流程的关键节点 | 主动记录（决策过程、计算结果、API调用） |
| **错误日志 (ErrorLog)** | 记录异常和错误 | 被动捕获（try/except 异常堆栈） |

---

## 交易日志

### 字段说明

| 字段 | 说明 | 示例 |
|------|------|------|
| `timestamp` | 日志时间 | 2026-05-08 15:02:33 |
| `function_name` | 函数完整路径 | `stock.scheduler.tasks_daily_open.execute_entry_order` |
| `log_level` | 日志级别 | INFO / SUCCESS / WARNING |
| `symbol` | 关联合约代码 | rb2610 |
| `log_message` | 日志内容 | "rb2610 开仓成功: 1 Unit(4手) @ 3200.00" |

### 日志级别定义

| 级别 | 使用场景 |
|------|---------|
| `INFO` | 一般流程记录（如开始执行、参数值） |
| `SUCCESS` | 操作成功（如开仓成功、止损平仓成功） |
| `WARNING` | 需要关注但不致命（如跳空保护跳过、两步开仓） |
| `ERROR` | 错误（ TradeLog 中较少使用，通常写入 ErrorLog） |

### 典型日志内容

```
[INFO]  开始执行收盘计算任务
[INFO]  rb2610 趋势因子: 0.357, 标签: weak_bull
[SUCCESS] rb2610 开仓成功: 1 Unit(4手) @ 3200.00
[WARNING] m2609 跳空幅度过大，禁止开仓
[INFO]  今日共生成 3 个开仓信号, 2 个止损信号
```

## 错误日志

### 字段说明

| 字段 | 说明 | 示例 |
|------|------|------|
| `timestamp` | 错误发生时间 | 2026-05-08 15:02:45 |
| `function_name` | 出错函数完整路径 | `stock.scheduler.tasks_daily_close.update_all_positions_stop_loss_price` |
| `error_message` | 错误详情和堆栈 | Traceback + 上下文信息 |

### 典型错误场景

| 错误 | 常见原因 | 处理建议 |
|------|---------|---------|
| 指标计算失败 | TqApi 数据不足、网络异常 | 检查 K 线数据长度，重试或跳过 |
| 止损更新失败 | highest_close 为 None | 初始化持仓跟踪价格 |
| 开仓失败 | 不在交易时间、TargetPosTask 超时 | 检查交易状态，手动补单 |
| 平仓失败 | 持仓已为空、API 连接中断 | 检查实盘持仓，手动平仓 |

## 页面功能

### 交易日志页面

- **时间范围筛选**：支持日期范围选择
- **函数名称搜索**：模糊匹配函数路径
- **日志级别筛选**：INFO / SUCCESS / WARNING
- **合约代码筛选**：按 symbol 过滤
- **内容搜索**：按日志内容关键词搜索

### 错误日志页面

- **时间范围筛选**：支持日期范围选择
- **函数名称搜索**：定位出错模块
- **错误内容搜索**：按错误信息关键词搜索
- **支持删除**：允许清理已处理的旧错误日志
