# 策略配置功能说明

## 功能概述

策略配置模块用于管理交易系统的全部可调参数，实现**"配置与代码分离"**。所有参数原在 `parameter_config.py` 中硬编码，现已迁移到数据库 `StrategyConfig` 表，并通过 `core/config_loader.py` 在启动时加载。

**核心变化**：`parameter_config.py` 已删除，所有代码统一通过 `get_config()` 从数据库读取参数。创建/修改配置后**无需重启服务**（当前进程级缓存，重启后或调用 `clear_cache()` 即可生效）。

## 字段一览

### 资金管理参数

| 字段 | 默认值 | 说明 | 对应代码常量 |
|------|--------|------|-------------|
| `max_units` | 3 | 单个品种最多持有 N 个 Unit | `POSITION_MAX_UNITS` |
| `entry_units` | 1 | 首次开仓 Unit 数 | `POSITION_MIN_UNITS` |
| `risk_per_unit` | 4000 | 每 Unit 风险资金基数（元） | `POSITION_RISK_BASE_AMOUNT` |
| `position_risk_multiplier` | 2 | ATR 风险倍数（止损距离 = N × ATR） | `POSITION_RISK_MULTIPLIER` |
| `protect_cost_enabled_ratio` | 2.5 | 盈利超过 N×ATR 时启用保本价保护 | `PROTECT_COST_ENABLED_RATIO` |
| `timeout_seconds` | 60 | TargetPosTask 订单等待超时（秒） | `TIMEOUT_SECONDS` |

**计算公式**：`unit_lots = risk_per_unit / (ATR × position_risk_multiplier × 合约乘数)`

### 技术指标参数

| 字段 | 默认值 | 说明 |
|------|--------|------|
| `atr_period` | 20 | ATR 计算周期。调小→对近期波动更敏感 |
| `entry_period` | 20 | 唐奇安通道突破周期。调小→信号更频繁 |
| `ma_periods` | "10,20,40" | 均线周期组合（逗号分隔，用于趋势因子） |

### 趋势因子参数

| 字段 | 默认值 | 说明 | 调整方向 |
|------|--------|------|---------|
| `trend_gap_limit` | 0.03 (3%) | 均线间距封顶上限。达此值时 factor 封顶 | 调小→因子更快达最大值 |
| `trend_factor_max` | 0.5 | factor 最大值。止损倍数范围=[2.0, 2×(1+max)] | 调大→强趋势时止损更宽 |
| `trend_label_strong_ratio` | 0.80 | trend_strength≥此值判定为 strong | 调小→更容易判为强趋势 |
| `trend_label_weak_ratio` | 0.30 | trend_strength≥此值判定为 weak | 调小→更容易判为有趋势 |

### 过滤参数

| 字段 | 默认值 | 说明 |
|------|--------|------|
| `gap_threshold` | 1.5 | 跳空幅度超过 N×ATR 时跳过开仓。调小→保护更敏感 |
| `product_codes` | 22 个品种 | 逗号分隔的交易品种列表。仅这些品种会同步合约和生成信号 |

### TqSDK 账户配置

| 字段 | 默认值 | 说明 |
|------|--------|------|
| `tqapi_account` | yupei1986 | TqSDK 登录账号 |
| `tqapi_password` | (密码) | TqSDK 登录密码（前端显示为 ••••••••） |

## 参数配置传递链路

```
用户在前端修改参数 → 保存到 DB (StrategyConfig 表)
                                         ↓
进程启动 / 重启 → config_loader._load_config() 读取 DB
                                         ↓
                    LRU 缓存（进程生命周期内有效）
                                         ↓
            各模块通过 get_config('KEY') 获取参数值
```

### 配置加载优先级

1. **数据库** `StrategyConfig` 表的第一条记录
2. 如果数据库无记录 → 回退到 `config_loader.py` 中的 `DEFAULTS` 硬编码默认值

### 热更新说明

当前实现使用 `functools.lru_cache` 缓存配置，进程生命周期内只加载一次。如需热更新：
- 调用 `clear_cache()` 清除缓存（计划任务或 API 触发）
- 或重启 Django 服务

## 与旧系统的对比

| 对比项 | 旧系统 (parameter_config.py) | 新系统 (DB + config_loader) |
|--------|------------------------------|------------------------------|
| 参数存储 | Python 文件硬编码 | 数据库 StrategyConfig 表 |
| 修改方式 | 编辑文件 + 重启服务 | 前端界面修改 + 下次加载生效 |
| 多套配置 | 不支持 | 支持多套命名配置方案 |
| 代码引用 | `from stock.parameter_config import X` | `from stock.core.config_loader import get_config` |
| 文件状态 | 已删除 | `core/config_loader.py` |
