# 策略运行说明 — 实盘操作手册（2026-05-09）

> 本文档面向实际操盘和系统运维人员，说明 FT 量化期货交易系统的日常运行流程、监控要点和异常处理。

---

## 一、每日运行时间表

```
时间          任务                          执行文件                        关键动作
─────         ────                          ──────────                      ────────
09:02         开盘任务（日盘）               tasks_daily_open.py             止损/开仓/移仓/加仓
14:57         临收盘止损                     tasks_exit_before_close.py      紧急市价平仓
15:02         收盘计算                       tasks_daily_close.py            指标/信号/绩效
21:02         开盘任务（夜盘，如有）          tasks_daily_open.py             同上
```

> 夜盘品种（如原油、黄金等）在 21:02 执行开盘任务。非夜盘品种无影响。
> 所有任务仅在 **周一至周五** 运行，非交易日自动跳过。

## 二、开盘任务详细流程（09:02 / 21:02）

### 2.1 任务启动前检查

系统启动条件：
1. **IP 白名单**：只有服务器 IP 为 `172.25.21.216` 时才会启动 APScheduler
2. **Redis 分布式锁**：`lock:open`（600秒过期），防止多实例重复执行
3. **交易日检查**：通过 TqSDK 交易日历 API 判断当日是否为交易日

### 2.2 信号执行优先级

```
第1步：STOP_LOSS（止损平仓）
        ├── 查询所有 PENDING 的 STOP_LOSS 信号
        ├── 遍历持仓记录，找到对应信号
        ├── TargetPosTask 目标持仓设为 0
        └── 更新 PositionState / ClosedPositionRecord / 信号状态

第2步：ENTRY（趋势开仓）
        ├── 查询所有 PENDING 的 ENTRY 信号
        ├── 检查交易时间、跳空保护、最小开仓限制
        ├── TargetPosTask 设置目标持仓（±order_volume）
        └── 创建/更新 PositionState / 更新信号状态

第3步：ROLLOVER（移仓换月）
        ├── 查询所有 PENDING 的 ROLLOVER 信号
        ├── 平旧合约 → 开新合约（同方向同手数）
        └── 删除旧 PositionState，创建新 PositionState

第4步：ADD_ON（金字塔加仓）
        ├── 查询所有 PENDING 的 ADD_ON 信号
        ├── 检查是否超限（projected_units > 3）
        ├── TargetPosTask 增加目标持仓
        └── 更新 PositionState（units/last_add_price）
```

### 2.3 执行后报告

任务完成后自动发送邮件（`send_open_report`）：
- 过去 24 小时内更新的信号记录
- 统计：成功 / 失败 / 待处理 / 取消 数量
- 每个信号的详细状态

## 三、收盘任务详细流程（15:02）

### 3.1 Step 1-3：准备工作

```
Step 1: 连接 TqApi（快期模拟盘 TqKq）
Step 2: 检查交易日历（非交易日直接返回）
Step 3: 同步合约列表
        └── 从 TqSdk 拉取全市场主力合约
        └── 若主力合约变更 → 标记 is_rollover_needed=True
```

### 3.2 Step 4：技术指标计算

对所有 `is_active=True` 且有持仓的品种计算：
- ATR(20) / MA(10/20/40)
- 唐奇安通道(20) —— 前20日高低点
- 趋势因子 & 趋势标签
- 突破信号

结果存入 `PositionState.indicators` JSON 字段。

### 3.3 Step 5：开仓信号生成

条件：
1. 收盘价突破唐奇安上轨/下轨（不含今日）
2. 该品种当前 `units == 0`
3. 不存在 PENDING 的 ENTRY 信号
4. 合约 `is_active=True`

满足则创建 `DailyStrategySignal(trade_type='ENTRY')`。

### 3.4 Step 6-7：止损更新

```
Step 6: 更新持仓最高/最低收盘价
        多头：highest_close = max(highest_close, latest_close)
        空头：lowest_close = min(lowest_close, latest_close)

Step 7: 更新止损价格
        动态止损 = 基准价 ± 2(1+factor) × ATR
        保本检查：盈利 > 2.5×ATR ?
        保本价 = 成本价 ± tick × 2
        最终止损 = max/min(动态止损, 保本价)
```

### 3.5 Step 8-10：其他信号生成

```
Step 8: 止损信号 — 收盘价击穿止损价 → STOP_LOSS
Step 9: 加仓信号 — 检查金字塔加仓条件 → ADD_ON
Step 10: 移仓信号 — is_rollover_needed=True → ROLLOVER
```

### 3.6 Step 11-12：报告与绩效

```
Step 11: 发送每日策略信号报告邮件（HTML格式）
         包含：今日所有信号、趋势因子、唐奇安通道、备注

Step 12: 更新三层绩效指标
         Layer 1: DailyEquitySnapshot（日权益快照）
         Layer 2: RollingPerformanceMetrics（20/60/120/250日窗口）
         Layer 3: AccountPerformanceSummary（账户总览）
```

## 四、临收盘止损（14:57）

### 4.1 触发场景

- 收盘前最后几分钟价格剧烈波动
- 某品种收盘价已击穿止损价
- 防止隔夜跳空导致次日无法执行止损

### 4.2 执行流程

```
1. 直接查询数据库：latest_close_price vs stop_loss_price
2. 多头：latest_close < stop_loss → 触发
3. 空头：latest_close > stop_loss → 触发
4. 立即市价平仓（TargetPosTask，目标持仓=0）
5. 创建 ClosedPositionRecord
6. 清空 PositionState
7. 更新信号状态
```

> 与开盘止损的区别：14:57 是**被动发现即平**，开盘任务是**按信号队列顺序执行**。

## 五、日常监控清单

### 5.1 每日必查（开盘后）

| 检查项 | 位置 | 正常状态 | 异常处理 |
|--------|------|---------|---------|
| 信号执行报告邮件 | 邮箱 | 收到成功/失败统计 | 未收到 → 检查邮件服务 |
| 持仓状态 | 前端-持仓管理 | units/direction 与实盘一致 | 不一致 → 检查 TqApi 同步 |
| 止损价更新 | 前端-持仓管理 | 每个持仓有有效止损价 | 为空 → 检查 indicators 是否计算成功 |
| 信号状态 | 前端-信号监控 | PENDING 信号在次日被处理 | 长期 PENDING → 检查开盘任务 |

### 5.2 每周必查

| 检查项 | 位置 | 正常状态 | 异常处理 |
|--------|------|---------|---------|
| 绩效指标 | Dashboard | 夏普/胜率/回撤合理 | 异常波动 → 检查策略参数 |
| 合约列表 | 前端-合约管理 | 主力合约最新 | 过期 → 手动执行 sync_contracts |
| 错误日志 | 前端-错误日志 | 无新增 ERROR | 有 ERROR → 按函数名定位修复 |
| 交易日志 | 前端-交易日志 | 记录完整无缺失 | 缺失 → 检查日志写入权限 |

### 5.3 每月必查

| 检查项 | 位置 | 正常状态 | 异常处理 |
|--------|------|---------|---------|
| 移仓换月 | 前端-持仓管理 | 无 is_rollover_needed=True | 有 → 检查合约同步或手动移仓 |
| 策略参数 | 前端-策略配置 | 参数符合预期 | 需调整 → 修改后无需重启 |
| 数据库容量 | 服务器 | 日权益快照/日志未爆满 | 过大 → 归档历史数据 |

## 六、常见异常及处理

### 6.1 开盘任务未执行

**现象**：09:02 后未收到邮件，信号状态未更新。

**排查**：
1. 检查服务器 IP 是否为 `172.25.21.216`
2. 检查 Redis：`lock:open` 是否被占用（600秒自动过期）
3. 检查是否为交易日：`is_trade_day()`
4. 查看后端日志：`stock/logs/` 或 `TradeLog` 表

**处理**：
- 若 Redis 锁未释放（异常中断）：手动删除 `lock:open`
- 若任务漏执行：等待下一个执行时点或手动触发

### 6.2 止损未触发

**现象**：价格已跌破止损线，但持仓未平。

**排查**：
1. 检查 `PositionState.stop_loss_price` 是否有值
2. 检查 `PositionState.indicators` 是否为空（导致止损未更新）
3. 检查是否存在 PENDING 的 STOP_LOSS 信号（防重复）
4. 检查 14:57 任务日志

**处理**：
- indicators 为空 → 手动调用 `calculate_indicators` 或检查 TqApi 数据
- 止损价异常 → 手动更新止损价或重启收盘任务
- 紧急情况下 → 通过 TqSdk 客户端手动平仓

### 6.3 开仓失败

**现象**：ENTRY 信号状态为 FAILED 或 CANCELLED。

**排查**：
1. 检查 remark 字段：
   - "跳空保护" → 跳空幅度 > 1.5×ATR
   - "开仓超时" → TargetPosTask 60秒未成交
   - "不在交易时间" → 合约已收盘或停牌
2. 检查最小开仓限制：是否触发两步开仓

**处理**：
- 跳空保护 → 次日视情况手动开仓或等待回调
- 超时 → 检查网络/TqApi 连接，可手动重试

### 6.4 移仓失败

**现象**：ROLLOVER 信号 FAILED，旧合约已平但新合约未开。

**排查**：
1. 检查新合约是否在 `FullContractList` 中
2. 检查新合约 `is_active=True`
3. 检查最小开仓限制

**处理**：
- 手动执行两步开仓或正常开仓
- 手动创建 PositionState 记录

### 6.5 绩效数据异常

**现象**：Dashboard 显示收益率/回撤与预期不符。

**排查**：
1. 检查 `DailyEquitySnapshot` 是否有当日记录
2. 检查 `AccountPerformanceSummary` 是否更新
3. 检查 `initial_balance` 是否正确

**处理**：
- 当日快照缺失 → 手动调用 `update_all_performance_metrics`
- 初始资金错误 → 修改 `TradingAccount.initial_balance`

## 七、手动操作指南

### 7.1 手动同步合约

```bash
cd backend
python manage.py sync_contracts
```

> 用于主力合约切换后手动更新合约列表。

### 7.2 手动触发收盘计算

```python
# Django shell
python manage.py shell
from stock.scheduler.tasks_daily_close import job_daily_close_calculation
job_daily_close_calculation()
```

### 7.3 手动更新绩效

```python
from stock.core.performance import update_all_performance_metrics
from stock.models import TradingAccount
import datetime

account = TradingAccount.objects.get(id=1)
api_account_data = {
    'balance': 1000000,
    'available': 800000,
    'float_profit': 5000,
    'margin': 200000,
    'risk_ratio': 0.2,
    'commission': 100,
    'close_profit': 3000,
    'pre_balance': 995000,
}
update_all_performance_metrics(account, api_account_data, datetime.date.today())
```

### 7.4 紧急平仓

若系统故障需人工干预：
1. 登录 TqSdk 客户端（快期模拟盘）
2. 找到对应合约持仓
3. 市价平仓
4. 手动更新数据库：
   - 创建 `ClosedPositionRecord`
   - 清空 `PositionState`
   - 更新 `DailyStrategySignal` 状态
