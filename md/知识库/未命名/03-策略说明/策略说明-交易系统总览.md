## 二、核心参数配置

| 参数 | 值 | 含义 |
|------|------|------|
| `POSITION_RISK_BASE_AMOUNT` | ¥4,000 | 每 Unit 承担的固定风险资金基数 |
| `POSITION_RISK_MULTIPLIER` | 2 | ATR 风险倍数（止损距离 = 2 × ATR） |
| `POSITION_MAX_UNITS` | 3 | 单个品种最多持仓 3 Unit |
| `PROTECT_COST_ENABLED_RATIO` | 2.5 | 盈利超过 2.5×ATR 启用保本 |
| `GAP_PROTECTION_RATIO` | 1.5 | 跳空幅度 > 1.5×ATR 时跳过开仓 |
| `TIMEOUT_SECONDS` | 60 | TargetPosTask 订单等待超时 |
| `TREND_GAP_LIMIT` | 0.03 (3%) | 趋势因子封顶上限（均线间距达 3% 时 factor 封顶） |
| `TREND_FACTOR_MAX` | 0.5 | 趋势因子最大值 |
| `TREND_LABEL_STRONG_RATIO` | 0.80 | 强趋势 label 阈值（trend_strength ≥ 80%） |
| `TREND_LABEL_WEAK_RATIO` | 0.30 | 弱趋势 label 阈值（trend_strength ≥ 30%） |
| **`skip_choppy_entry`** | **false** | **策略配置项：True=跳过震荡行情开仓，用于 A/B 对比验证** |

## 三、每日交易时间线

### 09:02 / 21:02 — 开盘任务 (`tasks_daily_open.py`)

1. 创建临时 TqApi 检查交易日历，非交易日直接退出
2. 遍历每个活跃 TradingAccount（逐账户处理信号执行）
3. 获取 Redis 分布式锁 (`lock:open:{account.id}`，600 秒过期，防重复执行)
4. **按优先级处理信号队列**：
   ```
   ├─ STOP_LOSS  (止损平仓) — 最高优先级
   ├─ ENTRY      (趋势开仓)
   ├─ ROLLOVER   (移仓换月)
   └─ ADD_ON     (金字塔加仓) — 最低优先级
   ```
5. 发送开盘执行报告邮件
6. 释放锁 + 断开 API

### 14:57 — 临收盘止损 (`tasks_exit_before_close.py`)

1. 直接查询数据库：最新价是否已击穿止损价
2. 对击穿品种立即市价平仓 (TargetPosTask)
3. 创建 ClosedPositionRecord + 清空 PositionState
4. 更新信号状态 SUCCESS / FAILED

> **为什么要 14:57 单独跑？** 防止收盘前最后几分钟价格剧烈波动导致信号隔夜跳空时无法执行。

### 15:02 — 收盘计算 (`tasks_daily_close.py`)

```
Step 1:  连接 TqApi
Step 2:  检查交易日历
Step 3:  同步合约列表 (sync_contract_list_from_tqsdk)
         - 从 TqSdk 拉取全市场主力合约
         - 对比数据库，若主力合约变更 → 标记 is_rollover_needed=True
Step 4:  技术指标计算 (calculate_indicators)
         - 拉取 60 日日线 → 计算 ATR(20) / MA(10/20/40) / 唐奇安通道(20)
         - 趋势分类 → 计算 trend_factor & trend_label
Step 5:  开仓信号生成 (ENTRY)
         - 收盘价突破唐奇安上轨/下轨 + 无持仓 + 无待执行 ENTRY → 生成信号
Step 6:  更新持仓最高/最低价 (update_all_positions_high_low_price)
Step 7:  更新止损价 (update_all_positions_stop_loss_price)
         - 动态跟踪止损 + 保本兜底
Step 8:  止损信号生成 (STOP_LOSS)
         - 收盘价击穿止损价 → 生成信号
Step 9:  加仓信号生成 (ADD_ON)
         - 检查金字塔加仓条件
Step 10: 移仓信号生成 (ROLLOVER)
         - is_rollover_needed=True 的持仓
Step 11: 发送每日信号报告邮件
Step 12: 三层绩效指标更新
```

## 四、策略核心：海龟法则变体

### 4.1 趋势过滤与趋势因子

基于 **MA10 / MA20 / MA40 排列 + 均线间距** 动态阈值：

| 均线排列 | 间距条件 | 趋势标签 | trend_factor |
|---------|---------|---------|-------------|
| MA10>MA20>MA40 | trend_strength ≥ 80% | strong_bull | 0.0 ~ 0.5 |
| MA10>MA20>MA40 | 30% ≤ trend_strength < 80% | weak_bull | 0.0 ~ 0.5 |
| MA10>MA20>MA40 | trend_strength < 30% | choppy | 0.0 |
| MA10<MA20<MA40 | trend_strength ≥ 80% | strong_bear | 0.0 ~ 0.5 |
| MA10<MA20<MA40 | 30% ≤ trend_strength < 80% | weak_bear | 0.0 ~ 0.5 |
| 交叉/混乱 | — | choppy | 0.0 |

**计算公式**：
```
gap_10_20 = |MA10 - MA20| / |MA20|
gap_20_40 = |MA20 - MA40| / |MA20|
max_gap   = max(gap_10_20, gap_20_40)
trend_strength = min(max_gap / TREND_GAP_LIMIT, 1.0)   # TREND_GAP_LIMIT = 0.03
trend_factor   = round(trend_strength * TREND_FACTOR_MAX, 3)  # TREND_FACTOR_MAX = 0.5
```

**止损倍数**：`2 × (1 + factor)`，范围 **[2.0, 3.0] ATR**

### 4.2 开仓信号 (ENTRY)

- **触发条件**：收盘价突破前 20 日高点/低点（**不包含今日**）
- **方向**：上破 → 做多 (direction=1)；下破 → 做空 (direction=-1)
- **过滤**：
  - 已有持仓 (units>0) → 跳过
  - 存在待执行 ENTRY 信号 (executed_status='PENDING') → 跳过
  - 跳空幅度 > 1.5×ATR → 跳过（跳空保护）
  - **配置项 `skip_choppy_entry=True` 且趋势标签为 `choppy`/`neutral` → 跳过（震荡行情不入场）**
- **数量**：首次开仓固定 1 Unit

### 4.3 仓位计算 (Unit Lots)

```
unit_lots = POSITION_RISK_BASE_AMOUNT / (ATR × POSITION_RISK_MULTIPLIER × 合约乘数)
         = 4000 / (ATR × 2 × volume_multiple)
```

**示例**：螺纹钢 ATR=50，每手 10 吨
```
unit_lots = 4000 / (50 × 2 × 10) = 4 手/Unit
最大持仓 = 3 Unit × 4 手 = 12 手
最大风险 = 3 × ¥4,000 = ¥12,000
```

### 4.4 金字塔加仓 (ADD_ON)

**前提**：只检查 units ∈ {1, 2} 的持仓，units = 3 时不再加仓。

#### 场景 A：当前 1 Unit，基准价 = first_open_price

| 方向 | 触发条件 | 加仓 Unit 数 | 加仓后总 Unit |
|------|---------|------------|-------------|
| 多头 | 涨超 1.0×ATR | +2 (直接满仓) | 3 |
| 多头 | 涨超 0.5×ATR | +1 | 2 |
| 空头 | 跌超 1.0×ATR | +2 (直接满仓) | 3 |
| 空头 | 跌超 0.5×ATR | +1 | 2 |

> `if ... elif` 结构保证当价差 > 1.0×ATR 时走 +2 分支，不会进入 +1 分支。

#### 场景 B：当前 2 Unit，基准价 = first_open_price

| 方向 | 触发条件 | 加仓 Unit 数 | 加仓后总 Unit |
|------|---------|------------|-------------|
| 多头 | 从 first_open_price 累计涨超 1.0×ATR | +1 | 3 |
| 空头 | 从 first_open_price 累计跌超 1.0×ATR | +1 | 3 |

#### 安全截断
```python
if units_current + add_units > 3:
    add_units = 3 - units_current
```

#### 防重复
同一品种存在未执行的 ADD_ON 信号 (PENDING) 时，跳过重复生成。

### 4.5 止损体系（双层）

#### 第一层：动态跟踪止损

| 方向 | 公式 | 特性 |
|------|------|------|
| 多头 | `highest_close - 2×(1+factor)×ATR` | 只升不降 |
| 空头 | `lowest_close + 2×(1+factor)×ATR` | 只降不升 |

#### 第二层：保本兜底

- **保本触发**：盈利 > `PROTECT_COST_ENABLED_RATIO × ATR` (= 2.5×ATR)
- **保本价**：`成本价 ± tick × 2`（根据品种最小变动价位动态计算）
- **单向开关**：`protect_cost_enalbed` 一旦开启永久保持，不会关闭

**最终止损**：
```
多头：max(动态止损, 保本价)
空头：min(动态止损, 保本价)
```

**通俗解释**：动态止损是主角，保本只是安全垫。价格一路向有利方向走 → 动态止损不断上移，保本价不会拖后腿；只有极端情况（波动突然加大）导致动态止损回落到成本价以下 → 保本价兜底。

#### 示例（多头）

| 收盘价 | highest_close | 动态止损 | 盈利 | 保本状态 | 最终止损 |
|--------|--------------|---------|------|---------|---------|
| 100 | 100 | 94 | 0 | 未触发 | 94 |
| 106 | 106 | 100 | 6 | 未触发 (6 < 7.5) | 100 |
| 108 | 108 | 102 | 8 | ✅ 触发 (8 > 7.5) | 102 (动态 > 保本) |
| 120 | 120 | 114 | 20 | 已启用 | 114 (继续跟踪) |
| ATR 跳至 12 | 120 | 96 | 20 | 已启用 | 101 (保本兜底) |

### 4.6 移仓换月 (ROLLOVER)

- **触发**：`sync_contract_list_from_tqsdk()` 发现主力合约变更
- **操作**：平旧合约 + 开新合约（同方向同手数）
- **时机**：在开盘任务中按 ENTRY 优先级之后执行
- **数据库处理**：旧合约 PositionState 删除，新合约创建（基于 20 日历史数据初始化 highest/lowest/止损价）

## 六、执行引擎细节

### 6.1 TargetPosTask 机制

```python
target_pos = TargetPosTask(api, symbol, target_position=4)
# TqSdk 自动处理: 报单 → 追单 → 撤单重报 → 直到持仓=target
wait_for_target_position(api, target_pos, symbol, 4, timeout=60)
target_pos.cancel()  # 释放资源
```

### 6.2 信号状态流转

```
PENDING → EXECUTING → SUCCESS
   ↓         ↓           ↓
         超时/失败 → FAILED / CANCELLED
```

### 6.3 两步开仓策略

当计划开仓手数 < 交易所最小开仓手数 (min_position) 时：
1. 第一步：开立等于 `min_position` 的仓位
2. 第二步：立即平仓多余部分 `min_position - planned_volume`
3. 结果：最终持仓等于原计划手数，同时符合交易所规则

### 6.4 Redis 分布式锁

| 锁名 | 用途 | 超时 |
|------|------|------|
| `lock:open:{account.id}` | 开盘任务防重（逐账户） | 600 秒 |
| `lock:scheduler` | 调度器启动防重 | — |

## 九、关键设计特点

1. **信号驱动，非实时**：收盘生成信号，开盘批量执行，避免盘中情绪干扰
2. **单向保本开关**：`protect_cost_enalbed` 一旦开启永不关闭，防止来回切换
3. **防跳空开仓**：避免隔夜跳空导致的追高/追低
4. **两步开仓**：应对交易所最小开仓手数限制
5. **趋势自适应止损**：强趋势时放宽止损 (factor=0.5 → 3.0 ATR)，震荡市收紧 (factor=0 → 2.0 ATR)
6. **连续趋势因子**：均线间距线性映射到 [0, 0.5]，止损倍数连续变化 [2.0, 3.0]，无跳变
7. **label 与 factor 同源**：都用 `trend_strength` 判定，调整 `TREND_GAP_LIMIT` 时 label 分级自动缩放
