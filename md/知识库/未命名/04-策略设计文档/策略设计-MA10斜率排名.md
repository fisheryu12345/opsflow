# 策略设计 — MA10 斜率排名（对比系统）

> 核心思想：每日计算全市场品种的 MA10 斜率，取上升趋势最强的前 5 名做多、下降趋势最强的前 5 名做空。不加仓、不移仓。

---

## 一、策略概述

### 1.1 定位

MA10 斜率排名是一个**独立运行的对比策略**，与海龟交易系统（`run_turtle.py`）并列运行在 510988 账户下。它不修改任何实盘代码，所有参数从 Python 文件硬编码加载，数据写入现有数据库表。

| 方面   | 说明                         |
| ---- | -------------------------- |
| 运行方式 | 独立管理命令 `run_ma10_slope.py` |
| 账户   | 510988（与海龟对比组同一账户）         |
| 策略类型 | 横截面排名 — 全市场选最强/最弱，非单品种突破   |
| 仓位   | 不加仓，每品种固定 1 单位             |
| 移仓   | 不移仓，持仓合约到期前主动离场            |

### 1.2 与 Donchian 系统的关键差异

| 方面       | Donchian 系统            | MA10 斜率排名（本策略）           |
| -------- | ---------------------- | ------------------------ |
| **入场依据** | 价格突破 20 日唐奇安通道         | MA10 斜率在全市场排名前/后 5       |
| **趋势衡量** | 二元判断（突破/不突破）           | 连续值（slope_pct），可跨品种比较    |
| **品种选择** | 所有已配置品种各自独立判断          | **全市场排名择优**，只做最强/最弱      |
| **加仓**   | 0.5/1.0 ATR 加仓，最多 3 单位 | **不加仓**，每品种 1 单位固定       |
| **移仓**   | 主力合约切换时自动移仓            | **不移仓**（持仓合约到期前主动离场）     |
| **出场逻辑** | ATR 移动止损               | **排名退化 + 趋势反转 + 硬止损 三层** |
| **风险分散** | 资金按品种数量均分              | **多空对冲**，5 多 + 5 空天然平衡   |
| **运行方式** | 收盘信号→次日开盘执行            | **同左**                   |

## 二、MA10 斜率计算

### 2.1 公式

```
slope_pct = (MA10[t] - MA10[t - slope_period]) / MA10[t - slope_period] * 100
```

- `MA10[t]` = 当日收盘后的 10 日移动平均线
- `slope_period` = 3（默认），即对比 MA10 在 3 个交易日内的变化率
- `slope_pct` 以百分比表示，正数 = 上升趋势，负数 = 下降趋势

### 2.2 为什么用百分比？

不同品种价格水平差异极大（铜 ≈ 70000 元/吨，玉米 ≈ 2400 元/吨），直接用 MA10 差值不可比。百分比将 slope 统一为可跨品种比较的标量。

### 2.3 为什么用 MA10 斜率而不是 MA5 或 MA20？

- **MA5 斜率**太敏感，噪音多，频繁换仓
- **MA20 斜率**太滞后，趋势启动后很久才反应
- **MA10 斜率**是平衡点：MA10 本身已平滑了 10 天的价格噪音，3 日变化率又能及时发现趋势变化

### 2.4 为什么 slope_period=3？

MA10 已经是 10 日均值，再取 5 日以上的变化率就过于平滑，对趋势变化反应太慢。3 日（约半个交易周）能较好平衡灵敏度和稳定性。

## 三、每日排名流程

### 3.1 处理步骤

```
15:00 市场收盘
  │
  ├─ Step 1: 遍历所有活跃主力合约（FullContractList）
  ├─ Step 2: 查询 KlineData，获取最近 13+ 天的 ma_10 数据
  │          （slope_period=3 + MA10 需要 10 天预热 + 3 天缓冲）
  ├─ Step 3: 计算每个品种的 ma_10_slope_pct
  ├─ Step 4: 排除数据不足 / 非主力 / 不活跃品种
  ├─ Step 5: 按 slope_pct 从大到小排序
  ├─ Step 6: 取前 5 名 = 多头候选，后 5 名 = 空头候选
  │
  └─ 生成/更新信号
```

## 四、入场规则

### 4.1 多头入场（全部条件必须同时满足）

| #   | 条件                   | 阈值/判断                       | 设计理由                 |
| --- | -------------------- | --------------------------- | -------------------- |
| 1   | 品种排名在前 5             | `ranking_top_n=5`           | 只做最强上升趋势，淘汰跟风品种      |
| 2   | MA10 斜率 > 最低阈值       | `min_slope_threshold=0.05`  | 斜率基本为零时 MA10 走平，不应入场 |
| 3   | 当前无该品种的多头持仓          | `position.direction != 1`   | 同一品种不重复开仓            |
| 4   | 当前总多头数 < 上限          | `max_positions_per_side=5`  | 控制总风险敞口              |
| 5   | 当日无 PENDING ENTRY 信号 | `_already_signaled_today()` | 防重复信号                |

### 4.2 空头入场（全部条件必须同时满足）

| # | 条件 | 阈值/判断 | 设计理由 |
|---|------|-----------|----------|
| 1 | 品种排名在后 5 | `ranking_top_n=5` | 只做最弱下降趋势 |
| 2 | MA10 斜率 < -最低阈值 | `-min_slope_threshold` | 排除走平品种 |
| 3 | 当前无该品种的空头持仓 | `position.direction != -1` | 不重复开仓 |
| 4 | 当前总多头数 < 上限 | `max_positions_per_side=5` | 风险敞口对称 |
| 5 | 当日无 PENDING ENTRY 信号 | `_already_signaled_today()` | 防重复 |

### 4.3 入场信号 → 执行流程

```
收盘后 (信号生成)
  └── compute_rankings() → 确定候选品种
      └── 对每个候选品种检查入场条件
          └── 通过 → 创建 DailyStrategySignal(trade_type=ENTRY, executed_status=PENDING)
          
次日开盘 (信号执行)
  └── 查询所有 PENDING ENTRY 信号
      ├── 先平仓所有 PENDING STOP_LOSS（若有）
      └── 再执行入场：TargetPosTask(symbol, target_lots)
          ├── 成功 → 更新信号为 SUCCESS, 创建 PositionState
          └── 失败 → 更新信号为 FAILED
```

### 4.5 浮动排名处理

排名每日变化，可能出现以下情况：

| 场景 | 处理方式 |
|------|----------|
| 今日 #1 → 明日 #3 | 正常持有，不触发任何操作 |
| 今日 #5 → 明日 #6 | 正常持有（还在前 12 名内） |
| 今日 #5 → 明日 #13 | **触发排名离场**（跌出前 12） |
| 今日 #5 → 明日 #11 | 正常持有（11 < 12，仍在缓冲区内） |

**为什么前 5 入场但到第 12 名才离场？**
这是趋势跟踪的核心技巧——用缓冲区减少"刚入场就出场"的反复磨损。5→12 的 7 名缓冲 ≈ 海龟的 20 日入场/10 日离场的 2:1 比例，实践证明可以有效过滤噪音。

## 五、离场规则

### 5.1 优先级

三种离场条件**任意一条触发即执行平仓**：

```
离场优先级：
  1. 硬止损（最高优先）  ← 盘中实时保护
  2. 趋势反转离场       ← 趋势方向已变
  3. 排名退化离场       ← 相对强度消失
```

### 5.2 离场条件详解

#### 条件 A：排名退化离场（主要离场方式）

| 持仓方向 | 触发条件 | 说明 |
|----------|----------|------|
| 多头 | 品种排名不在前 12 | 从 #5 跌到 #13+，相对强度显著消失 |
| 空头 | 品种排名不在倒数 12 | 同理 |

**为什么不直接用前 5 / 后 5 离场？**
- 如果前 5 入场、掉出前 5 就离场，每天都有大量换手
- MA10 斜率日间波动大，排名在 #3~#8 之间来回浮动是常态
- `exit_threshold_rank=12` 提供了 7 名的缓冲空间

#### 条件 B：趋势反转离场（辅助离场）

| 持仓方向 | 触发条件 | 设计理由 |
|----------|----------|----------|
| 多头 | `ma_10_slope_pct < 0` | MA10 已拐头向下，即使还在排名前 12（可能其他品种跌更多），自身趋势已结束 |
| 空头 | `ma_10_slope_pct > 0` | MA10 已拐头向上，同理 |

**为什么需要这个条件？**
"矮子里拔将军"场景：全市场都在下跌，但某品种 MA10 斜率 -0.1% 可能排在前 5。虽然排名仍在多头区，但自身斜率已为负，说明该品种实际也在下跌，只是跌得少——应离场。

#### 条件 C：硬止损（安全网）

| 持仓方向 | 触发条件 | 说明 |
|----------|----------|------|
| 多头 | `price <= entry_price - 2.5 × ATR` | 入场价反向 2.5 倍 ATR |
| 空头 | `price >= entry_price + 2.5 × ATR` | 同上 |

**为什么 2.5 倍 ATR 而不是 2 倍？**
- 2 倍 ATR 对于日间波动较大的品种容易误触发
- 2.5 倍提供了更多缓冲，同时能有效控制单笔亏损不超过账户权益的 2.5%
- 排名离场是主要方式，硬止损只是最后的保护

## 六、仓位与资金管理

### 6.1 仓位计算

```
品种风险 = 账户权益 × risk_per_trade_pct
          = 默认 1,000,000 × 1.0% = 10,000 元/品种

单位手数 = floor(品种风险 / (atr_stop_multiplier × ATR × 合约乘数))
         = floor(10,000 / (2.5 × 50 × 10))
         = floor(8) = 8 手
```

### 6.2 风险敞口

| 指标 | 值 |
|------|-----|
| 每品种风险 | 权益的 1.0% |
| 最大多头数 | 5 |
| 最大空头数 | 5 |
| 单边最大风险 | 权益的 5.0% |
| 总最大风险 | 权益的 10.0%（多空同时存在时） |
| 空头自然对冲 | 多空并存降低净风险敞口 |

### 6.3 不加仓的理由

MA10 斜率排名策略的核心是**品种间的横向比较**，而非单品种的纵向增强。以下从三个维度说明为什么不需要加仓。

#### 维度一：横向选择 vs 纵向加仓 —— 逻辑矛盾

| 维度 | 海龟系统（有加仓） | MA10 斜率排名（无加仓） |
|------|-------------------|----------------------|
| **入场依据** | 价格突破 20 日 Donchian 通道 | 全市场 MA10 斜率排名前/后 5 |
| **加仓逻辑** | 价格继续朝突破方向运动 → 趋势被确认 → 加大仓位捕捉利润 | **不适用**——入场依据来自横向排名，非单品种趋势强度 |
| **风险分布** | 少数品种上集中仓位，靠加仓捕捉主趋势 | 5 多 5 空，每个品种 1 单位，天然分散 |
| **品种角色** | 每个品种独立交易，各自判断加仓 | 品种可替换，#6 随时可能取代 #5 |

海龟加仓的根源是：**入场信号（20 日突破）相对迟钝，需要靠加仓来补足仓位**——第一次只进 1/3 目标仓位，价格再涨才把剩下的 2/3 加上去。而 MA10 斜率排名一旦进入前 5，**一次就进满 1 单位**，不存在分批建仓的需求。

#### 维度二：三个具体问题

| 问题 | 说明 |
|------|------|
| **分散 vs 集中** | 策略本质是靠多品种分散捕捉趋势。加仓意味着在单一品种上投入双倍资金，排挤了其他潜在强势品种的入场名额（#5 本可持有，却因 #1 加仓占用了仓位上限而被挡在门外） |
| **排名流动性** | MA10 斜率日间波动大，排名每天变化。今日 #1 明日可能掉到 #8，若持有双倍仓位，不仅损失更大，还占用了两个名额，影响其他品种入场 |
| **拥挤效应** | 加仓机制隐含"斜率越高、仓位越重"，但 #1 恰好是最近涨幅最大的品种，追高风险最高。加仓在最贵的价位上放大了风险 |

#### 维度三：如何应对趋势持续性

如果某品种趋势持续性特别强（连续多日排名前 3），更好的处理方式是**让它自然复利**——只要持续排在前 5，策略自动会持续持有它，无需加仓也能充分捕捉趋势。

若未来需要增强趋势暴露度，正确的方向是：

1. **缩小排名阈值**（如从 top 5 改为 top 3）—— 更精选，而非加仓
2. **缩短排名重算周期**（如引入盘中重排）—— 更快响应排名变化
3. **增加对冲仓位**（如从 5:5 改为 6:6）—— 增加品种多样性

以上三者都不需要在单一品种上加仓。

## 七、参数一览

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `SLOPE_PERIOD` | 3 | MA10 斜率计算周期（日） |
| `RANKING_TOP_N` | 5 | 每方向候选品种数量 |
| `EXIT_THRESHOLD_RANK` | 12 | 排名退出阈值 |
| `MIN_SLOPE_THRESHOLD` | 0.05(%) | 最小斜率阈值 |
| `MAX_POSITIONS_PER_SIDE` | 5 | 每方向最大持仓数 |
| `RISK_PER_TRADE_PCT` | 1.0(%) | 每笔风险占权益百分比 |
| `ATR_STOP_MULTIPLIER` | 2.5 | 硬止损 ATR 倍数 |
| `ACCOUNT_NAME` | 510988 | 运行账户 |

参数硬编码在 `run_ma10_slope.py` 的 `DEFAULTS` 字典中，**不从数据库加载**，因此不需要在 models.py 中新增字段或修改 `config_loader.py`。

```
        │   ├─ 排名退化检查 (rank > 12?)
        │   ├─ 趋势反转检查 (slope < 0 且为多头?)
        │   └─ 硬止损检查 (价格 vs 入场价 ± 2.5×ATR)
        │   └─ 需平仓 → 立即执行 TargetPosTask(0)
        │
 15:02 ├─ 生成入场信号
        │   ├─ Top 5 → DailyStrategySignal(trade_type=ENTRY, executed_status=PENDING)
        │   └─ Bottom 5 → 同上
        │
        │  ═══════════ 隔夜 ════════════
        │
 09:00 ─── 开盘 ────────────────────────────────────────────────────
        │
 09:01 ├─ 执行 PENDING 信号
        │   ├─ 先平仓: STOP_LOSS → TargetPosTask(0)
        │   ├─ 再开仓: ENTRY → TargetPosTask(+/-lots)
        │   └─ 跳空保护检查 (gap > 1.5×ATR?)
        │
 09:02 └─ 更新持仓状态
        │   ├─ PositionState 更新
        │   └─ 信号状态 SUCCESS / FAILED
        │
        │  ═══════════ 盘中 ════════════
        │
 09:02   ── wait_update 循环 ──
 ～      │
 15:00   ├─ 检查硬止损 (price vs entry_price ± 2.5×ATR)
         ├─ 触发 → 立即平仓
         └─ 检查定时重启 (8:55/20:55)
```

### 8.2 monitor 模式主循环

```
while True:
    api.wait_update()
    now = datetime.now()
    
    if 15:00 <= now.time() < 15:02:
        # 收盘后执行排名 + 信号生成
        compute_rankings(api, account)
        check_exit_conditions(api, account)
        generate_entry_signals(account)
        time.sleep(60)  # 避免重复执行
    
    if now.hour == 9 and now.minute in (1, 2):
        # 开盘执行 PENDING 信号
        execute_pending_signals(api, account)
    
    # 实时硬止损检查
    for each open position:
        check_hard_stop(api, position, quote)
    
    # 定时重启
    if now.hour in (8, 20) and now.minute == 55:
        raise RestartSignal()
```

**不设盘中排名重算**：排名只在收盘后更新。盘中不重排，避免因盘中价格波动导致排名反复变化产生频繁交易。

## 十、策略的理论基础

### 10.1 趋势跟踪的核心逻辑

1. **优势来源**：期货市场存在趋势性（商品周期、产业利润驱动），趋势一旦形成往往持续
2. **切入点**：MA10 斜率正/负且排名靠前 = 处于该商品方向性最强的阶段
3. **持有期**：趋势持续即持有，趋势减弱或消失即离场

### 10.2 为什么是排名（相对强度）而非阈值（绝对强度）？

| 方式 | 优点 | 缺点 | 本策略选择 |
|------|------|------|-----------|
| 绝对阈值 | 规则简单明确 | 不同市场环境效果差异大；震荡市可能无交易 | ✗ |
| 相对排名 | 始终有交易信号；适应不同市场环境 | 弱市中可能交易弱趋势 | ✔ |

### 10.3 多空对冲的数学意义

同时持有 5 多 5 空，市场系统性风险（如加息导致全市场下跌）被自然对冲，策略收益主要来自**品种间的趋势强弱差异**而非市场方向性暴露。

### 10.4 与海龟系统的互补关系

| 环境 | 海龟系统 | MA10 斜率排名 |
|------|----------|---------------|
| 强趋势市场（如 2020 年商品牛市） | 表现好 | 表现好（全市场都在趋势中） |
| 结构性行情（部分涨部分跌） | 中规中矩 | **最佳**——精选最强/最弱 |
| 震荡市（全市场无方向） | 反复止损磨损 | 同样磨损（矮子里拔将军） |
| 大反转（V 型反转） | 趋势确认后才入场 | **更快反应**——MA10 斜率快速变化 |

## Alpha 因子分析

### 策略定位

MA10 斜率排名是系统内**唯一一个横截面（cross-sectional）策略**，与其余所有时间序列（time-series）策略的 Alpha 结构完全不同。它不预测单品种价格方向，而是比较全市场品种的相对强弱。

### 因子拆解

#### 因子一：MA10 斜率（趋势动量因子）

**公式**：
```
slope_pct = (MA10[t] - MA10[t-3]) / MA10[t-3] × 100
```

**数理本质**：MA10 是 10 日均价（已平滑噪音），3 日变化率测量短期动量。这一信号同时具备：
- **趋势跟踪特性**：斜率正 → 价格在上涨，负 → 在下跌
- **平滑性**：MA10 比日收盘价更平滑，减少假信号
- **可比性**：百分比标准化后，cu（铜 70000）和 MA（甲醇 1800）的斜率可以直接比较

**因子属性**：

| 属性 | 值 |
|------|-----|
| 因子类型 | 时间序列动量（滞后版） |
| 计算周期 | 10 日均值 + 3 日变化率 |
| 信号特征 | 连续值（非二元信号），无阈值跳变 |
| 对噪音的敏感度 | 低（MA10 已过滤日间波动） |
| 对趋势变化反应 | 中（滞后约 3-5 个交易日） |

#### 因子二：横截面排名（相对强度因子）

这是本策略**最核心的 Alpha 来源**，也是与海龟系统最根本的差异。

**逻辑**：
```
多头候选 = top_5(所有品种, 按 slope_pct 降序)
空头候选 = bottom_5(所有品种, 按 slope_pct 升序)
```

**经济学含义**：在商品期货市场中，同一时期不同品种的趋势差异很大。有的品种在大涨（如 2020 年铁矿），有的在大跌（如同期原油）。横截面排名策略捕捉的是**品种间的趋势分化**，而非某个品种的绝对趋势。

| 特征 | 海龟系统（时间序列） | MA10 排名（横截面） |
|------|-------------------|-------------------|
| 对比基准 | 自身历史价格 | 全市场其他品种 |
| 信号含义 | "这个品种现在在涨" | "这个品种比大多数涨得更好" |
| 市场环境依赖 | 需要整体趋势 | 需要品种间分化 |
| 多空必要性 | 可选（单向也可） | **必要**（多空对冲降低风险） |

**横截面的数学优势**：
```
横截面策略的收益 = 品种间分化度 × 排名选股能力 - 交易成本

当全市场无趋势时（所有品种涨跌互现），分化度可能仍然很高，
横截面策略依然能找到多空机会，而时间序列策略则无信号。
```

#### 因子三：排名缓冲区（噪声过滤因子）

```
入场排名：Top 5  /  Bottom 5
出场排名：Top 12 /  Bottom 12
缓冲区宽度：7 名（≈ 趋势跟踪的 2:1 入场/出场比例）
```

**代数意义**：
```
换手率降低 = 1 - (入场阈值 / 出场阈值) = 1 - (5/12) ≈ 58%
```

即缓冲区减少了约 58% 的潜在换手。排名的日间波动是噪声，缓冲区防止了"今天入场、明天出场"的反复磨损。

#### 因子四：三层离场体系

| 离场方式 | 条件 | Alpha 贡献方式 |
|---------|------|---------------|
| 排名退化 | rank > 12 | 相对强度消失→离场 |
| 趋势反转 | slope 由正转负（多头）/ 由负转正（空头） | 自身趋势已变→离场 |
| 硬止损 | price < entry ± 2.5×ATR | 价格异常波动→保护 |

这三层离场不是冗余，而是覆盖三种不同的风险场景：

```
排名退化：大家都在涨，但你在涨的品种中掉队了 → 相对强度消失
趋势反转：你持仓的方向在 MA10 层面已经转向 → 绝对趋势消失
硬止损：价格在极端反向波动 → 尾部风险保护
```

#### 因子五：多空对冲（市场中性因子）

同时持有 5 多 + 5 空，净敞口理论为零。

**代数意义**：
```
策略收益 ≈ Long_Alpha - Short_Alpha ≈ 2 × 选股Alpha
市场收益 ≈ 0（多空对冲后）

如果 MA10 斜率排名有效（多头选中的品种跑赢空头选中的品种），
那么无论市场涨跌，策略都能盈利。
```

**多空对冲的局限性**：
- 在单边暴涨/暴跌行情中，多空可能双亏（多头赚的少、空头亏的多）
- MA10 斜率在极端行情中可能所有品种同向，排名意义下降

### 与海龟系统的 Alpha 结构对比

| 维度 | 海龟系统（时间序列） | MA10 排名（横截面） |
|------|-------------------|-------------------|
| Alpha 来源 | 单品种趋势延续 | 品种间相对强弱 |
| 市场环境依赖 | 需要全局趋势 | 需要品种分化 |
| 震荡市表现 | 差（反复假突破） | 中（分化度仍可能存在） |
| 强趋势市表现 | 极好 | 好 |
| 结构性行情 | 一般 | **极好** |
| 资金容量 | 大（单品种独立） | 中（受限于流动性品种数量） |
| 策略容量 | 不限品种数 | 最多 5 多 + 5 空 |

### 因子贡献度估算

| 因子 | Alpha 贡献 | 说明 |
|------|-----------|------|
| MA10 斜率排名 | ~55% | 核心信号来源——选出最强/最弱品种 |
| 多空对冲 | ~20% | 降低系统性风险暴露，提升风险调整收益 |
| 排名缓冲区 | ~12% | 减少噪音交易，降低换手成本 |
| 三层离场 + 硬止损 | ~8% | 多维度出场保护 |
| ATR 头寸规模 | ~5% | 波动率自适应 |
