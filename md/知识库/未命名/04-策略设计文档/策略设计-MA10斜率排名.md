# 策略设计 — MA10 斜率排名（2026-05-12）

> 核心思想：每日计算全市场品种的 MA10 斜率，取上升趋势最强的前 5 名做多、下降趋势最强的前 5 名做空。不依赖唐奇安通道突破，不加仓、不移仓。

---

## 一、与现有 Donchian 系统的关键差异

| 方面 | Donchian 系统 | MA10 斜率排名（本策略） |
|------|--------------|------------------------|
| **入场依据** | 价格突破 20 日唐奇安通道 | MA10 斜率在全市场排名前/后 5 |
| **趋势衡量** | 二元判断（突破/不突破） | 连续值（slope_pct），可跨品种比较 |
| **品种选择** | 所有已配置品种各自独立判断 | **全市场排名择优**，只做最强/最弱 |
| **加仓** | 0.5/1.0 ATR 加仓，最多 3 单位 | **不加仓**，每品种 1 单位固定 |
| **移仓** | 主力合约切换时自动移仓 | **不移仓**（持仓合约到期前主动离场） |
| **出场逻辑** | ATR 移动止损 | **排名退化 + 趋势反转 + 硬止损 三层** |
| **风险分散** | 资金按品种数量均分 | **多空对冲**，5 多 + 5 空天然平衡 |
| **运行方式** | 收盘生成信号→次日开盘执行 | **同左**，沿用现有调度框架 |

## 二、MA10 斜率计算

### 2.1 公式

```
slope_pct = (MA10[t] - MA10[t - slope_period]) / MA10[t - slope_period] * 100
```

- `MA10[t]` = 当日收盘后的 10 日移动平均线
- `slope_period` = 3（默认），即对比 MA10 在 3 个交易日内的变化率
- `slope_pct` 以百分比表示，正数 = 上升趋势，负数 = 下降趋势

### 2.2 为什么用百分比？

不同品种价格水平差异极大（铜 ≈ 70000 元/吨，玉米 ≈ 2400 元/吨），直接用 MA10 差值不可比。百分比将 slope 统一为 [0%, +∞) 的标量，使跨品种排序有意义。

### 2.3 为什么用 MA10 斜率而不是 MA5 或 MA20？

- **MA5 斜率**太敏感，噪音多，频繁换仓
- **MA20 斜率**太滞后，趋势启动后很久才反应
- **MA10 斜率**是平衡点：MA10 本身已平滑了 10 天的价格噪音，3 日变化率又能及时发现趋势变化

### 2.4 为什么 slope_period=3？

MA10 已经是 10 日均值，再取 5 日以上的变化率就过于平滑，对趋势变化反应太慢。3 日（约半个交易周）能较好平衡灵敏度和稳定性。

### 2.5 计算原理图

下面以螺纹钢（rb）为例，展示 MA10 与斜率的关系：

```
价格

 │     ╲        ╱  ╲        ← slope_pct = +0.88%

```

斜率绝对值越大 = 趋势越陡峭 = 该品种在对应方向上的 momentum 越强。

## 三、每日排名流程

### 3.1 处理步骤

```
15:00 市场收盘
  │
  ├─ Step 1: 遍历所有活跃主力合约（FullContractList）
  ├─ Step 2: 查询 KlineData，获取最近 13+ 天的 ma_10 数据
  │          （slope_period=3 + MA10 需要 10 天预热 + 3 天缓冲）
  ├─ Step 3: 计算每个品种的 ma_10_slope_pct
  ├─ Step 4: 排除数据不足 / 非主力 / 不活跃品种
  ├─ Step 5: 按 slope_pct 从大到小排序
  ├─ Step 6: 取前 5 名 = 多头候选，后 5 名 = 空头候选
  │
  └─ 15:32 生成/更新信号 → 次日 09:00-09:02 执行
```

## 四、入场规则

### 4.1 多头入场（全部条件必须同时满足）

| # | 条件 | 阈值/判断 | 设计理由 |
|---|------|-----------|----------|
| 1 | 品种排名在前 5 | `ranking_top_n=5` | 只做最强上升趋势，淘汰跟风品种 |
| 2 | MA10 斜率 > 最低阈值 | `min_slope_threshold=0.05` | 斜率基本为零时 MA10 走平，不应入场 |
| 3 | 当前无该品种的多头持仓 | `position.direction != 1` | 同一品种不重复开仓 |
| 4 | 当前总多头数 < 上限 | `max_positions_per_side=5` | 控制总风险敞口 |
| 5 | 当日无 PENDING ENTRY 信号 | `check_duplicate_pending_signal` | 防重复信号 |

### 4.2 空头入场（全部条件必须同时满足）

| # | 条件 | 阈值/判断 | 设计理由 |
|---|------|-----------|----------|
| 1 | 品种排名在后 5 | `ranking_top_n=5` | 只做最弱下降趋势 |
| 2 | MA10 斜率 < -最低阈值 | `-min_slope_threshold` | 排除走平品种 |
| 3 | 当前无该品种的空头持仓 | `position.direction != -1` | 不重复开仓 |
| 4 | 当前总空头数 < 上限 | `max_positions_per_side=5` | 风险敞口对称 |
| 5 | 当日无 PENDING ENTRY 信号 | `check_duplicate_pending_signal` | 防重复 |

### 4.3 入场执行

- 信号于收盘后生成，状态 `PENDING`
- **次日 09:00-09:02 开盘执行**（沿用 `tasks_daily_open.py` 框架）
- 手数计算：`unit_lots = floor(equity × 1% / (2.5 × ATR × 合约乘数))`
- 跳空保护：沿用现有 `gap_protection_ratio` 逻辑，开仓时判断

### 4.5 浮动排名处理

排名每日变化，可能出现以下情况：

| 场景 | 处理方式 |
|------|----------|
| 今日 #1 → 明日 #3 | 正常持有，不触发任何操作 |
| 今日 #5 → 明日 #6 | 正常持有（还在前 12 名内） |
| 今日 #5 → 明日 #13 | **触发排名离场**（跌出前 12） |
| 今日 #5 → 明日 #11 | 正常持有（11 < 12，仍在缓冲区内） |

**为什么前 5 入场但到第 12 名才离场？**
这是趋势跟踪的核心技巧——用缓冲区减少"刚入场就出场"的反复磨损。5→12 的 7 名缓冲 ≈ 海龟的 20 日入场/10 日离场的 2:1 比例，实践证明可以有效过滤噪音。

## 五、离场规则

### 5.1 优先级

三种离场条件**任意一条触发即执行平仓**：

```
离场优先级：
  1. 硬止损（最高优先）  ← 盘中实时保护
  2. 趋势反转离场       ← 趋势方向已变
  3. 排名退化离场       ← 相对强度消失
```

### 5.2 离场条件详解

#### 条件 A：排名退化离场（主要离场方式）

| 持仓方向 | 触发条件 | 说明 |
|----------|----------|------|
| 多头 | 品种排名不在前 12 | 从 #5 跌到 #13+，相对强度显著消失 |
| 空头 | 品种排名不在倒数 12 | 同理 |

**为什么不直接用前 5 / 后 5 离场？**
- 如果前 5 入场、掉出前 5 就离场，每天都有大量换手
- MA10 斜率日间波动大，排名在 #3~#8 之间来回浮动是常态
- `exit_threshold_rank=12` 提供了 7 名的缓冲空间，等效于海龟系统的 2:1 入场/离场比例

#### 条件 B：趋势反转离场（辅助离场）

| 持仓方向 | 触发条件 | 设计理由 |
|----------|----------|----------|
| 多头 | `ma_10_slope_pct < 0` | MA10 已拐头向下，即使还在排名前 12（可能其他品种跌更多），自身趋势已结束 |
| 空头 | `ma_10_slope_pct > 0` | MA10 已拐头向上，同理 |

**为什么需要这个条件？**
"矮子里拔将军"场景：全市场都在下跌，但 M10 斜率 -0.1% 可能排在前 5。虽然排名仍在多头区，但自身斜率已为负，说明该品种实际也在下跌，只是跌得少——应离场。

#### 条件 C：硬止损（安全网）

| 持仓方向 | 触发条件 | 说明 |
|----------|----------|----------|
| 多头 | `price <= entry_price - 2.5 × ATR` | 入场价反向 2.5 倍 ATR |
| 空头 | `price >= entry_price + 2.5 × ATR` | 同上 |

**为什么 2.5 倍 ATR 而不是 2 倍？**
- 2 倍 ATR 对于日间波动较大的品种容易误触发
- 2.5 倍提供了更多缓冲，同时能有效控制单笔亏损不超过账户权益的 2.5%
- 排名离场是主要方式，硬止损只是最后的保护

### 5.3 缓冲区示意图

```
排名    区域          操作
────────────────────────────────────────────
 #1     ┌─────────┐
 #2     │  多头区  │  ← 入场做多
 #3     │ (Top 5) │
 #4     │         │
 #5     └─────────┘
 #6     ┌─────────┐
 #7     │  缓冲区  │  ← 持有但不入场
 #8     │(6 ~ 12) │    排名在此波动时不触发操作
 #9     │         │
 #10    │         │
 #11    │         │
 #12    └─────────┘
 #13+          ← 触发排名离场（多头跌出缓冲区）

 ...         (中间品种不操作)

 #N-12  ┌─────────┐
 #N-11  │  缓冲区  │  ← 持有但不入场（空头侧）
 #N-10  │(6 ~ 12) │
 ...    │         │
 #N-6   └─────────┘
 #N-5   ┌─────────┐
 #N-4   │  空头区  │  ← 入场做空
 #N-3   │(Bottom5)│
 #N-2   │         │
 #N-1   │         │
 #N     └─────────┘
────────────────────────────────────────────
```

**特点**：
- 多头/空头各 5 个入场位（共 10 个可能持仓）
- 缓冲区内 7 个名次（6~12 及 N-12~N-6）当持有状态，不触发新操作
- 从 #5 跌到 #13 才离场，避免日间排名波动导致频繁交易
- 缓冲区大小 ≈ 入场区 × 2.4 倍，与海龟系统 20 日入场/10 日离场的比例接近

- 平仓后该品种在本周内不再入场（从 PENDING 信号触发时就已占位）
- 平仓记录写入 `ClosedPositionRecord`，与现有系统一致
- 每日收盘重新排名，符合条件的品种次日可重新入场（不设冷却期）

## 六、仓位与资金管理

### 6.1 仓位计算

```
品种风险 = 账户权益 × risk_per_trade_pct
          = 默认 1,000,000 × 1.0% = 10,000 元/品种

单位手数 = floor(品种风险 / (atr_stop_multiplier × ATR × 合约乘数))
         = floor(10,000 / (2.5 × 50 × 10))
         = floor(8) = 8 手
```

### 6.2 风险敞口

| 指标 | 值 |
|------|-----|
| 每品种风险 | 权益的 1.0% |
| 最大多头数 | 5 |
| 最大空头数 | 5 |
| 单边最大风险 | 权益的 5.0% |
| 总最大风险 | 权益的 10.0%（多空同时存在时） |
| 空头自然对冲 | 多空并存降低净风险敞口 |

### 6.3 不加仓的理由

MA10 斜率排名策略的核心是**品种间的横向比较**。如果对已有持仓的品种加仓，意味着在同一个品种上追加风险，这与"分散到最强趋势品种"的理念矛盾。排名靠前的 5 个品种本身就代表了当前最强的趋势，不需要在单一品种上集中风险。

## 七、参数一览

| 参数 | 默认值 | 类型 | 说明 |
|------|--------|------|------|
| `slope_period` | 3 | int | MA10 斜率计算周期（日） |
| `ranking_top_n` | 5 | int | 每方向候选品种数量 |
| `exit_threshold_rank` | 12 | int | 排名退出阈值（入场排名 × ~2.4） |
| `min_slope_threshold` | 0.05 | Decimal(%) | 最小 |斜率| 阈值 |
| `max_positions_per_side` | 5 | int | 每方向最大持仓数 |
| `risk_per_trade_pct` | 1.0 | Decimal(%) | 每笔风险占权益百分比 |
| `atr_stop_multiplier` | 2.5 | Decimal | 硬止损 ATR 倍数 |

**不设盘中排名重算**：排名只在收盘后更新。盘中不重排，避免因盘中价格波动导致排名反复变化产生频繁交易。

## 九、策略的理论基础

### 9.1 趋势跟踪的核心逻辑

1. **优势来源**：期货市场存在趋势性（商品周期、产业利润驱动），趋势一旦形成往往持续
2. **切入点**：MA10 斜率正/负且排名靠前 = 处于该商品方向性最强的阶段
3. **持有期**：趋势持续即持有，趋势减弱或消失即离场

### 9.2 为什么是排名（相对强度）而非阈值（绝对强度）？

| 方式 | 优点 | 缺点 | 本策略选择 |
|------|------|------|-----------|
| 绝对阈值 | 规则简单明确 | 不同市场环境效果差异大；震荡市可能无交易 | ✗ |
| 相对排名 | 始终有交易信号；适应不同市场环境 | 弱市中可能交易弱趋势 | ✔ |

### 9.3 多空对冲的数学意义

同时持有 5 多 5 空，市场系统性风险（如加息导致全市场下跌）被自然对冲，策略收益主要来自**品种间的趋势强弱差异**而非市场方向性暴露。

### 9.4 与现有 Donchian 系统的互补关系

| 环境 | Donchian 系统 | MA10 斜率排名 |
|------|---------------|---------------|
| 强趋势市场（如 2020 年商品牛市） | 表现好 | 表现好（全市场都在趋势中） |
| 结构性行情（部分涨部分跌） | 中规中矩 | **最佳**——精选最强/最弱 |
| 震荡市（全市场无方向） | 反复止损磨损 | 同样磨损（矮子里拔将军） |
| 大反转（V 型反转） | 趋势确认后才入场 | **更快反应**——MA10 斜率快速变化 |
