# 原版海龟交易系统对比组

## 功能概述

新增独立的 Django 管理命令 `run_turtle.py`，实现**原版海龟交易系统**规则（仅最大仓位改为 3 个单位）。通过自定义命令方式运行，**不修改任何现有文件**，所有数据写入现有 DB 表，与现有系统运行结果进行对比分析。

> 设计目标：在同一数据库内，用不同账户分别运行现有系统和海龟系统，通过账户筛选在 Dashboard 和报表中对比两者的绩效差异。

---

## 与原系统的关键差异

| 方面 | 现有系统 | 海龟系统 |
|------|----------|----------|
| 运行方式 | **收盘后生成信号 → 下一开盘执行** | **盘中实时监控，触发即执行** |
| 入场判断 | 基于日收盘价突破唐奇安通道 | 基于盘中最新价突破唐奇安通道 |
| 出场信号 | ATR 移动止损（最高收盘价 - 2×ATR） | **10日唐奇安反向突破**（盘中实时判断） |
| 仓位计算 | 固定 `POSITION_RISK_BASE_AMOUNT=4000` | 动态 `Equity × 1% / (2 × N × 合约乘数)` |
| 硬止损 | 无 | **入场价 - 2×N**（盘中实时监控） |
| 加仓触发 | `last_add_price` + 0.5/1.0×ATR，收盘判断 | **盘中每 0.5×N 加 1 单位**，实时触发 |
| 单位追踪 | 仅记录最新加仓价 | JSON 存储所有单位入场价 |
| 风险比例 | 无 | 每单位风险 = 账户权益的 1% |
| 品种间联动 | 各品种独立 | 资金按比例分配到各品种（1% / 品种） |

## 实时监控流程

原版海龟的核心理念是**盘中实时突破即入场**，不是收盘后才判断。

### 整体流程

```
┌─ 启动命令 ─────────────────────────────────────────────┐
│  1. 创建 TqApi 连接                                      │
│  2. 获取账户信息、品种配置                                │
│  3. 获取日 K 线数据，预计算所有品种的唐奇安通道 + N 值     │
│  4. 进入 wait_update 实时循环                            │
│     ├── 监控所有活跃品种的最新价                           │
│     ├── 无持仓 → 检查 20 日通道突破（入场）               │
│     ├── 有持仓 → 检查 10 日反向突破（出场）               │
│     ├── 有持仓 → 检查 2N 硬止损                          │
│     ├── 有持仓 units<3 → 检查 0.5N 加仓                  │
│     ├── 移仓检查                                         │
│     └── 每日开盘时重新计算 N 值                            │
│  5. 收盘自动退出                                        │
└─────────────────────────────────────────────────────────┘
```

### TqSDK 实时循环核心模式

```python
api = create_tqapi(account)

# 预获取所有品种的日 K 线 + 报价
klines = {}      # symbol → kline_serial
quotes = {}      # symbol → quote
for sym in symbols:
    klines[sym] = api.get_kline_serial(sym, duration_seconds=86400, data_length=65)
    quotes[sym] = api.get_quote(sym)

# 预计算各品种的唐奇安通道
levels = {}      # symbol → {h20, l20, h10, l10, n_value}
for sym, kline in klines.items():
    levels[sym] = compute_donchian_levels(kline)

# 实时循环
while True:
    api.wait_update()

    for symbol in symbols:
        if not is_changing(quotes[symbol], 'last_price'):
            continue

        price = quotes[symbol].last_price

        position = PositionState.objects.filter(account=account, symbol=symbol).first()

        if not position or position.units == 0:
            # 入场检查
            if price > levels[symbol]['h20']:
                execute_entry(symbol, direction=1, ...)
            elif price < levels[symbol]['l20']:
                execute_entry(symbol, direction=-1, ...)
        else:
            # 出场检查（10日反向突破）
            if position.direction == 1 and price < levels[symbol]['l10']:
                execute_exit(position, ...)
            elif position.direction == -1 and price > levels[symbol]['h10']:
                execute_exit(position, ...)

            # 硬止损（2N）
            if position.direction == 1 and price <= hard_stop:
                execute_exit(position, ...)
            elif position.direction == -1 and price >= hard_stop:
                execute_exit(position, ...)

            # 加仓检查（0.5N）
            if position.units < 3 and price >= next_add_trigger:
                execute_add_on(position, ...)
```

## 实现方案

### 新建文件

`backend/stock/management/commands/run_turtle.py`（单文件，约 700-900 行）

### 代码模块

| 模块 | 行数 | 说明 |
|------|------|------|
| 常量 & 导入 | ~40 | 复用 `create_tqapi`, `safe_close_api`, `wait_for_target_position`, `record_and_reset_position` 等 |
| 唐奇安通道计算 | ~50 | 基于日 K 计算 20日/10日高低点 + ATR(N值) |
| 实时监控循环 | ~200 | `wait_update` 主循环，监控所有品种并判断信号 |
| 入场执行 | ~120 | 仓位计算 → TargetPosTask → PositionState 创建 |
| 出场执行 | ~80 | 平仓 → 成交价确认 → ClosedPositionRecord 记录 |
| 加仓执行 | ~80 | 增加 1 单位 → PositionState 更新 |
| 移仓 | ~80 | 旧合约平仓 → 新合约开仓 → PositionState 迁移 |
| 工具函数 | ~100 | 防重复信号、仓位刷新、行情检查、品种级别资金分配 |

### N 值（ATR）管理

```
┌─ 每日开盘 ──────────────────────────────────────┐
│  用最新日 K 线计算今 N'ATR(20) → 盘中实时用当天 N │
│  判断新入场                                     │
└─────────────────────────────────────────────────┘
                        │
        ┌───────────────┴───────────────┐
        ▼                               ▼
  无持仓: 用今天 N 计算            有持仓: 用入场时锁定的 N
  单位大小和入场位置                计算加仓/止损位置

入场后锁定：一旦开仓，该笔交易全程使用开仓日的 N 值
  ├─ 加仓触发价 = 最后单位价格 ± 0.5 × 锁定N
  ├─ 硬止损价   = 入场价格 ± 2 × 锁定N
  └─ 所有单位共用一个锁定 N，不重新计算
```

### 单位价格追踪

通过 `PositionState.indicators` JSON 字段存储：

```json
{
  "locked_n": 120.5,
  "unit_prices": [4250.0, 4310.0],
  "hard_stop_price": 4010.0,
  "exit_10h": 4380.0,
  "exit_10l": 4120.0
}
```

- `locked_n` — 入场时锁定的 N（所有单位共用）
- `unit_prices` — 每个单位入场价格列表，用于加仓触发判断（从最后一个起算 0.5×N）

### 品种级别资金分配

原版海龟规则中，每个品种分配固定的资金比例：
- 每品种风险 = `账户权益 × 1%`
- 每品种单位数 = `floor(品种风险 / (2 × N × 合约乘数))`
- 同一品种最多 3 个单位（修改自原版的 4 个）
