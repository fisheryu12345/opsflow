# 双均线MA10/20交叉交易系统对比组

## 功能概述

新增独立的 Django 管理命令 `run_ma_crossover.py`，实现**双均线 MA10/20 交叉交易系统**规则（金叉做多、死叉做空、无加仓）。通过自定义命令方式运行，**不修改任何现有文件**，所有数据写入现有 DB 表，与现有系统和原版海龟系统的运行结果进行对比分析。

> 设计目标：在同一数据库内，用不同账户分别运行现有海龟系统、原版海龟系统、双均线交叉系统，通过账户筛选在 Dashboard 和报表中对比三者的绩效差异。

---

## 与原系统的关键差异

| 方面 | 现有系统（海龟变异） | 双均线对照系统 |
|------|---------------------|---------------|
| 入场信号 | 收盘价突破唐奇安通道(20日高低点) | 收盘 MA10 上穿/下穿 MA20 |
| 出场信号 | ATR 移动止损（最高收盘价-2×ATR） | **MA10 反向穿越 MA20**（无独立止损） |
| 执行模式 | 收盘生成信号→下一开盘执行 | 同左（T+1 模式） |
| 仓位计算 | 固定 `risk_per_unit=4000` | 动态 `equity × 2% / (2 × ATR × 合约乘数)` |
| 最大单位数 | 3（金字塔加仓） | **1（无加仓）** |
| ATR 使用 | 入场锁定 N 值全程使用 | 每次开仓**重新计算** |
| 加仓触发 | 0.5/1.0×N 从上次加仓价 | **无加仓** |
| 风险比例 | 每单位约 1% 权益 | **每笔 2% 权益**（单次入场） |
| 持仓 unit 数 | 0-3 浮动 | 固定 0 或 1 |
| 硬止损 | 入场价 - 2×N | 无（依赖交叉反向出场） |

## 命令接口

双均线系统通过 `--account-name` 参数指定交易账户进行对比测试。

```bash
# 全模式：检测 MA 交叉 + 立即执行（推荐）
python manage.py run_ma_crossover --mode full --account-name ma_510989

# 信号模式：仅检测交叉并写入 PENDING 信号（收盘后运行）
python manage.py run_ma_crossover --mode signal --account-name ma_510989

# 执行模式：执行待处理信号（开盘运行）
python manage.py run_ma_crossover --mode execute --account-name ma_510989

# 强制平仓：平掉该账户所有持仓
python manage.py run_ma_crossover --mode exit_all --account-name ma_510989
```

## 核心逻辑

双均线是**收盘交叉判断 → 下一开盘执行**的 T+1 模式，不同于原版海龟的盘中实时触发。

### 整体流程

```
┌─ 运行命令（推荐 --mode full）──────────────────────────┐
│                                                         │
│  1. 连接 TqApi                                          │
│  2. 获取 TradingAccount + AccountContractConfig          │
│  3. 遍历每个活跃品种                                     │
│     │                                                    │
│     ├─ 获取日 K 线数据（TqSDK 实时接口, 30根）            │
│     ├─ 计算 MA10 和 MA20                                 │
│     │   └─ ma10_curr = mean(closes[-11:-1])              │
│     │   └─ ma20_curr = mean(closes[-21:-1])              │
│     │   └─ ma10_prev = mean(closes[-12:-2])              │
│     │   └─ ma20_prev = mean(closes[-22:-2])              │
│     ├─ 判断交叉方向                                       │
│     │   ├─ ma10_prev≤ma20_prev 且 ma10_curr>ma20_curr    │
│     │   │   └─ 金叉 → direction=1（做多）                │
│     │   ├─ ma10_prev≥ma20_prev 且 ma10_curr<ma20_curr    │
│     │   │   └─ 死叉 → direction=-1（做空）               │
│     │   └─ 其他 → 无信号，跳过                           │
│     │                                                    │
│     ├─ 检查 PositionState 现有持仓                        │
│     │   ├─ 无持仓 → 直接开新仓                           │
│     │   ├─ 同方向 → 不变（跳过）                         │
│     │   └─ 反方向 → 先平仓 → 再开新仓                    │
│     │                                                    │
│     ├─ 计算仓位大小                                       │
│     │   └─ lots = floor(equity × 2% / (2 × ATR × 乘数))  │
│     │                                                    │
│     ├─ 执行 TargetPosTask                                 │
│     ├─ 写入 DailyStrategySignal + PositionState           │
│     └─ 记录 TradeLog                                     │
│                                                         │
│  4. 移仓检查（检测主力合约变更）                           │
│  5. 关闭 TqApi 连接                                      │
└─────────────────────────────────────────────────────────┘
```

### 仓位计算公式

```
每笔风险金额 = 当前权益 × 2%
每手风险价值 = 2 × ATR(20) × 合约乘数
开仓手数     = floor(每笔风险金额 / 每手风险价值)
最终手数     = max(开仓手数, 1)
```

**示例**（螺纹钢 rb，权益 100 万，ATR=45，乘数=10）：
```
每笔风险金额 = 1,000,000 × 2% = 20,000
每手风险价值 = 2 × 45 × 10 = 900
开仓手数     = floor(20,000 / 900) = 22 手
```

### 信号 Remark 编码

由于 `DailyStrategySignal` 没有 MA 专用字段，MA 值存储在 `remark` 字段：

```
MA10=3812.50, MA20=3795.80, Cross=GOLDEN
MA10=3750.00, MA20=3780.50, Cross=DEATH
```

### PositionState.indicators 编码

```json
{
  "ma_10": 3812.50,
  "ma_20": 3795.80,
  "entry_atr": 45.20
}
```

## 实现方案

### 新建文件

`backend/stock/management/commands/run_ma_crossover.py`（单文件，约 550-650 行）

### 代码模块

| 模块 | 行数 | 说明 |
|------|------|------|
| 导入 & 常量 | ~30 | 复用 `create_tqapi`, `safe_close_api`, `calculate_atr`, `wait_for_target_position`, `record_and_reset_position` |
| Command 入口 & 参数 | ~30 | `--mode`, `--account-name` 参数解析 |
| 模式分派 | ~40 | `signal` / `execute` / `full` / `exit_all` 四模式 |
| MA 交叉检测 | ~50 | K 线获取 → MA10/20 计算 → 金叉/死叉判定 |
| 仓位计算 | ~25 | equity × 2% / (2 × ATR × 乘数) |
| 信号执行 | ~100 | 平反向仓 → 开新仓 → PositionState 更新 |
| 平仓处理 | ~60 | TargetPosTask 平仓 → record_and_reset_position |
| 移仓处理 | ~80 | 主力合约变更检测 → 关旧开新 |
| 工具函数 | ~30 | 活跃合约获取、重复信号检查 |

### 信号类型定义

| `trade_type` | 创建时机 | `executed_status` 流转 |
|-------------|----------|----------------------|
| `ENTRY` | MA 交叉检测到 | PENDING(信号模式) 或 EXECUTING(full模式) → SUCCESS/FAILED |
| `STOP_LOSS` | 反向交叉需要平仓 | EXECUTING → SUCCESS/FAILED |
| `ROLLOVER` | 主力合约变更 | EXECUTING → SUCCESS/FAILED |

### 关键代码复用（import，不修改）

| 函数 | 来源 | 用途 |
|------|------|------|
| `create_tqapi(account)` | `stock/infrastructure/tqapi.py` | 创建 TqApi 连接 |
| `safe_close_api(api)` | `stock/infrastructure/tqapi.py` | 安全关闭连接 |
| `calculate_atr(api, symbol, 20)` | `stock/core/atr.py` | ATR(20) 计算 |
| `wait_for_target_position(...)` | `stock/infrastructure/order_execution.py` | 等待持仓目标达到 |
| `record_and_reset_position(...)` | `stock/infrastructure/order_execution.py` | 平仓记录 + PositionState 重置 |
| `check_duplicate_pending_signal(...)` | `stock/core/signal_checker.py` | 防重复信号 |

## 验证步骤

1. **确认账户配置**
   - 创建双均线对照账户（如 `ma_510989`）
   - 确认已配置 `AccountContractConfig` 交易品种
   - 确认已配置 `StrategyConfig`

2. **信号模式测试**
   ```bash
   python manage.py run_ma_crossover --mode signal --account-name ma_510989
   ```
   → 检查 `DailyStrategySignal` 表是否写入 MA 交叉信号
   → 验证 `remark` 字段格式：`MA10=xxx, MA20=xxx, Cross=GOLDEN/DEATH`

3. **全模式测试（模拟盘）**
   ```bash
   python manage.py run_ma_crossover --mode full --account-name ma_510989
   ```
   → 观察日志输出
   → 检查 `PositionState` 表是否更新持仓
   → 检查 `DailyStrategySignal.executed_status` 是否更新为 SUCCESS

4. **平仓测试**
   ```bash
   python manage.py run_ma_crossover --mode exit_all --account-name ma_510989
   ```
   → 检查 `ClosedPositionRecord` 是否记录平仓
   → 检查 `PositionState` 是否重置

5. **对比分析**
   - 通过 Dashboard 切换不同账户对比三者绩效
   - 在信号监控页对比信号类型和频率
   - 在平仓记录页对比盈亏分布
