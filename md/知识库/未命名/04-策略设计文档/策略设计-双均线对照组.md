# 双均线MA10/20交叉交易系统对比组

## 功能概述

新增独立的 Django 管理命令 `run_ma_crossover.py`，实现**双均线 MA10/20 交叉交易系统**规则（金叉做多、死叉做空、无加仓）。通过自定义命令方式运行，**不修改任何现有文件**，所有数据写入现有 DB 表，与现有系统和原版海龟系统的运行结果进行对比分析。

> 设计目标：在同一数据库内，用不同账户分别运行现有海龟系统、原版海龟系统、双均线交叉系统，通过账户筛选在 Dashboard 和报表中对比三者的绩效差异。

---

## 与原系统的关键差异

| 方面 | 现有系统（海龟变异） | 双均线对照系统 |
|------|---------------------|---------------|
| 入场信号 | 收盘价突破唐奇安通道(20日高低点) | 收盘 MA10 上穿/下穿 MA20 |
| 出场信号 | ATR 移动止损（最高收盘价-2×ATR） | **MA10 反向穿越 MA20**（无独立止损） |
| 执行模式 | 收盘生成信号→下一开盘执行 | 同左（T+1 模式） |
| 仓位计算 | 固定 `risk_per_unit=4000` | 动态 `equity × 2% / (2 × ATR × 合约乘数)` |
| 最大单位数 | 3（金字塔加仓） | **1（无加仓）** |
| ATR 使用 | 入场锁定 N 值全程使用 | 每次开仓**重新计算** |
| 加仓触发 | 0.5/1.0×N 从上次加仓价 | **无加仓** |
| 风险比例 | 每单位约 1% 权益 | **每笔 2% 权益**（单次入场） |
| 持仓 unit 数 | 0-3 浮动 | 固定 0 或 1 |
| 硬止损 | 入场价 - 2×N | 无（依赖交叉反向出场） |

## 命令接口

双均线系统通过 `--account-name` 参数指定交易账户进行对比测试。

```bash
# 全模式：检测 MA 交叉 + 立即执行（推荐）
python manage.py run_ma_crossover --mode full --account-name ma_510989

# 信号模式：仅检测交叉并写入 PENDING 信号（收盘后运行）
python manage.py run_ma_crossover --mode signal --account-name ma_510989

# 执行模式：执行待处理信号（开盘运行）
python manage.py run_ma_crossover --mode execute --account-name ma_510989

# 强制平仓：平掉该账户所有持仓
python manage.py run_ma_crossover --mode exit_all --account-name ma_510989
```

## 核心逻辑

双均线是**收盘交叉判断 → 下一开盘执行**的 T+1 模式，不同于原版海龟的盘中实时触发。

### 整体流程

```
┌─ 运行命令（推荐 --mode full）──────────────────────────┐
│                                                         │
│  1. 连接 TqApi                                          │
│  2. 获取 TradingAccount + AccountContractConfig          │
│  3. 遍历每个活跃品种                                     │
│     │                                                    │
│     ├─ 获取日 K 线数据（TqSDK 实时接口, 30根）            │
│     ├─ 计算 MA10 和 MA20                                 │
│     │   └─ ma10_curr = mean(closes[-11:-1])              │
│     │   └─ ma20_curr = mean(closes[-21:-1])              │
│     │   └─ ma10_prev = mean(closes[-12:-2])              │
│     │   └─ ma20_prev = mean(closes[-22:-2])              │
│     ├─ 判断交叉方向                                       │
│     │   ├─ ma10_prev≤ma20_prev 且 ma10_curr>ma20_curr    │
│     │   │   └─ 金叉 → direction=1（做多）                │
│     │   ├─ ma10_prev≥ma20_prev 且 ma10_curr<ma20_curr    │
│     │   │   └─ 死叉 → direction=-1（做空）               │
│     │   └─ 其他 → 无信号，跳过                           │
│     │                                                    │
│     ├─ 检查 PositionState 现有持仓                        │
│     │   ├─ 无持仓 → 直接开新仓                           │
│     │   ├─ 同方向 → 不变（跳过）                         │
│     │   └─ 反方向 → 先平仓 → 再开新仓                    │
│     │                                                    │
│     ├─ 计算仓位大小                                       │
│     │   └─ lots = floor(equity × 2% / (2 × ATR × 乘数))  │
│     │                                                    │
│     ├─ 执行 TargetPosTask                                 │
│     ├─ 写入 DailyStrategySignal + PositionState           │
│     └─ 记录 TradeLog                                     │
│                                                         │
│  4. 移仓检查（检测主力合约变更）                           │
│  5. 关闭 TqApi 连接                                      │
└─────────────────────────────────────────────────────────┘
```

### 仓位计算公式

```
每笔风险金额 = 当前权益 × 2%
每手风险价值 = 2 × ATR(20) × 合约乘数
开仓手数     = floor(每笔风险金额 / 每手风险价值)
最终手数     = max(开仓手数, 1)
```

**示例**（螺纹钢 rb，权益 100 万，ATR=45，乘数=10）：
```
每笔风险金额 = 1,000,000 × 2% = 20,000
每手风险价值 = 2 × 45 × 10 = 900
开仓手数     = floor(20,000 / 900) = 22 手
```

### 信号 Remark 编码

由于 `DailyStrategySignal` 没有 MA 专用字段，MA 值存储在 `remark` 字段：

```
MA10=3812.50, MA20=3795.80, Cross=GOLDEN
MA10=3750.00, MA20=3780.50, Cross=DEATH
```

### PositionState.indicators 编码

```json
{
  "ma_10": 3812.50,
  "ma_20": 3795.80,
  "entry_atr": 45.20
}
```

## Alpha 因子分析

### 策略定位

双均线交叉是系统内**最简单的趋势跟踪策略**——仅有两个 MA 周期的交叉作为全部决策依据，无加仓、无独立止损。它在本系统中的定位是**最低复杂度基准**（复杂度为 1），用于与其他策略对比"增加复杂度是否带来 alpha 提升"。

### 因子拆解

#### 因子一：MA10/20 交叉（单一趋势信号）

**公式**：
```
买入信号 (金叉): MA10_prev ≤ MA20_prev AND MA10_curr > MA20_curr
卖出信号 (死叉): MA10_prev ≥ MA20_prev AND MA10_curr < MA20_curr
```

其中 `MA10_curr = mean(close[-11:-1])`（不含当日收盘，避免前视偏差）。

**数理本质**：MA10 与 MA20 的交叉表示短期趋势相对于中期趋势的方向改变。

```
MA10 > MA20 → 短期均值 > 中期均值 → 近期价格在上涨 → 做多
MA10 < MA20 → 短期均值 < 中期均值 → 近期价格在下跌 → 做空
```

**和 Donchian 突破的本质区别**：

| 维度 | MA 交叉 | Donchian 突破 |
|------|--------|--------------|
| 信号性质 | 均值回归式趋势判断 | 极值突破式趋势判断 |
| 反应速度 | 慢（MA 是滞后指标） | 快（突破即触发） |
| 假信号率 | 震荡市中高（反复交叉） | 震荡市中高（反复突破） |
| 参数敏感性 | 极低（10/20 是经典参数） | 低（20 是经典参数） |
| 出场逻辑 | 同入场（反向交叉=出场） | 独立止损逻辑 |

**关键局限**：MA 交叉的滞后性意味着入场价比最优入场点通常晚 3-7 个交易日。

#### 因子二：反向交叉出场（无独立止损）

```
出场条件：与入场完全对称
  多头：MA10 下穿 MA20 → 平仓
  空头：MA10 上穿 MA20 → 平仓
```

这是本策略**最显著的风险特征**——没有独立的止损逻辑。

**代数含义**：
```
单笔最大亏损无理论上限。
如果价格快速反向运动但 MA10 尚未穿越 MA20，亏损持续扩大。
```

| 维度 | 有独立止损（海龟/V2） | 无独立止损（双均线） |
|------|---------------------|-------------------|
| 尾部风险 | ✅ 可控（止损限制最大亏损） | ❌ 不可控 |
| 趋势启动后小幅回调 | 可能被止损震出 | ✅ 穿越后才出场，不会被震出 |
| 连续亏损容忍度 | 高（每笔亏损有限） | 低（每笔亏损可能很大） |
| 资金曲线平滑度 | 较高 | 较低 |

#### 因子三：权益比例头寸规模

**公式**：
```
lots = floor(equity × 2% / (2 × ATR × volume_multiple))
```

- 每笔风险固定为权益的 2%（比原版海龟的 1%/Unit 更激进）
- 无加仓意味着这是一次性下注，全部风险在入场时即确定

### 因子对比：与海龟 V2 的核心差异

| 维度 | 双均线交叉 | 海龟 V2 |
|------|----------|---------|
| Alpha 因子数量 | 1（MA 交叉） | 6（突破 + ATR 头寸 + 动态止损 + 加仓 + 趋势因子 + Gap 保护） |
| 因子多样性 | 极低 | 高 |
| 预期 Sharpe | 低（0.3-0.5） | 中高（0.8-1.2） |
| 最大回撤预期 | 25-40% | 15-25% |
| 独立止损 | ❌ 无 | ✅ ATR 动态止损 |
| 趋势过滤 | ❌ 无 | ✅ 趋势因子 / V3 choppy 过滤 |

### 因子贡献度估算

| 因子 | Alpha 贡献 | 说明 |
|------|-----------|------|
| MA10/20 交叉 | ~80% | 唯一的信号来源 |
| 权益% 头寸 | ~15% | 动态风险调整 |
| 反向交叉出场 | ~5% | 对称入场/出场逻辑（但无风险保护） |

> 双均线交叉的 Alpha 几乎全部来自 MA 交叉信号本身。缺少止损、缺少过滤、缺少加仓，使其风险调整收益显著低于其他策略。作为独立策略的期望 Sharpe 通常低于 0.5。

### 双均线在本系统中的价值

尽管 Alpha 预期较低，双均线交叉在本系统中具有重要的**研究价值**：

1. **复杂度基准**：最简单的策略，作为"增加复杂度是否值得"的对照组
2. **参数敏感性基准**：最少的参数（4 个），用于对比多参数策略是否过拟合
3. **交易频率基准**：最低的交易频率（年 20-50 次），对比高频策略的边际成本
4. **Alpha 下限基准**：如果双均线不赚钱，说明市场环境不适合趋势跟踪

## 验证步骤

1. **确认账户配置**
   - 创建双均线对照账户（如 `ma_510989`）
   - 确认已配置 `AccountContractConfig` 交易品种
   - 确认已配置 `StrategyConfig`

2. **信号模式测试**
   ```bash
   python manage.py run_ma_crossover --mode signal --account-name ma_510989
   ```
   → 检查 `DailyStrategySignal` 表是否写入 MA 交叉信号
   → 验证 `remark` 字段格式：`MA10=xxx, MA20=xxx, Cross=GOLDEN/DEATH`

3. **全模式测试（模拟盘）**
   ```bash
   python manage.py run_ma_crossover --mode full --account-name ma_510989
   ```
   → 观察日志输出
   → 检查 `PositionState` 表是否更新持仓
   → 检查 `DailyStrategySignal.executed_status` 是否更新为 SUCCESS

4. **平仓测试**
   ```bash
   python manage.py run_ma_crossover --mode exit_all --account-name ma_510989
   ```
   → 检查 `ClosedPositionRecord` 是否记录平仓
   → 检查 `PositionState` 是否重置

5. **对比分析**
   - 通过 Dashboard 切换不同账户对比三者绩效
   - 在信号监控页对比信号类型和频率
   - 在平仓记录页对比盈亏分布
