# 趋势因子（Trend Factor）设计文档（2026-05-09 更新）

> 文件位置：`backend/stock/core/indicators.py`  
> 配置位置：`backend/stock/core/config_loader.py`（原 `parameter_config.py`，已删除）

---

## 一、设计目标

| 目标 | 说明 |
|------|------|
| **识别趋势/震荡** | 通过均线排列判断当前市场是否有方向性 |
| **量化趋势强度** | 用均线间距比例衡量趋势的"强劲程度" |
| **动态调整止损** | 强趋势时放大止损（让利润奔跑），弱趋势/震荡时收紧止损（保护本金） |
| **避免跳变** | 连续值设计，防止阈值附近的止损突变 |

---

## 二、核心逻辑（三步）

### Step 1: 均线排列判断

```
MA10 > MA20 > MA40 → 多头排列（有趋势）
MA10 < MA20 < MA40 → 空头排列（有趋势）
其他（交叉/粘合）   → 无趋势（震荡）
```

**为什么是排列？**  
只有三条均线同向有序排列，才说明价格运动有**一致性方向**。如果均线交叉混乱，即使短期波动很大，也视为无方向性的震荡。

### Step 2: 趋势强度计算

```python
gap_10_20 = abs(MA10 - MA20) / abs(MA20)
gap_20_40 = abs(MA20 - MA40) / abs(MA20)
max_gap   = max(gap_10_20, gap_20_40)
```

- 计算两组均线间距（短-中、中-长）
- 以 **MA20** 为基准做归一化（比例形式，跨品种通用）
- 取**较大者**作为趋势强度代表——只要有一组均线在加速发散，就说明趋势在强化

### Step 3: 连续映射到 [0, TREND_FACTOR_MAX]

```python
trend_strength = min(max_gap / TREND_GAP_LIMIT, 1.0)
trend_factor   = round(trend_strength * TREND_FACTOR_MAX, 3)
```

**label 分级（与 factor 同源）**：

```python
if trend_strength >= TREND_LABEL_STRONG_RATIO:   # 0.80
    → strong_bull / strong_bear
elif trend_strength >= TREND_LABEL_WEAK_RATIO:   # 0.30
    → weak_bull / weak_bear
else:
    → choppy
```

> `trend_label` 完全基于 `trend_strength` 的比例，不需要额外的绝对阈值。`trend_factor` 和 `trend_label` 同源，不会出现"label 说强趋势但 factor 很小"的矛盾。

**这一步的本质是"归一化"**：

假设考试满分 150 分（`TREND_GAP_LIMIT = 3%`）：

| 实际得分 (max_gap) | 计算 | 百分比强度 |
|-------------------|------|-----------|
| 30 分 (0.3%) | 30 / 150 = **0.2** | 20% |
| 75 分 (0.75%) | 75 / 150 = **0.5** | 50% |
| 150 分 (1.5%) | 150 / 150 = **1.0** | 100% |
| 300 分 (3.0%) | 300 / 150 = 2.0 → min(..., 1.0) = **1.0** | 100%（封顶） |

**为什么封顶？**  
考 200 分还是 150 分，在"满分 150"的体系里都是满分。间距 3% 和 1.5% 都视为"完全趋势"，`trend_factor` 不再继续增加，防止极端行情下止损过大。

## 三、止损倍数公式

当前代码中，止损倍数直接内嵌在 `tasks_daily_close.py` 的止损计算逻辑中：

```python
# 多头止损价 = 最高价 - 2(1+factor) × ATR
# 空头止损价 = 最低价 + 2(1+factor) × ATR
```

**止损倍数 = 2 × (1 + factor)**，范围 **[2.0, 3.0]**

| factor | 公式 | 止损倍数 |
|--------|------|----------|
| 0.0（震荡） | `2 × (1 + 0) = 2` | **2.0 ATR** |
| 0.25（中等） | `2 × (1 + 0.25) = 2.5` | **2.5 ATR** |
| 0.5（极强） | `2 × (1 + 0.5) = 3` | **3.0 ATR** |

## 四、唐奇安通道突破信号

```python
def check_breakout_signal(klines, entry_period=20):
    # 取前 entry_period 根 K 线（不含最新一根）
    high_prior = klines["high"].iloc[-entry_period - 1:-1]
    low_prior = klines["low"].iloc[-entry_period - 1:-1]
    
    entry_high = float(high_prior.max())
    entry_low = float(low_prior.min())
    latest_close = float(klines.iloc[-1]['close'])
    
    if latest_close > entry_high:
        → 向上突破，signal_direction = 1
    elif latest_close < entry_low:
        → 向下突破，signal_direction = -1
```

> 关键：突破判断使用**前 20 日的高低点，不包含当日 K 线**，避免当日收盘即突破当日高点的自指问题。

## 五、关键设计决策

| 决策 | 理由 |
|------|------|
| **均线排列作为前提** | 只有均线同向排列，价格才有明确方向性。交叉/混乱时无论间距多大都不视为趋势 |
| **用间距比例而非绝对值** | 不同品种价格差异大（螺纹 3500 vs 股指 3500），比例才能跨品种通用 |
| **取 `max(gap_10_20, gap_20_40)`** | 只要有一组均线发散足够大，就说明趋势在加速，取大者更敏感 |
| **`TREND_GAP_LIMIT = 0.03` 封顶** | 防止极端行情下因子过大导致止损过宽。3% 的均线间距已属于非常强趋势 |
| **连续值 `[0, 0.5]`** | 避免阶梯阈值导致的跳变问题。比如间距从 0.99% 到 1.01%，阶梯设计会让止损从 2.0 ATR 跳到 3.0 ATR，连续设计只会从 2.64 ATR 微涨到 2.68 ATR |
| **label 与 factor 同源** | 都用 `trend_strength` 判定，不需要额外的绝对阈值。调整 `TREND_GAP_LIMIT` 时 label 分级自动跟着缩放 |

## 六、配置参数说明

所有控制变量集中在 `backend/stock/core/config_loader.py`（原 `parameter_config.py`，已删除）：

| 变量 | 默认值 | 作用 |
|------|--------|------|
| `TREND_GAP_LIMIT` | 0.03 (3%) | **趋势因子封顶上限**。均线间距达到此值时，`trend_factor` 达到最大值，不再继续增加 |
| `TREND_FACTOR_MAX` | 0.5 | **趋势因子最大值**。决定 `trend_factor` 的上限 |
| `TREND_LABEL_STRONG_RATIO` | 0.80 | **强趋势 label 比例**。`trend_strength` ≥ 80% 时判定为强趋势（对应 `max_gap = 2.4%`） |
| `TREND_LABEL_WEAK_RATIO` | 0.30 | **弱趋势 label 比例**。`trend_strength` ≥ 30% 时判定为弱趋势（对应 `max_gap = 0.9%`） |

### 参数调整指南

| 你想达到的效果 | 调整哪个变量 | 方向 |
|---------------|------------|------|
| 让趋势因子更快达到最大值 | `TREND_GAP_LIMIT` | **调小**（如 0.03 → 0.02） |
| 扩大止损的浮动范围 | 修改代码中止损公式系数 | **调大**（如 2 → 2.5，则范围变为 [2.5, 3.75]） |
| 提高最低止损底线 | 修改代码中止损公式系数 | **调大**（如 2 → 2.5） |
| 更容易被判定为"强趋势" | `TREND_LABEL_STRONG_RATIO` | **调小**（如 0.80 → 0.60） |
| 更容易被判定为"有趋势" | `TREND_LABEL_WEAK_RATIO` | **调小**（如 0.30 → 0.20） |

**等价关系**（当前配置下 `TREND_GAP_LIMIT = 3%`）：

| `trend_strength` | 对应 `max_gap` | label |
|-----------------|----------------|-------|
| ≥ 80% | ≥ 2.4% | strong |
| ≥ 30% | ≥ 0.9% | weak |
| < 30% | < 0.9% | choppy |

> 调整 `TREND_GAP_LIMIT` 时，label 对应的绝对间距会自动跟着缩放，不需要手动同步改两个绝对阈值。

## 七、实盘举例（螺纹钢 rb）

假设当前 **ATR(20) = 35 元**，不同市场状态下的计算：

### 场景 A：强趋势（均线大幅发散）

```
MA10 = 3600, MA20 = 3550, MA40 = 3480
```

**排列判断**：`3600 > 3550 > 3480` ✓ 多头排列

**间距计算**：
```
gap_10_20 = |3600 - 3550| / 3550 = 1.41%
gap_20_40 = |3550 - 3480| / 3550 = 1.97%
max_gap   = 1.97%
```

**趋势因子**：
```
trend_strength = min(1.97% / 3%, 1.0) = 0.657
trend_factor   = 0.657 × 0.5 = 0.328
```

**止损设置**：
```
止损倍数 = 2 × (1 + 0.328) = 2.66 ATR
止损距离 = 2.66 × 35 = 93 元
```

> **解读**：趋势较强，止损适度放大到 2.66 ATR。

### 场景 B：极强趋势（均线极度发散）

```
MA10 = 3700, MA20 = 3550, MA40 = 3400
```

**间距计算**：
```
gap_10_20 = 4.23%
gap_20_40 = 4.23%
max_gap   = 4.23%
```

**趋势因子**：
```
trend_strength = min(4.23% / 3%, 1.0) = 1.0
trend_factor   = 1.0 × 0.5 = 0.5
```

**止损设置**：
```
止损倍数 = 2 × (1 + 0.5) = 3.0 ATR
止损距离 = 3.0 × 35 = 105 元
```

> **解读**：趋势极强，给足波动空间，避免被正常回调洗出。

### 场景 C：弱趋势（均线刚刚发散）

```
MA10 = 3555, MA20 = 3550, MA40 = 3540
```

**排列判断**：`3555 > 3550 > 3540` ✓ 多头排列

**间距计算**：
```
gap_10_20 = 0.14%
gap_20_40 = 0.28%
max_gap   = 0.28%
```

**趋势因子**：
```
trend_strength = 0.28% / 3% = 0.093
trend_factor   = 0.093 × 0.5 = 0.047
```

**止损设置**：
```
止损倍数 = 2 × (1 + 0.047) = 2.09 ATR
止损距离 = 2.09 × 35 ≈ 73 元
```

> **解读**：趋势很弱，止损基本维持基准水平（2.09 ATR），不盲目放大。

### 场景 D：震荡（均线交叉/粘合）

```
MA10 = 3550, MA20 = 3552, MA40 = 3548
```

**排列判断**：
- `3550 > 3552`？否
- `3550 < 3552 < 3548`？否
- → **非多头排列，非空头排列**

**趋势因子**：
```
trend_factor = 0.0
```

**止损设置**：
```
止损倍数 = 2.0 ATR
止损距离 = 2.0 × 35 = 70 元
```

> **解读**：均线混乱，视为无趋势，止损收紧到基准 2.0 ATR。

## 八、连续 vs 阶梯 对比

假设螺纹 MA20=3550，均线多头排列，观察间距从 0.2% 增长到 3.0% 时止损倍数的变化：

| max_gap | **连续设计（当前）** | 阶梯设计（旧版） |
|---------|-------------------|----------------|
| 0.2%    | 2.07 ATR          | 2.00 ATR       |
| 0.5%    | 2.17 ATR          | 2.00 ATR       |
| 0.9%    | 2.30 ATR          | 2.00 ATR       |
| **1.0%**| **2.33 ATR** ⬆️ 微涨 | **3.00 ATR** ⬆️ 跳涨 |
| 1.5%    | 2.50 ATR          | 3.00 ATR       |
| 2.0%    | 2.67 ATR          | 3.00 ATR       |
| 3.0%    | 3.00 ATR          | 3.00 ATR       |
| 4.0%    | 3.00 ATR（封顶）   | 3.00 ATR       |

**阶梯设计的痛点**：间距从 0.99% → 1.01%，止损突然从 2.0 ATR 跳到 3.0 ATR，多放大了 50%。这 0.02% 的差距可能是随机噪声导致的，不应该引发止损策略的剧烈变化。

**连续设计的优势**：平滑过渡，每增加一点间距就对应多一点止损空间，没有突变点。

## 十、趋势标签的新用途：开仓过滤

### 10.1 背景

最初 `trend_label` 的设计目标仅用于**动态调整止损距离** — 强趋势时放大止损让利润奔跑，震荡时收紧止损保护本金。入场本身**只看唐奇安通道突破**，不参考趋势标签。

### 10.2 新增用途

`skip_choppy_entry` 配置项引入后，`trend_label` 有了第二个用途：**可选用作开仓过滤器**。

当账户策略配置 `skip_choppy_entry = True` 时，`execute_entry_order` 会检查信号的 `trend_label`：

```
trend_label ∈ {choppy, neutral}  → 取消开仓（CANCELLED）
trend_label ∈ {strong_bull, weak_bull, strong_bear, weak_bear} → 正常开仓
```

### 10.3 设计要点

| 要点 | 说明 |
|------|------|
| **可选启用** | 默认关闭（`skip_choppy_entry = False`），完全不影响现有逻辑 |
| **仅限开仓** | 不影响止损、加仓、移仓等其他信号类型 |
| **A/B 验证** | 多账户场景下，可让 A 账户正常开仓、B 账户跳过震荡，对比绩效 |
| **无需重启** | 配置保存在数据库，修改后下次开盘任务自动生效 |

### 10.4 与止损用途的关系

```
trend_label 的第一职责（止损调整 — 始终生效）：
   strong_bear/strong_bull → factor 较大 → 止损放宽（3.0 ATR）
   choppy                  → factor = 0   → 止损收紧（2.0 ATR）

trend_label 的第二职责（开仓过滤 — 按需启用）：
   skip_choppy_entry = False → trend_label 不影响开仓（默认行为）
   skip_choppy_entry = True  → choppy/neutral 的开仓被取消
```

两个用途互不干扰，底层依赖同一套 `trend_strength` 计算逻辑。
