# 趋势因子（Trend Factor）设计文档（2026-05-26 更新 · 实盘启动）

> 涉及文件: `backend/stock/core/indicators.py`, `backend/stock/backtest/indicators.py`
> 配置: `backend/stock/core/config_loader.py`, `backend/stock/backtest/config.py`
> DB 字段: `StrategyConfig.gap_atr_limit`

---

## 一、设计目标

| 目标 | 说明 |
|------|------|
| **识别趋势/震荡** | 通过均线排列判断当前市场是否有方向性 |
| **量化趋势强度** | 用均线间距（以 ATR 为单位）衡量趋势的"强劲程度" |
| **动态调整止损** | 强趋势时放大止损（让利润奔跑），弱趋势/震荡时收紧止损（保护本金） |
| **跨品种可比** | ATR 归一化后，不同价格水平的品种趋势强度可直接比较 |
| **避免跳变** | 连续值设计，防止阈值附近的止损突变 |

---

## 二、核心逻辑（三步）

### Step 1: 均线排列判断

```
MA10 > MA20 > MA40 → 多头排列（有趋势）
MA10 < MA20 < MA40 → 空头排列（有趋势）
其他（交叉/粘合）   → 无趋势（震荡）
```

**为什么是排列？**  
只有三条均线同向有序排列，才说明价格运动有**一致性方向**。如果均线交叉混乱，即使短期波动很大，也视为无方向性的震荡。

### Step 2: 趋势强度计算（ATR 归一化）

```python
# 以 ATR 为单位衡量均线乖离
gap_10_20 = abs(MA10 - MA20) / atr_20
gap_20_40 = abs(MA20 - MA40) / atr_20
max_gap   = max(gap_10_20, gap_20_40)
```

- 计算两组均线间距（短-中、中-长），以 **ATR 为单位**
- 取**较大者**作为趋势强度代表 — 只要有一组均线在加速发散，就说明趋势在强化
- ATR 已包含波动率信息，不同品种、不同价格水平下的乖离程度可直接比较

### Step 3: 连续映射到 [0, TREND_FACTOR_MAX]

```python
trend_strength = min(max_gap / GAP_ATR_LIMIT, 1.0)
trend_factor   = round(trength_strength * TREND_FACTOR_MAX, 3)
```

**label 分级（与 factor 同源）：**

```python
if trend_strength >= TREND_LABEL_STRONG_RATIO:   # 0.80
    → strong_bull / strong_bear
elif trend_strength >= TREND_LABEL_WEAK_RATIO:   # 0.30
    → weak_bull / weak_bear
else:
    → choppy
```

> `trend_label` 完全基于 `trend_strength` 的比例，不需要额外的绝对阈值。`trend_factor` 和 `trend_label` 同源，不会出现"label 说强趋势但 factor 很小"的矛盾。

**GAP_ATR_LIMIT = 2.0 的物理含义：**  
均线乖离达到 2 倍 ATR 时，`trend_strength = 1.0`（满趋势强度）。换句话说，MA10 与 MA20 的差距达到 ATR 的 2 倍，就被视为"足够强的趋势"。

| 实际乖离 (max_gap) | 计算 | trend_strength |
|--------------------|------|---------------|
| 0.5 ATR | 0.5 / 2.0 = **0.25** | 25% |
| 1.0 ATR | 1.0 / 2.0 = **0.50** | 50% |
| 1.5 ATR | 1.5 / 2.0 = **0.75** | 75% |
| 2.0 ATR | 2.0 / 2.0 = **1.00** | 100%（封顶） |
| 3.0 ATR | 3.0 / 2.0 = 1.5 → min(..., 1.0) = **1.00** | 100%（封顶） |

## 三、止损倍数公式

止损计算在 `scheduler/tasks_daily_close.py` 中：

```python
# 多头止损价 = 最高价 - 2(1+factor) × ATR
# 空头止损价 = 最低价 + 2(1+factor) × ATR
```

**止损倍数 = 2 × (1 + factor)**，范围 **[2.0, 3.0]**

| factor | 公式 | 止损倍数 |
|--------|------|----------|
| 0.0（震荡/choppy） | `2 × (1 + 0) = 2` | **2.0 ATR** |
| 0.25（中等趋势）   | `2 × (1 + 0.25) = 2.5` | **2.5 ATR** |
| 0.5（强趋势封顶）  | `2 × (1 + 0.5) = 3` | **3.0 ATR** |

## 四、三种计算方案对比

趋势因子的计算经历了三个阶段的演进：

### 方案 A（V2 原版 — 已弃用）：价格百分比归一化

```python
gap_10_20 = abs(MA10 - MA20) / abs(MA20)   # 间距占 MA20 百分比
gap_20_40 = abs(MA20 - MA40) / abs(MA20)
max_gap = max(gap_10_20, gap_20_40)
trend_strength = min(max_gap / 0.03, 1.0)   # TREND_GAP_LIMIT=3%
```

| 特性 | 说明 |
|------|------|
| 归一化基准 | MA20 价格（百分比） |
| 封顶阈值 | 3% 均线间距 |
| 品种可比性 | ❌ 不可比 — 高价品种（沪深300 4000+）的百分比天生小，低价品种（甲醇 1800）的百分比天生大 |
| 价格水平依赖 | ❌ 依赖 — 同一品种在不同价格区间，同样的百分比间距物理含义不同 |

**问题实例：** 沪深300 在 4000 点时 MA10-MA20=40 点（1.0%），甲醇在 1800 点时 MA10-MA20=18 点（同样是 1.0%）。绝对值沪深高了 122%，但百分比一样。而趋势交易的止损是按 ATR（绝对值）计算的，百分比因子与 ATR 止损之间存在维度不匹配。

### 方案 B（ATR 归一化 — 现用）：GAP_ATR_LIMIT=2.0

```python
gap_10_20 = abs(MA10 - MA20) / atr_20    # 间距以 ATR 为单位
gap_20_40 = abs(MA20 - MA40) / atr_20
max_gap = max(gap_10_20, gap_20_40)
trend_strength = min(max_gap / GAP_ATR_LIMIT, 1.0)   # GAP_ATR_LIMIT=2.0
```

| 特性 | 说明 |
|------|------|
| 归一化基准 | ATR（波动率） |
| 封顶阈值 | 2 倍 ATR 乖离 |
| 品种可比性 | ✅ 可比 — 同样 1.5 ATR 乖离，无论螺纹钢还是甲醇，趋势强度相同 |
| 价格水平依赖 | ✅ 不依赖 — ATR 自动适应价格水平变化 |
| 与止损公式匹配 | ✅ 止损公式 `2×(1+factor)×ATR` 也是 ATR 维度，输入输出维度一致 |

### 方案 C（ATR 归一化宽松版 — 备选）：GAP_ATR_LIMIT=3.0

与方案 B 公式完全相同，仅 `GAP_ATR_LIMIT` 取值不同：

| 参数 | 方案 B（推荐） | 方案 C（备选） |
|------|--------------|--------------|
| GAP_ATR_LIMIT | **2.0** | 3.0 |
| 达到满强度的乖离 | 2 倍 ATR | 3 倍 ATR |
| 饱和率（tf≥0.4） | **~11%**（理想） | ~6%（偏低） |
| 平均 trend_factor | ~0.164 | ~0.124 |

**为什么选 2.0 而不是 3.0？**  
从 22 个品种 3900+ 样本的参数扫描结果看，3.0 的饱和率仅 6% 左右，意味着多数有趋势行情中因子持续偏低，趋势强度无法充分传导到止损距离上。2.0 的饱和率约 11%，在"足够敏感"和"不过度饱和"之间取得平衡。

## 五、前后效果对比（V2 百分比 vs ATR 归一化）

### 5.1 跨品种均匀性（核心改进）

使用账户 510976 的 22 个活跃品种，3900+ 根日K线样本进行对比：

| 指标 | V2 百分比法 (TREND_GAP_LIMIT=3%) | ATR 归一化 (GAP_ATR_LIMIT=2.0) | 改进 |
|------|-------------------------------|-------------------------------|------|
| **CV（品种均值变异系数）** | 0.317 | 0.190 | **↓ 40%** |
| 品种均值极差 | 0.307 | 0.214 | ↓ 30% |
| 品种均值标准差 | 0.076 | 0.046 | ↓ 40% |

**CV 从 0.317→0.190 的含义：** 品种间的 trend_factor 差异减少了 40%。也就是说，同样强度的趋势行情在不同品种上得到的因子值更加一致了。

### 5.2 分布对比

| Bucket | V2 百分比法 | ATR 归一化 (2.0) | 解读 |
|--------|-----------|------------------|------|
| =0（choppy） | 34.4% | 34.4% | 均线排列条件不变，一致 |
| 0~0.1 | 21.5% | 11.8% | 百分比法把许多弱排列也算作有趋势 |
| 0.2~0.3 | 10.0% | 15.7% | ATR 法在中段更集中 |
| 0.4~0.5（饱和） | 5.5% | 10.4% | ATR 法强趋势信号更多 |

### 5.3 具体品种差异

百分比法下，部分"高价品种"（如 lh, lc, ru）的 trend_factor 系统性偏低，因为 MA 间距占 MA20 的百分比小。ATR 归一化后这些品种的 trend_factor 回到了正常水平。

| 品种 | V2 百分比均值 | ATR 归一化均值 | 变化 |
|------|-------------|---------------|------|
| rb（螺纹钢 3500） | 0.152 | 0.175 | ↑ 15% |
| lc（生猪） | 0.088 | 0.142 | ↑ 61%（高价低百分比问题修正） |
| MA（甲醇 1800） | 0.158 | 0.171 | ↑ 8% |
| ru（橡胶） | 0.105 | 0.153 | ↑ 46%（高价低百分比问题修正） |
| TA（PTA） | 0.162 | 0.164 | ≈ 持平 |

> 高价品种（lc, ru 等）在百分比法下因子系统性偏低，导致这些品种的止损距离长期偏小，容易在正常波动中被洗出。ATR 归一化后修正了这一问题。

### 5.4 对止损的实际影响

| GAP_ATR_LIMIT | 平均止损距离 | 相对 baseline 变化 |
|--------------|------------|------------------|
| 1.5 | 76.2 点 | -13% |
| **2.0（当前）** | **87.3 点** | **baseline** |
| 2.5 | 94.1 点 | +8% |
| 3.0 | 98.4 点 | +13% |

> GAP_ATR_LIMIT 每增加 0.5，平均止损距离扩大约 6-8 点。当前 2.0 在止损收紧度和趋势保护之间取得平衡。

## 六、ATR 不可用时的处理

```python
if not (atr and atr > 0):
    return 0.0, 'choppy'   # ATR 不可用时直接返回 choppy
```

**不再降级为百分比法。** 理由：
- ATR 不可用通常发生在数据初始化阶段（K 线数据不足 20 根）
- 此时行情本身也不稳定，使用保守的 choppy（止损 2.0 ATR）是合理选择
- 百分比法与 ATR 止损公式维度不匹配，混合使用会导致止损逻辑不一致

## 七、连续 vs 阶梯 对比

假设品种 ATR=50，均线多头排列，观察趋势因子对止损的影响：

| trend_strength | **连续设计（当前）** | 阶梯设计（旧版） |
|----------------|-------------------|----------------|
| 0.0（choppy）  | 2.0 ATR（100 元）| 2.0 ATR |
| 0.2（弱趋势）  | 2.2 ATR（110 元）| 2.0 ATR |
| 0.4（弱趋势）  | 2.4 ATR（120 元）| 2.0 ATR |
| **0.3→0.31**  | **2.31 ATR** ⬆️ 微涨 | **→ 3.0 ATR** ⬆️ 跳涨 50% |
| 0.6（强趋势）  | 2.6 ATR（130 元）| 3.0 ATR |
| 0.8（强趋势）  | 2.8 ATR（140 元）| 3.0 ATR |
| 1.0（封顶）    | 3.0 ATR（150 元）| 3.0 ATR |

**阶梯设计的痛点：** trend_strength 从 0.29→0.31（ATR 乖离从 0.58→0.62），止损突然从 2.0 ATR 跳到 3.0 ATR，多放大了 50%。这 4% 的乖离变化可能是随机噪声导致的，不应该引发止损策略的剧烈变化。

**连续设计的优势：** 平滑过渡，每增加一点乖离就对应多一点止损空间，没有突变点。

## 八、配置参数说明

所有控制变量集中在 `backend/stock/core/config_loader.py`：

| 变量 | 默认值 | 作用 |
|------|--------|------|
| `GAP_ATR_LIMIT` | 2.0 | **趋势因子封顶上限（ATR 倍数）**。MA 乖离达到此 ATR 倍数时 `trend_factor` 达到最大值 |
| `TREND_FACTOR_MAX` | 0.5 | **趋势因子最大值**。决定 `trend_factor` 的上限 |
| `TREND_LABEL_STRONG_RATIO` | 0.80 | **强趋势 label 比例**。`trend_strength` ≥ 80% 时判定为强趋势 |
| `TREND_LABEL_WEAK_RATIO` | 0.30 | **弱趋势 label 比例**。`trend_strength` ≥ 30% 时判定为弱趋势 |

**等价关系**（当前配置 `GAP_ATR_LIMIT = 2.0`）：

| trend_strength | 对应 max_gap | label | 止损倍数 |
|---------------|--------------|-------|----------|
| ≥ 80% | ≥ 1.6 ATR | strong_bull/bear | > 2.80 ATR |
| ≥ 30% | ≥ 0.6 ATR | weak_bull/bear | > 2.30 ATR |
| < 30% | < 0.6 ATR | choppy | 2.0 ATR |

### 参数调整指南

| 你想达到的效果 | 调整哪个变量 | 方向 |
|---------------|------------|------|
| 让趋势因子更快达到最大值 | `GAP_ATR_LIMIT` | **调小**（如 2.0 → 1.5） |
| 让趋势因子更慢达到最大值 | `GAP_ATR_LIMIT` | **调大**（如 2.0 → 2.5） |
| 扩大止损的浮动范围 | 修改止损公式系数 | **调大**（如 2 → 2.5，则范围变为 [2.5, 3.75]） |
| 更容易被判定为"强趋势" | `TREND_LABEL_STRONG_RATIO` | **调小**（如 0.80 → 0.60） |
| 更容易被判定为"有趋势" | `TREND_LABEL_WEAK_RATIO` | **调小**（如 0.30 → 0.20） |

## 九、实盘举例

假设品种 **ATR(20) = 50 元**，不同市场状态下的计算（基于新 ATR 归一化方案）：

### 场景 A：强趋势（均线大幅发散）

```
MA10 = 3600, MA20 = 3550, MA40 = 3480
ATR = 50
```

**排列判断：** `3600 > 3550 > 3480` ✓ 多头排列

**间距计算：**
```
gap_10_20 = |3600 - 3550| / 50 = 1.0 ATR
gap_20_40 = |3550 - 3480| / 50 = 1.4 ATR
max_gap   = 1.4 ATR
```

**趋势因子：**
```
trend_strength = min(1.4 / 2.0, 1.0) = 0.70
trend_factor   = 0.70 × 0.5 = 0.350
```

**止损设置：**
```
止损倍数 = 2 × (1 + 0.350) = 2.70 ATR
止损距离 = 2.70 × 50 = 135 元
```

> **解读（新方案）：** 趋势较强，止损放大到 2.70 ATR。
> 
> **V2 原版对比：** `gap = 1.97%, gap_limit=3%, strength=0.657, factor=0.328`。新方案因子更高（0.350 vs 0.328），因为 ATR 归一化下 1.4 ATR 的乖离算较强信号，而百分比法下 1.97% 的乖离只到 3% 阈值的 65%。

### 场景 B：极强趋势（均线极度发散）

```
MA10 = 3700, MA20 = 3550, MA40 = 3400
ATR = 50
```

**间距计算：**
```
gap_10_20 = 150 / 50 = 3.0 ATR
gap_20_40 = 150 / 50 = 3.0 ATR
max_gap   = 3.0 ATR
```

**趋势因子：**
```
trend_strength = min(3.0 / 2.0, 1.0) = 1.0
trend_factor   = 1.0 × 0.5 = 0.5
```

**止损设置：**
```
止损倍数 = 2 × (1 + 0.5) = 3.0 ATR
止损距离 = 3.0 × 50 = 150 元
```

> **解读：** 趋势极强，给足波动空间。新旧方案在此场景下一致（均达到上限）。

### 场景 C：弱趋势（均线刚刚发散）

```
MA10 = 3555, MA20 = 3550, MA40 = 3540
ATR = 50
```

**间距计算：**
```
gap_10_20 = 5 / 50 = 0.10 ATR
gap_20_40 = 10 / 50 = 0.20 ATR
max_gap   = 0.20 ATR
```

**趋势因子：**
```
trend_strength = 0.20 / 2.0 = 0.10
trend_factor   = 0.10 × 0.5 = 0.05
```

**止损设置：**
```
止损倍数 = 2 × (1 + 0.05) = 2.10 ATR
止损距离 = 2.10 × 50 = 105 元
```

> **解读：** 趋势很弱，止损基本维持基准水平。
>
> **V2 原版对比：** `gap=0.28%, strength=0.093, factor=0.047`。新方案因子略高（0.050 vs 0.047），差异不大。

### 场景 D：震荡（均线交叉/粘合）

```
MA10 = 3550, MA20 = 3552, MA40 = 3548
```

**排列判断：**
- `3550 > 3552`？否
- `3550 < 3552 < 3548`？否
- → **非多头排列，非空头排列**

**趋势因子：**
```
trend_factor = 0.0
```

**止损设置：**
```
止损倍数 = 2.0 ATR
止损距离 = 2.0 × 50 = 100 元
```

> **解读：** 均线混乱，视为无趋势，止损收紧到基准 2.0 ATR（新旧方案一致，排列条件不变）。

## 十、完整计算链路图

```
┌─────────────────┐
│  K线数据(MA10,   │
│  MA20, MA40,    │
│  ATR_20)        │
└────────┬────────┘
         ▼
┌─────────────────────────────┐
│ Step 1: 均线排列判断          │
│ 多头排列? 空头排列? 其他?      │
└────────┬────────────────────┘
         ▼
    ┌────────┐
    │  其他   │ ──→ trend_factor = 0, label = "choppy"
    └────┬───┘
         ▼
┌──────────────────────────────────┐
│ Step 2: ATR 归一化间距计算        │
│ gap_10_20 = |MA10-MA20| / ATR   │
│ gap_20_40 = |MA20-MA40| / ATR   │
│ max_gap   = max(...)             │
└────────┬─────────────────────────┘
         ▼
┌─────────────────────────────┐
│ Step 3: 归一化映射           │
│ trend_strength = min(        │
│   max_gap / GAP_ATR_LIMIT,  │
│   1.0)                      │
│ trend_factor = round(        │
│   trend_strength *          │
│   TREND_FACTOR_MAX, 3)      │
└────────┬────────────────────┘
         ▼
┌─────────────────────────────┐
│ Step 3.5: label 分级         │
│ 基于 trend_strength 比例      │
│   ≥ STRONG_RATIO → strong    │
│   ≥ WEAK_RATIO   → weak      │
│   else           → choppy    │
└────────┬────────────────────┘
         ▼
┌─────────────────────────────┐
│ Step 4: 计算止损倍数          │
│ stop_mult = 2 × (1 + factor) │
│ stop_price = based on        │
│   HH/LL and stop_mult × ATR  │
└─────────────────────────────┘
```

## 十一、演进历史

| 时间 | 版本 | 计算方式 | 封顶参数 | 状态 |
|------|------|---------|---------|------|
| 2026-04 前 | V2 原始 | `\|MA10-MA20\| / \|MA20\|` | `TREND_GAP_LIMIT=0.03` | ❌ 已弃用 |
| 2026-05 初 | 过渡版 | `\|MA10-MA20\| / ATR` | `GAP_ATR_LIMIT=3.0` | ❌ 已弃用 |
| 2026-05-17 | **当前** | `\|MA10-MA20\| / ATR` | **`GAP_ATR_LIMIT=2.0`** | ✅ **现用** |

**关键变更点：**

1. **归一化基准变更：** MA20 价格百分比 → ATR。解决了高价品种趋势因子系统性偏低的问题。
2. **参数改名：** `TREND_GAP_LIMIT` → `GAP_ATR_LIMIT`，DB 字段同时迁移。
3. **默认值调整：** 从 `0.03`（3% 百分比）→ 经过参数扫描（1.5~3.0 区间）确定 `2.0`（2 倍 ATR）。
4. **删除降级逻辑：** ATR 不可用时直接返回 `(0.0, 'choppy')`，不再降级为百分比法。
