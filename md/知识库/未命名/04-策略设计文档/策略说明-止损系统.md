## 一、整体架构

止损系统采用**动态跟踪止损（Trailing Stop）**设计，核心思想：

- **强趋势** → 止损放宽（给足回调空间，让利润奔跑）
- **弱趋势/震荡** → 止损收紧（保护本金）
- **保本机制** → 盈利超过阈值后，确保不亏本金

## 二、止损计算纯函数（core/stop_loss.py）

`core/stop_loss.py` 提供了三个纯函数，封装止损系统的核心计算逻辑。**注意**：当前线上系统（`tasks_daily_close.py`）内联实现了这些计算逻辑，而 `core/stop_loss.py` 的纯函数被回测模块（`backtest/strategy.py`）镜像使用。两者公式一致，但实现细节有微小差异（详见表）。

### 2.1 compute_stop_loss — 动态止损价计算

```python
def compute_stop_loss(
    direction: int,
    highest_close: Optional[Decimal],
    lowest_close: Optional[Decimal],
    atr_value: Decimal,
    factor: Decimal,
    tick: Decimal,
) -> Optional[Decimal]:
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `direction` | int | 1=多头, -1=空头 |
| `highest_close` | Decimal | 持仓期间最高收盘价（多头） |
| `lowest_close` | Decimal | 持仓期间最低收盘价（空头） |
| `atr_value` | Decimal | 20日 ATR |
| `factor` | Decimal | 趋势因子 [0, 0.5] |
| `tick` | Decimal | 品种最小变动价位（如螺纹钢 1） |

**公式**：

```
stop_distance = 2 × (1 + factor) × atr_value

多头：raw_stop = highest_close - stop_distance
     止损  = floor(raw_stop / tick) × tick      ← 向下取整到 tick

空头：raw_stop = lowest_close + stop_distance
     止损  = ceil(raw_stop / tick) × tick       ← 向上取整到 tick
```

**tick 取整意义**：确保止损价是合约可交易的有效价格。向下取整（多头）让止损更宽松一些，避免因取整问题导致止损过紧被毛刺触发。

### 2.2 check_protect_cost_condition — 保本启用检查

```python
def check_protect_cost_condition(
    direction: int,
    latest_close_price: Optional[Decimal],
    cost_price: Optional[Decimal],
    atr_value: Decimal,
    protect_cost_enabled: bool,
    protect_cost_ratio: float,
) -> tuple:
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `protect_cost_enabled` | bool | 当前是否已启用保本（存储状态） |
| `protect_cost_ratio` | float | 保本启用阈值倍数，默认 2.5 |

**逻辑**：

```
仅当 protect_cost_enabled == False（未启用）时检查：
  if direction == 1:
      profit_diff = latest_close_price - cost_price
      if profit_diff > protect_cost_ratio × atr_value:
          protect_cost_enabled = True

  if direction == -1:
      profit_diff = cost_price - latest_close_price
      if profit_diff > protect_cost_ratio × atr_value:
          protect_cost_enabled = True
```

**返回值**：`(protect_cost_enabled, protect_price)`

- `protect_enabled`：是否启用保本（单向开关，0→1 不回溯）
- `protect_price`：保本价（启用后有意义）= **直接取 cost_price**（该纯函数不带 tick 偏移）

> **注意**：线上 `tasks_daily_close.py` 中，保本价比此纯函数多一个 `± tick × 2` 偏移（见第三章）。回测模块同样使用 `cost_price ± tick × 2` 的偏移版本。`core/stop_loss.py` 的纯函数是基础逻辑，偏移量由调用方自行计算。

### 2.3 apply_protect_price — 保本兜底

```python
def apply_protect_price(dynamic_stop_loss: Decimal, protect_price: Optional[Decimal], direction: int) -> Decimal:
```

**逻辑**：

```
if protect_price is None:
    return dynamic_stop_loss

if direction == 1 and dynamic_stop_loss < protect_price:
    return protect_price        # 多头止损不能低于保本价
elif direction == -1 and dynamic_stop_loss > protect_price:
    return protect_price        # 空头止损不能高于保本价

return dynamic_stop_loss        # 动态止损更优，继续用动态止损
```

### 3.2 更新最高/最低收盘价（Step 6）

```python
def update_all_positions_high_low_price(account):
    多头持仓: highest_close = max(highest_close, latest_close_price)
    空头持仓: lowest_close  = min(lowest_close,  latest_close_price)
```

> 这是跟踪止损的**基准价**，持仓后每天更新，只朝有利方向移动。
> 多头关注 `highest_close`（只上移），空头关注 `lowest_close`（只下移）。
> 两个方向同时记录，`highest_close` 给多头用，`lowest_close` 给空头用。

### 3.3 更新止损价格（Step 7）—— 核心函数

```python
def update_all_positions_stop_loss_price(api, account):
    # 1. 订阅所有持仓的 TqSDK Position
    tq_positions[position.symbol] = api.get_position(position.symbol)
    api.wait_update(deadline=time.time() + 5)

    for position in 所有有持仓:
        # 2. 从 TqSDK Position 读取成本价（open_price_long / open_price_short）
        cost_price = tq_pos.open_price_long/short (优先) 或 DB cost_price (回退)

        # 3. 读取计算所需数据
        atr_value = position.indicators['atr_20']
        factor    = position.indicators['trend_factor']
        tick      = FullContractList.objects.get(symbol=position.symbol).price_tick

        if 多头:
            dynamic_stop_loss = highest_close - 2 × (1 + factor) × atr_value
            # 注：此处不做 tick 取整（与 core/stop_loss.py 的纯函数不同）
        if 空头:
            dynamic_stop_loss = lowest_close + 2 × (1 + factor) × atr_value
            # 注：此处不做 tick 取整

        # 4. 保本功能检查（见 3.5）
        ...

        # 5. 更新 DB
        PositionState.objects.filter(id=position.id).update(
            stop_loss_price=dynamic_stop_loss,
            trend_info=f'{atr_value}, {factor}, {trend_label}',
            protect_cost_enabled=protect_cost_enabled,
        )
```

**与 core/stop_loss.py 纯函数的差异**：

| 方面 | tasks_daily_close.py（线上） | core/stop_loss.py（纯函数） |
|------|--------------------------|---------------------------|
| tick 取整 | **不做**取整 | 做 floor/ceil 取整 |
| 成本价来源 | TqSDK Position (wait_update 刷新) | 参数传入 |
| 保本价偏移 | `cost_price ± tick × 2` | `= cost_price`（无偏移） |
| 保本启用条件 | 盈利 > 2.5×ATR **且 units>=3** | 仅盈利 > ratio×ATR |

> 这些差异是历史形成的。`core/stop_loss.py` 是后来提取的纯函数版本，用于回测模块。线上系统保持内联实现以确保稳定性。

### 3.4 止损公式推导

| factor | 公式 | 止损倍数 |
|--------|------|----------|
| 0.0（震荡） | `2 × (1 + 0) = 2` | **2.0 ATR** |
| 0.25（中等） | `2 × (1 + 0.25) = 2.5` | **2.5 ATR** |
| 0.5（极强） | `2 × (1 + 0.5) = 3` | **3.0 ATR** |

**止损距离**：
- 多头：`最高价 - 止损价 = 2(1+factor) × ATR`
- 空头：`止损价 - 最低价 = 2(1+factor) × ATR`

### 3.5 保本机制（2026-05-13 更新）

#### 启用条件（两条同时满足）

```
条件一：盈利 > PROTECT_COST_ENABLED_RATIO × ATR   # 默认 2.5 × ATR
条件二：position.units >= 3                        # 持仓达到 3 个单位
```

> **条件二的用意**：保本机制只在持仓达到 3 个单位（满仓）时才启用。1 单位或 2 单位时即使有浮盈，也让止损继续动态跟踪，给予行情更多回调空间。这是为了防止轻仓时因保本兜底而过早被震荡出局。

#### 保本价计算

```python
# 从 FullContractList 获取品种最小变动价位 tick
tick = FullContractList.objects.filter(symbol=position.symbol).first().price_tick

多头保本价 = 成本价 + tick × 2   # 比成本价高 2 个最小跳动
空头保本价 = 成本价 - tick × 2   # 比成本价低 2 个最小跳动
```

> **为什么偏移 2 个 tick**：不加偏移时，保本价 = 成本价，保本兜底将止损锁定在成本价，但价格刚好在成本价时扣除交易手续费后仍可能亏损。偏移 2 tick 给了一个微小缓冲区。此前版本使用固定 ±1 元作为保本偏移量，未考虑品种价格差异（碳酸锂 1 元几乎无效，黄金 1 元过宽）。现改为按品种 `price_tick × 2` 计算，更合理。

#### 兜底逻辑

```python
if 启用保本:
    多头: if 动态止损 < 保本价: 止损 = 保本价
    空头: if 动态止损 > 保本价: 止损 = 保本价
```

> 保本只是**底线**，动态跟踪止损仍然每天计算。如果动态止损比保本价更优（多头更高），就继续用动态止损。

#### 单向开关设计

```python
protect_cost_enabled = position.protect_cost_enabled  # 从数据库读取

# 只有在"当前未启用保本"时，才检查是否满足开启条件
if not protect_cost_enabled and cost_price and position.latest_close_price and position.units >= 3:
    ...  # 满足盈利 > 2.5×ATR 时设为 True
```

- 数据库里已经是 True：直接跳过首次检查，进入兜底逻辑，不会改回 False
- 数据库里是 False：只在首次满足条件时设为 True，之后永久保持
- 开关状态只会 **0 → 1 变化一次**，不会来回跳动

### 4.2 临收盘触发（14:57，stop_loss_executor.py）

```python
def check_and_execute_stop_loss(api, account=None):
    """检查持仓止损并执行平仓。"""
```

**详细执行流程**：

1. **账户遍历**
   - 参数 `account` 不为 None 时，只处理指定账户
   - 为 None 时遍历所有 `is_active=True` 的账户

   ```python
       account=default_account,
   ```

3. **订阅实时行情**
   ```python
   for p in positions:
       quotes[p.symbol] = api.get_quote(p.symbol)
   api.wait_update(deadline=time.time() + 10)
   ```

4. **★ 实时止损重算（2026-05-14 新增）★**
   
   背景：14:57 运行时，DB 中 `stop_loss_price` 是昨日 15:32 收盘时计算的。经过一天行情波动，这个止损价可能已过时（太紧导致误触发，或太宽未能及时止损）。
   
   解决方案：对每个持仓，先用 `calculate_indicators(api, symbol)` 获取**实时** ATR 和 trend_factor，再结合 DB 中已有 `highest_close`/`lowest_close`，按收盘任务相同的公式重新计算止损价。
   
   ```python
   recalc_stop_loss = None
   try:
       indicators = calculate_indicators(api, position.symbol)
       if indicators and indicators.get('atr_20') is not None:
           fresh_atr = Decimal(str(indicators['atr_20']))
           fresh_factor = Decimal(str(indicators.get('trend_factor', 0)))
           highest_close = position.highest_close
           lowest_close = position.lowest_close

           if position.direction == 1 and highest_close:
               recalc_stop_loss = highest_close - Decimal('2') * (Decimal('1') + fresh_factor) * fresh_atr
           elif position.direction == -1 and lowest_close:
               recalc_stop_loss = lowest_close + Decimal('2') * (Decimal('1') + fresh_factor) * fresh_atr

           # 输出对比日志
           print(f"[RECALC] {position.symbol} 方向={position.direction} 现价={current_price:.2f} "
                 f"highest_close={highest_close} lowest_close={lowest_close} "
                 f"atr={fresh_atr} factor={fresh_factor} "
                 f"DB止损={float(position.stop_loss_price):.2f} 重算止损={float(recalc_stop_loss):.2f}")
   except Exception as e:
       print(f"[WARN] {position.symbol} 止损重算失败: {e}，使用原止损价")

   # 重算失败时回退到 DB 原止损价
   final_stop_loss = float(recalc_stop_loss) if recalc_stop_loss is not None else float(position.stop_loss_price)
   ```

   **公式对照**：

   | 要素 | 收盘任务 (tasks_daily_close.py) | 紧急重算 |
   |------|----------------------------------|----------|
   | 基准价 | `position.highest_close` | `position.highest_close`（相同） |
   | ATR | `indicators['atr_20']`（DB存储） | `calculate_indicators()` 实时获取 |
   | factor | `indicators['trend_factor']`（DB存储） | `calculate_indicators()` 实时获取 |
   | 止损公式 | `highest_close - 2×(1+factor)×ATR` | 完全相同 |
   | 保本兜底 | 启用（+2tick偏移） | **不启用**（留给收盘任务处理） |
   | 回退 | 不适用 | `calculate_indicators` 失败 → 原DB止损价 |

   > **为什么不做保本兜底**：紧急重算的目的是判断"现在是否应该止损"，而不是替代收盘任务的完整止损更新。14:57 距收盘仅 35 分钟，重算结果与 15:32 收盘任务的计算几乎一致。保本兜底由收盘任务在 15:32 统一处理，避免重复逻辑。

5. **触发判断**
   ```python
   triggered = False
   if position.direction == 1 and current_price < final_stop_loss:
       triggered = True
       remark = f"多头紧急止损: 实时价{current_price:.2f} < 重算止损{final_stop_loss:.2f}"
   elif position.direction == -1 and current_price > final_stop_loss:
       triggered = True
       remark = f"空头紧急止损: 实时价{current_price:.2f} > 重算止损{final_stop_loss:.2f}"
   ```
   - 使用 **实时价** 对比 **重算止损价**（非 DB 原值）
   - 重算失败时使用 DB 原止损价作为回退

6. **防重复检查**
   ```python
   if check_duplicate_pending_signal(account, symbol, 'STOP_LOSS'):
       continue  # 已有未执行止损信号，跳过
   ```

7. **创建信号 + 执行平仓**
   ```
   [信号] DailyStrategySignal.create(trade_type='STOP_LOSS', status='PENDING')
          → 立即标记 'EXECUTING'（防其他任务重复触发）
   [下单] TargetPosTask(api, symbol).set_target_volume(0)
          → 等待 is_finished() / 超时 TIMEOUT_SECONDS=60s
   [成交] 最多 3 次 wait_update 等待成交回报
          → 有成交：加权均价
          → 无成交：fallback 到 quote.last_price（以全部持仓量计算）
   [滑点] record_slippage(trade_type='STOP_LOSS', signal_price=止损价, fill_price=成交价)
   [DB]  record_and_reset_position() + 信号标记 SUCCESS (同一事务)
   [异常] 下单成功但 DB 失败 → 告警人工核对（信号仍标记 SUCCESS）
   [失败] 下单失败 → 信号标记 FAILED
   ```

8. **结果汇总**
   - `exit_count > 0`：输出今日执行数量
   - `exit_count == 0`：输出"未发现触发止损的持仓"

> **14:57 临收盘止损的意义**：防止收盘前最后几分钟价格剧烈波动，导致信号隔夜跳空时无法执行。这是一个**紧急风控**机制，与收盘后的信号生成形成双保险。

### 4.3 止损触发 - 开盘执行

收盘后 Step 8 生成的 STOP_LOSS 信号在下一个交易日的开盘任务（`job_daily_open`）中执行：

```
开盘任务 → execute_signals()
  优先级：STOP_LOSS > ROLLOVER > ADD_ON
  TargetPosTask 反向平仓
  创建 ClosedPositionRecord + 重置 PositionState
```

## 七、常见问题与排查

### 7.1 所有品种 factor 都接近 0.5

**原因**：`TREND_GAP_LIMIT` 太小，正常趋势行情下均线间距很容易超过上限。  
**解决**：调大 `TREND_GAP_LIMIT`（如 0.03 → 0.05）。

### 7.2 factor = 0 但明显有趋势

**原因**：均线未形成多头排列/空头排列（MA10/MA20/MA40 交叉）。  
**排查**：查看 print 输出的 `is_bull` / `is_bear` 和 MA 值。

### 7.3 止损价没有每天更新

**排查步骤**：
1. 检查 `PositionState.indicators` 是否为空（Step 4 是否成功）
2. 检查 `highest_close` / `lowest_close` 是否有值（Step 6）
3. 查看 `update_all_positions_stop_loss_price` 的 print 输出

### 7.4 保本机制没有生效

**排查**：
1. 检查 `cost_price` 是否有值（来自 TqApi `get_position().open_price_long/short`）
2. 盈利是否超过 `PROTECT_COST_ENABLED_RATIO × ATR`
3. 检查 `position.units >= 3` 是否满足（这条容易被忽略）
4. 检查 `protect_cost_enabled` 字段是否已更新为 True
5. 检查 `FullContractList.price_tick` 是否有值（保本价依赖 tick size）

### 7.5 止损已触发但没有平仓

**排查**：
1. 检查 `DailyStrategySignal` 是否有 `trade_type='STOP_LOSS'` 且 `executed_status='PENDING'` 的记录
2. 检查 `check_duplicate_pending_signal` 是否阻止了执行
3. 检查临收盘任务的 `TIMEOUT_SECONDS` 是否过短（默认 60s）
4. 检查账户的 TqApi 连接是否正常（登录凭证、网络）

### 7.6 临收盘止损未执行

**排查**：
1. 检查 `tasks_exit_before_close` 是否有 Redis 锁冲突（`lock:exit_before_close`）
2. 检查当天是否为交易日（`skip_if_not_trade_day` 可能跳过）
3. 检查账户 `is_active=True` 状态
4. 检查 `quote.last_price` 是否有值（商品期货无夜盘时段行情会停更）

## 八、完整计算示例（螺纹 rb2610）

假设参数：
- `TREND_GAP_LIMIT = 0.03`, `TREND_FACTOR_MAX = 0.5`
- `highest_close = 3273`, `ATR = 26.35`
- `MA10 = 3300`, `MA20 = 3270`, `MA40 = 3200`
- `tick = 1`（螺纹钢最小变动价位）

### Step 1: 趋势因子计算

```
排列: 3300 > 3270 > 3200 → 多头排列 ✓
gap_10_20 = |3300-3270|/3270 = 0.92%
gap_20_40 = |3270-3200|/3270 = 2.14%
max_gap = 2.14%

trend_strength = min(2.14% / 3%, 1.0) = 0.713
trend_factor = 0.713 × 0.5 = 0.357

label: trend_strength=0.713 ≥ 0.80? 否
       trend_strength=0.713 ≥ 0.30? 是
       → weak_bull
```

### Step 2: 止损价计算

```
止损倍数 = 2 × (1 + 0.357) = 2.714
止损距离 = 2.714 × 26.35 = 71.51

动态止损价 = 3273 - 71.51 = 3201.49
```

### Step 3: 保本检查（假设成本价 3200，units=3）

```
盈利 = 3273 - 3200 = 73
保本阈值 = 2.5 × 26.35 = 65.88
units = 3 ≥ 3 ✓

73 > 65.88 且 units≥3 → 启用保本 ✓
保本价 = 3200 + 1 × 2 = 3202

动态止损 3201.49 < 保本价 3202 → 兜底生效，最终止损 = 3202
```

### Step 4: 若 units=2（未满仓）

```
盈利 = 73 > 65.88 ✓
units = 2 < 3 ✗ → 条件二不满足，不启用保本

最终止损 = 3201.49（继续动态跟踪）
```
