# 趋势因子（Trend Factor）设计文档

> 文件位置：`backend/stock/utils/calculate_indicators.py`  
> 配置位置：`backend/stock/parameter_config.py`

---

## 一、设计目标

| 目标 | 说明 |
|------|------|
| **识别趋势/震荡** | 通过均线排列判断当前市场是否有方向性 |
| **量化趋势强度** | 用均线间距比例衡量趋势的"强劲程度" |
| **动态调整止损** | 强趋势时放大止损（让利润奔跑），弱趋势/震荡时收紧止损（保护本金） |
| **避免跳变** | 连续值设计，防止阈值附近的止损突变 |

---

## 二、核心逻辑（三步）

### Step 1: 均线排列判断

```
MA10 > MA20 > MA40 → 多头排列（有趋势）
MA10 < MA20 < MA40 → 空头排列（有趋势）
其他（交叉/粘合）   → 无趋势（震荡）
```

**为什么是排列？**  
只有三条均线同向有序排列，才说明价格运动有**一致性方向**。如果均线交叉混乱，即使短期波动很大，也视为无方向性的震荡。

---

### Step 2: 趋势强度计算

```python
max_gap = max(|MA10 - MA20|, |MA20 - MA40|) / MA20
```

- 计算两组均线间距（短-中、中-长）
- 以 **MA20** 为基准做归一化（比例形式，跨品种通用）
- 取**较大者**作为趋势强度代表——只要有一组均线在加速发散，就说明趋势在强化

---

### Step 3: 连续映射到 [0, TREND_FACTOR_MAX]

```python
trend_strength = min(max_gap / TREND_GAP_LIMIT, 1.0)
trend_factor   = trend_strength * TREND_FACTOR_MAX
```

**label 分级（与 factor 同源）**：

```python
if trend_strength >= TREND_LABEL_STRONG_RATIO:   # 0.67
    → strong_bull / strong_bear
elif trend_strength >= TREND_LABEL_WEAK_RATIO:   # 0.20
    → weak_bull / weak_bear
else:
    → choppy
```

> `trend_label` 完全基于 `trend_strength` 的比例，不需要额外的绝对阈值。`trend_factor` 和 `trend_label` 同源，不会出现"label 说强趋势但 factor 很小"的矛盾。

**这一步的本质是"归一化"**：

假设考试满分 150 分（`TREND_GAP_LIMIT = 1.5%`）：

| 实际得分 (max_gap) | 计算 | 百分比强度 |
|-------------------|------|-----------|
| 30 分 (0.3%) | 30 / 150 = **0.2** | 20% |
| 75 分 (0.75%) | 75 / 150 = **0.5** | 50% |
| 150 分 (1.5%) | 150 / 150 = **1.0** | 100% |
| 200 分 (2.0%) | 200 / 150 = 1.33 → min(..., 1.0) = **1.0** | 100%（封顶） |

**为什么封顶？**  
考 200 分还是 150 分，在"满分 150"的体系里都是满分。间距 2% 和 1.5% 都视为"完全趋势"，`trend_factor` 不再继续增加，防止极端行情下止损过大。

---

## 三、止损倍数公式

```python
from stock.parameter_config import TREND_STOP_BASE, TREND_STOP_EXTRA

stop_multiplier = TREND_STOP_BASE + trend_factor * TREND_STOP_EXTRA
```

当前配置：
- `TREND_STOP_BASE = 2.0` → 最低止损 2.0 ATR
- `TREND_STOP_EXTRA = 2.0` → 最高额外增加 1.0 ATR
- **最终范围：[2.0, 3.0] ATR**

---

## 四、关键设计决策

| 决策 | 理由 |
|------|------|
| **均线排列作为前提** | 只有均线同向排列，价格才有明确方向性。交叉/混乱时无论间距多大都不视为趋势 |
| **用间距比例而非绝对值** | 不同品种价格差异大（螺纹 3500 vs 股指 3500），比例才能跨品种通用 |
| **取 `max(gap_10_20, gap_20_40)`** | 只要有一组均线发散足够大，就说明趋势在加速，取大者更敏感 |
| **`TREND_GAP_LIMIT = 1.5%` 封顶** | 防止极端行情下因子过大导致止损过宽。1.5% 的均线间距已属于非常强趋势 |
| **连续值 `[0, 0.5]`** | 避免阶梯阈值导致的跳变问题。比如间距从 0.99% 到 1.01%，阶梯设计会让止损从 2.0 ATR 跳到 3.0 ATR，连续设计只会从 2.64 ATR 微涨到 2.68 ATR |
| **label 与 factor 同源** | 都用 `trend_strength` 判定，不需要额外的绝对阈值。调整 `TREND_GAP_LIMIT` 时 label 分级自动跟着缩放 |

## 五、配置参数说明

所有控制变量集中在 `backend/stock/parameter_config.py`：

| 变量 | 默认值 | 作用 |
|------|--------|------|
| `TREND_GAP_LIMIT` | 0.015 (1.5%) | **趋势因子封顶上限**。均线间距达到此值时，`trend_factor` 达到最大值，不再继续增加 |
| `TREND_FACTOR_MAX` | 0.5 | **趋势因子最大值**。决定 `trend_factor` 的上限 |
| `TREND_STOP_BASE` | 2.0 | **止损基准倍数**（供下游使用）。`trend_factor = 0` 时的止损倍数 |
| `TREND_STOP_EXTRA` | 2.0 | **止损额外倍数**（供下游使用）。`trend_factor` 拉满时额外增加的倍数 |
| `TREND_LABEL_STRONG_RATIO` | 0.67 | **强趋势 label 比例**。`trend_strength` ≥ 67% 时判定为强趋势（对应 `max_gap = 1.0%`） |
| `TREND_LABEL_WEAK_RATIO` | 0.20 | **弱趋势 label 比例**。`trend_strength` ≥ 20% 时判定为弱趋势（对应 `max_gap = 0.3%`） |

### 参数调整指南

| 你想达到的效果 | 调整哪个变量 | 方向 |
|---------------|------------|------|
| 让趋势因子更快达到最大值 | `TREND_GAP_LIMIT` | **调小**（如 0.015 → 0.012） |
| 扩大止损的浮动范围 | `TREND_STOP_EXTRA` | **调大**（如 2.0 → 3.0，则范围变为 [2.0, 3.5]） |
| 提高最低止损底线 | `TREND_STOP_BASE` | **调大**（如 2.0 → 2.5） |
| 更容易被判定为"强趋势" | `TREND_LABEL_STRONG_RATIO` | **调小**（如 0.67 → 0.50） |
| 更容易被判定为"有趋势" | `TREND_LABEL_WEAK_RATIO` | **调小**（如 0.20 → 0.10） |

**等价关系**（当前配置下 `TREND_GAP_LIMIT = 1.5%`）：

| `trend_strength` | 对应 `max_gap` | label |
|-----------------|----------------|-------|
| ≥ 67% | ≥ 1.0% | strong |
| ≥ 20% | ≥ 0.3% | weak |
| < 20% | < 0.3% | choppy |

> 调整 `TREND_GAP_LIMIT` 时，label 对应的绝对间距会自动跟着缩放，不需要手动同步改两个绝对阈值。 |

## 六、实盘举例（螺纹钢 rb）

假设当前 **ATR(20) = 35 元**，不同市场状态下的计算：

---

### 场景 A：强趋势（均线大幅发散）

```
MA10 = 3600, MA20 = 3550, MA40 = 3480
```

**排列判断**：`3600 > 3550 > 3480` ✓ 多头排列

**间距计算**：
```
gap_10_20 = |3600 - 3550| / 3550 = 1.41%
gap_20_40 = |3550 - 3480| / 3550 = 1.97%
max_gap   = 1.97%
```

**趋势因子**：
```
trend_strength = min(1.97% / 1.5%, 1.0) = 1.0
trend_factor   = 1.0 × 0.5 = 0.5
```

**止损设置**：
```
止损倍数 = 2.0 + 0.5 × 2.0 = 3.0 ATR
止损距离 = 3.0 × 35 = 105 元
```

> **解读**：趋势极强，给足波动空间，避免被正常回调洗出。

---

### 场景 B：中等趋势（均线适度发散）

```
MA10 = 3570, MA20 = 3550, MA40 = 3510
```

**排列判断**：`3570 > 3550 > 3510` ✓ 多头排列

**间距计算**：
```
gap_10_20 = 0.56%
gap_20_40 = 1.13%
max_gap   = 1.13%
```

**趋势因子**：
```
trend_strength = 1.13% / 1.5% = 0.753
trend_factor   = 0.753 × 0.5 = 0.377
```

**止损设置**：
```
止损倍数 = 2.0 + 0.377 × 2.0 = 2.75 ATR
止损距离 = 2.75 × 35 = 96 元
```

> **解读**：趋势明确但不如场景 A 强，止损适度放大到 2.75 ATR。

---

### 场景 C：弱趋势（均线刚刚发散）

```
MA10 = 3555, MA20 = 3550, MA40 = 3540
```

**排列判断**：`3555 > 3550 > 3540` ✓ 多头排列

**间距计算**：
```
gap_10_20 = 0.14%
gap_20_40 = 0.28%
max_gap   = 0.28%
```

**趋势因子**：
```
trend_strength = 0.28% / 1.5% = 0.187
trend_factor   = 0.187 × 0.5 = 0.094
```

**止损设置**：
```
止损倍数 = 2.0 + 0.094 × 2.0 = 2.19 ATR
止损距离 = 2.19 × 35 ≈ 77 元
```

> **解读**：趋势很弱，止损基本维持基准水平（2.19 ATR），不盲目放大。

---

### 场景 D：趋势衰竭（间距收窄）

```
MA10 = 3590, MA20 = 3580, MA40 = 3540
```

仍然多头排列，但短期涨速放缓。

**间距计算**：
```
gap_10_20 = 0.28%
gap_20_40 = 1.12%
max_gap   = 1.12%
```

**趋势因子**：
```
trend_factor = min(1.12% / 1.5%, 1.0) × 0.5 = 0.373
```

**止损设置**：
```
止损倍数 = 2.0 + 0.373 × 2.0 = 2.75 ATR
```

**对比场景 B**：
- 场景 B：`MA10=3570`，`gap_10_20=0.56%`，`gap_20_40=1.13%` → `factor=0.377`
- 场景 D：`MA10=3590`，`gap_10_20=0.28%`，`gap_20_40=1.12%` → `factor=0.373`

场景 D 的价格更高，但因为短期均线贴近（`gap_10_20` 缩小），因子自动从 0.377 微降到 0.373，止损倍数从 2.75 微降到 2.74——**平滑过渡，无跳变**。

---

### 场景 E：震荡（均线交叉/粘合）

```
MA10 = 3550, MA20 = 3552, MA40 = 3548
```

**排列判断**：
- `3550 > 3552`？否
- `3550 < 3552 < 3548`？否
- → **非多头排列，非空头排列**

**趋势因子**：
```
trend_factor = 0.0
```

**止损设置**：
```
止损倍数 = 2.0 ATR
止损距离 = 2.0 × 35 = 70 元
```

> **解读**：均线混乱，视为无趋势，止损收紧到基准 2.0 ATR。

---

## 七、连续 vs 阶梯 对比

假设螺纹 MA20=3550，均线多头排列，观察间距从 0.2% 增长到 2.0% 时止损倍数的变化：

| max_gap | **连续设计（当前）** | 阶梯设计（旧版） |
|---------|-------------------|----------------|
| 0.2%    | 2.13 ATR          | 2.00 ATR       |
| 0.5%    | 2.33 ATR          | 2.00 ATR       |
| 0.99%   | 2.66 ATR          | 2.00 ATR       |
| **1.0%**| **2.67 ATR** ⬆️ 微涨 | **3.00 ATR** ⬆️ 跳涨 |
| 1.2%    | 2.80 ATR          | 3.00 ATR       |
| 1.5%    | 3.00 ATR          | 3.00 ATR       |
| 2.0%    | 3.00 ATR（封顶）   | 3.00 ATR       |

**阶梯设计的痛点**：间距从 0.99% → 1.01%，止损突然从 2.0 ATR 跳到 3.0 ATR，多放大了 50%。这 0.02% 的差距可能是随机噪声导致的，不应该引发止损策略的剧烈变化。

**连续设计的优势**：平滑过渡，每增加一点间距就对应多一点止损空间，没有突变点。

## 八、完整计算链路图

```
┌─────────────────┐
│  K线数据(MA10,   │
│  MA20, MA40)    │
└────────┬────────┘
         ▼
┌─────────────────────────────┐
│ Step 1: 均线排列判断          │
│ 多头排列? 空头排列? 其他?      │
└────────┬────────────────────┘
         ▼
    ┌────────┐
    │  其他   │ ──→ trend_factor = 0, label = "choppy"
    └────┬───┘
         ▼
┌─────────────────────────────┐
│ Step 2: 计算最大间距比例      │
│ max_gap = max(gap_10_20,    │
│             gap_20_40)      │
└────────┬────────────────────┘
         ▼
┌─────────────────────────────┐
│ Step 3: 归一化映射           │
│ trend_strength = min(        │
│   max_gap / GAP_LIMIT, 1.0) │
│ trend_factor = trend_strength│
│   * TREND_FACTOR_MAX         │
└────────┬────────────────────┘
         ▼
┌─────────────────────────────┐
│ Step 3.5: label 分级         │
│ 基于 trend_strength 比例      │
│   ≥ STRONG_RATIO → strong    │
│   ≥ WEAK_RATIO   → weak      │
│   else           → choppy    │
└────────┬────────────────────┘
         ▼
┌─────────────────────────────┐
│ Step 4: 计算止损倍数          │
│ stop = TREND_STOP_BASE +     │
│   trend_factor *             │
│   TREND_STOP_EXTRA           │
└─────────────────────────────┘
```
